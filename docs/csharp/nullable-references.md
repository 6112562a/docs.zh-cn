# <a name="nullable-reference-types"></a><span data-ttu-id="bca88-104">可为空引用类型</span><span class="sxs-lookup"><span data-stu-id="bca88-104">Nullable reference types</span></span>

<span data-ttu-id="bca88-105">C#8.0 引入了“可为空引用类型”和“不可为空引用类型”，使你能够对引用类型变量的属性作出重要声明   ：</span><span class="sxs-lookup"><span data-stu-id="bca88-105">C# 8.0 introduces **nullable reference types** and **non-nullable reference types** that enable you to make important statements about the properties for reference type variables:</span></span>

- <span data-ttu-id="bca88-106">引用不应为 null  。</span><span class="sxs-lookup"><span data-stu-id="bca88-106">**A reference is not supposed to be null**.</span></span> <span data-ttu-id="bca88-107">当变量不应为 null 时，编译器会强制执行规则，以确保在不首先检查它们是否为 null 的情况下取消引用这些变量是安全的：</span><span class="sxs-lookup"><span data-stu-id="bca88-107">When variables aren't supposed to be null, the compiler enforces rules that ensure it is safe to dereference these variables without first checking that it isn't null:</span></span>
  - <span data-ttu-id="bca88-108">必须将变量初始化为非 null 值。</span><span class="sxs-lookup"><span data-stu-id="bca88-108">The variable must be initialized to a non-null value.</span></span>
  - <span data-ttu-id="bca88-109">变量永远不能赋值为 `null`。</span><span class="sxs-lookup"><span data-stu-id="bca88-109">The variable can never be assigned the value `null`.</span></span>
- <span data-ttu-id="bca88-110">引用可为 null  。</span><span class="sxs-lookup"><span data-stu-id="bca88-110">**A reference may be null**.</span></span> <span data-ttu-id="bca88-111">当变量可以为 null 时，编译器会强制执行不同的规则以确保你已正确检查空引用：</span><span class="sxs-lookup"><span data-stu-id="bca88-111">When variables may be null, the compiler enforces different rules to ensure that you've correctly checked for a null reference:</span></span>

  - <span data-ttu-id="bca88-113">这些变量可以使用默认的 `null` 值进行初始化，也可以在其他代码中赋值为 `null`。</span><span class="sxs-lookup"><span data-stu-id="bca88-113">These variables may be initialized with the default `null` value and may be assigned the value `null` in other code.</span></span>

<span data-ttu-id="bca88-114">在 C# 的早期版本中，无法从变量声明中确定设计意图，与处理引用变量相比，这个新功能提供了显著的好处。</span><span class="sxs-lookup"><span data-stu-id="bca88-114">This new feature provides significant benefits over the handling of reference variables in earlier versions of C# where the design intent couldn't be determined from the variable declaration.</span></span> <span data-ttu-id="bca88-115">编译器不提供针对引用类型的空引用异常的安全性：</span><span class="sxs-lookup"><span data-stu-id="bca88-115">The compiler didn't provide safety against null reference exceptions for reference types:</span></span>

- <span data-ttu-id="bca88-116">引用可为 null  。</span><span class="sxs-lookup"><span data-stu-id="bca88-116">**A reference can be null**.</span></span> <span data-ttu-id="bca88-117">将引用类型初始化为 null 或稍后将其指定为 null 时，编译器不会发出警告。</span><span class="sxs-lookup"><span data-stu-id="bca88-117">No warnings are issued when a reference type is initialized to null, or later assigned to null.</span></span>
- <span data-ttu-id="bca88-118">假定引用不为 null  。</span><span class="sxs-lookup"><span data-stu-id="bca88-118">**A reference is assumed to be not null**.</span></span> <span data-ttu-id="bca88-119">当引用类型被取消引用时，编译器不会发出任何警告。</span><span class="sxs-lookup"><span data-stu-id="bca88-119">The compiler doesn't issue any warnings when reference types are dereferenced.</span></span> <span data-ttu-id="bca88-120">（使用可为空引用，只要取消引用可能为 null 的变量，编译器就会发出警告）。</span><span class="sxs-lookup"><span data-stu-id="bca88-120">(With nullable references,  the compiler issues warnings whenever you dereference a variable that may be null).</span></span>

<span data-ttu-id="bca88-121">通过添加可为空引用类型，你可以更清楚地声明你的意图。</span><span class="sxs-lookup"><span data-stu-id="bca88-121">With the addition of nullable reference types, you can declare your intent more clearly.</span></span> <span data-ttu-id="bca88-122">`null` 值是表示变量不引用值的正确方法。</span><span class="sxs-lookup"><span data-stu-id="bca88-122">The `null` value is the correct way to represent that a variable doesn't refer to a value.</span></span> <span data-ttu-id="bca88-123">请勿使用此功能从代码中删除所有 `null` 值。</span><span class="sxs-lookup"><span data-stu-id="bca88-123">Don't use this feature to remove all `null` values from your code.</span></span> <span data-ttu-id="bca88-124">相反，应该向编译器和其他读取代码的开发人员声明你的意图。</span><span class="sxs-lookup"><span data-stu-id="bca88-124">Rather, you should declare your intent to the compiler and other developers that read your code.</span></span> <span data-ttu-id="bca88-125">通过声明意图，编译器会在你编写与该意图不一致的代码时通知你。</span><span class="sxs-lookup"><span data-stu-id="bca88-125">By declaring your intent, the compiler informs you when you write code that is inconsistent with that intent.</span></span>

<span data-ttu-id="bca88-126">使用与[可为空值类型](programming-guide/nullable-types/index.md)相同的语法记录**可为空引用类型**：将 `?` 附加到变量的类型。</span><span class="sxs-lookup"><span data-stu-id="bca88-126">A **nullable reference type** is noted using the same syntax as [nullable value types](programming-guide/nullable-types/index.md): a `?` is appended to the type of the variable.</span></span> <span data-ttu-id="bca88-127">例如，以下变量声明表示可为空的字符串变量 `name`：</span><span class="sxs-lookup"><span data-stu-id="bca88-127">For example, the following variable declaration represents a nullable string variable, `name`:</span></span>

<span data-ttu-id="bca88-128">未将 `?` 附加到类型名称的任何变量都是“不可为空引用类型”  。</span><span class="sxs-lookup"><span data-stu-id="bca88-128">Any variable where the `?` is not appended to the type name is a **non-nullable reference type**.</span></span> <span data-ttu-id="bca88-129">这包括启用此功能时现有代码中的所有引用类型变量。</span><span class="sxs-lookup"><span data-stu-id="bca88-129">That includes all reference type variables in existing code when you have enabled this feature.</span></span>

<span data-ttu-id="bca88-130">编译器使用静态分析来确定可为空引用是否为非 null。</span><span class="sxs-lookup"><span data-stu-id="bca88-130">The compiler uses static analysis to determine if a nullable reference is known to be non-null.</span></span> <span data-ttu-id="bca88-131">如果你在一个可为空引用可能是 null 时对其取消引用，编译器将向你发出警告。</span><span class="sxs-lookup"><span data-stu-id="bca88-131">The compiler warns you if you dereference a nullable reference when it may be null.</span></span> <span data-ttu-id="bca88-132">通过在变量名后面使用 null 包容运算符 (`!`) 即可覆盖此行为  。</span><span class="sxs-lookup"><span data-stu-id="bca88-132">You can override this behavior by using the **null-forgiving operator** (`!`) following a variable name.</span></span> <span data-ttu-id="bca88-133">例如，若知道 `name` 变量不为 null 但编译器仍发出警告，则可以编写以下代码来覆盖编译器的分析：</span><span class="sxs-lookup"><span data-stu-id="bca88-133">For example, if you know the `name` variable isn't null but the compiler issues a warning, you can write the following code to override the compiler's analysis:</span></span>

<span data-ttu-id="bca88-190">编译器在已启用的可为空注释上下文中使用以下规则：</span><span class="sxs-lookup"><span data-stu-id="bca88-190">The compiler uses the following rules in an enabled nullable annotation context:</span></span>

1. <span data-ttu-id="bca88-206">该变量已明确分配给非 null 值。</span><span class="sxs-lookup"><span data-stu-id="bca88-206">The variable has been definitely assigned to a non-null value.</span></span>

<span data-ttu-id="bca88-208">如果可为空警告上下文为 `enabled`，只要取消引用“可能为 null”状态的变量或表达式，编译器就会生成警告  。</span><span class="sxs-lookup"><span data-stu-id="bca88-208">The compiler generates warnings whenever you dereference a variable or expression in a **maybe null** state when the nullable warning context is `enabled`.</span></span> <span data-ttu-id="bca88-209">此外，当可为空注释上下文为 `enabled` 且将“可能为 null”变量或表达式分配给不可为空引用类型时，将生成警告  。</span><span class="sxs-lookup"><span data-stu-id="bca88-209">Furthermore, warnings are generated when a **maybe null** variable or expression is assigned to a nonnullable reference type when the nullable annotation context is `enabled`.</span></span>
