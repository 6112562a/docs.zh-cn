# <a name="compiler-warning-level-3-cs0414"></a><span data-ttu-id="09289-102">编译器警告（等级 3）CS0414</span><span class="sxs-lookup"><span data-stu-id="09289-102">Compiler Warning (level 3) CS0414</span></span>
<span data-ttu-id="09289-103">私有字段已被赋值，但从未使用过其值</span><span class="sxs-lookup"><span data-stu-id="09289-103">The private field 'field' is assigned but its value is never used</span></span>  
  
 <span data-ttu-id="09289-104">在编译器可以验证是否从未引用过变量的几种情况下，会出现此警告：</span><span class="sxs-lookup"><span data-stu-id="09289-104">This warning can occur in several scenarios in which the compiler can verify that a variable is never referenced:</span></span>  
  
- <span data-ttu-id="09289-105">私有字段被赋予一个常数值，但随后从未读取。</span><span class="sxs-lookup"><span data-stu-id="09289-105">A private field is assigned a constant value but is never subsequently read.</span></span> <span data-ttu-id="09289-106">不必要的赋值会影响性能。</span><span class="sxs-lookup"><span data-stu-id="09289-106">The unnecessary assignment could effect performance.</span></span> <span data-ttu-id="09289-107">可考虑删除字段。</span><span class="sxs-lookup"><span data-stu-id="09289-107">Consider removing the field.</span></span>  
  
- <span data-ttu-id="09289-108">私有或内部静态字段仅在初始值设定项中被赋予常量值。</span><span class="sxs-lookup"><span data-stu-id="09289-108">A private or internal static field is assigned a constant value only in the initializer.</span></span> <span data-ttu-id="09289-109">请考虑将字段更改为常量。</span><span class="sxs-lookup"><span data-stu-id="09289-109">Consider changing the field to a const.</span></span>  
  
- <span data-ttu-id="09289-110">私有或内部字段被赋予常量值，并且仅用于 #ifdef 指令排除的块。</span><span class="sxs-lookup"><span data-stu-id="09289-110">A private or internal field is assigned constant values and only used in blocks that are excluded by #ifdef directives.</span></span> <span data-ttu-id="09289-111">请考虑在 #ifdef 块中放置字段。</span><span class="sxs-lookup"><span data-stu-id="09289-111">Consider putting the field inside the #ifdef block.</span></span>  
  
- <span data-ttu-id="09289-112">在多个位置为私有或内部字段赋予常量值，但从未访问过。</span><span class="sxs-lookup"><span data-stu-id="09289-112">A private or internal field is assigned constant values in multiple locations but not otherwise accessed.</span></span> <span data-ttu-id="09289-113">如果不需要该字段，请考虑将其删除。</span><span class="sxs-lookup"><span data-stu-id="09289-113">If you do not need the field, consider removing it.</span></span> <span data-ttu-id="09289-114">否则，以某种适当的方式使用。</span><span class="sxs-lookup"><span data-stu-id="09289-114">Otherwise, use it in some appropriate way.</span></span>  
  
 <span data-ttu-id="09289-115">在其他情况下，或建议的解决方法不可接受，请使用 #pragma 0414。</span><span class="sxs-lookup"><span data-stu-id="09289-115">In other situations, or where the suggested workaround is not acceptable, use #pragma 0414.</span></span>  
  
 <span data-ttu-id="09289-116">下面的示例演示可能生成 CS0414 的一种方法：</span><span class="sxs-lookup"><span data-stu-id="09289-116">The following sample shows one way in which CS0414 will be generated:</span></span>  
  
```csharp  
// CS0414  
// compile with: /W3  
class C  
{  
   private int i = 1;  // CS0414  
  
   public static void Main()  
   { }  
}  
```  
  
 <span data-ttu-id="09289-117">**注意** 如果变量 `i` 被声明为 `protected or public`，则不会生成任何错误，因为编译器无法知道派生的类是不是会使用它，或者某些其他客户端代码是否会实例化类并引用该变量</span><span class="sxs-lookup"><span data-stu-id="09289-117">**Note** If the variable `i` is declared as `protected or public`, no error will be generated because the compiler cannot know whether a derived class might use it or some other client code might instantiate the class and reference the variable</span></span>
