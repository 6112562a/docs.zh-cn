<span data-ttu-id="1bc03-103">方法“method”的类型参数“type parameter”的约束必须与接口方法“method”的类型参数“type parameter”的约束相匹配。</span><span class="sxs-lookup"><span data-stu-id="1bc03-103">The constraints for type parameter 'type parameter' of method 'method' must match the constraints for type parameter 'type parameter' of interface method 'method'.</span></span> <span data-ttu-id="1bc03-104">请考虑改用显式接口实现。</span><span class="sxs-lookup"><span data-stu-id="1bc03-104">Consider using an explicit interface implementation instead.</span></span>  
  
 <span data-ttu-id="1bc03-105">如果一个虚泛型方法在派生类中被重写，并且该方法在派生类中的约束与它在基类中的约束不匹配，则会发生此错误。</span><span class="sxs-lookup"><span data-stu-id="1bc03-105">This error occurs if a virtual generic method is overridden in a derived class and the constraints on the method in the derived class do not match the constraints on the method in the base class.</span></span> <span data-ttu-id="1bc03-106">若要避免此错误，请确保 `where` 子句在两个声明中相同，或者显式实现接口。</span><span class="sxs-lookup"><span data-stu-id="1bc03-106">To avoid this error, make sure the `where` clause is identical in both declarations, or implement the interface explicitly.</span></span>  
  
## <a name="example"></a><span data-ttu-id="1bc03-107">示例</span><span class="sxs-lookup"><span data-stu-id="1bc03-107">Example</span></span>  
 <span data-ttu-id="1bc03-108">下面的示例生成 CS0425：</span><span class="sxs-lookup"><span data-stu-id="1bc03-108">The following example generates CS0425:</span></span>  
  
```csharp  
// CS0425.cs  
  
class C1  
{  
}  
  
class C2  
{  
}  
  
interface IBase  
{  
    void F<ItemType>(ItemType item) where ItemType : C1;  
}  
  
class Derived : IBase  
{  
    public void F<ItemType>(ItemType item) where ItemType : C2  // CS0425  
    {  
    }  
}  
  
class CMain  
{  
    public static void Main()  
    {  
    }  
}  
```
