<span data-ttu-id="4f76b-103">"class" 或 "struct" 约束必须在其他任何约束之前</span><span class="sxs-lookup"><span data-stu-id="4f76b-103">The 'class' or 'struct' constraint must come before any other constraints</span></span>  
  
 <span data-ttu-id="4f76b-104">对泛型类型或方法的类型参数的约束必须按特定顺序发生： `class` 或 `struct` 必须是第一个（如果存在），然后是任何接口约束，最后是任何构造函数约束。</span><span class="sxs-lookup"><span data-stu-id="4f76b-104">The constraints on the type parameter of a generic type or method must occur in a specific order: `class` or `struct` must be first, if present, then any interface constraints, and finally any constructor constraints.</span></span> <span data-ttu-id="4f76b-105">此错误是由于 `class` 或 `struct` 约束没有先出现所致。</span><span class="sxs-lookup"><span data-stu-id="4f76b-105">This error is caused by the `class` or `struct` constraint not appearing first.</span></span> <span data-ttu-id="4f76b-106">若要解决此错误，请重新排列约束子句。</span><span class="sxs-lookup"><span data-stu-id="4f76b-106">To resolve this error, reorder the constraint clauses.</span></span>  
  
## <a name="example"></a><span data-ttu-id="4f76b-107">示例</span><span class="sxs-lookup"><span data-stu-id="4f76b-107">Example</span></span>  
 <span data-ttu-id="4f76b-108">下面的示例生成 CS0449。</span><span class="sxs-lookup"><span data-stu-id="4f76b-108">The following sample generates CS0449.</span></span>  
  
```csharp  
// CS0449.cs  
// compile with: /target:library  
interface I {}  
public class C4   
{  
   public void F1<T>() where T : class, struct, I {}   // CS0449  
   public void F2<T>() where T : I, struct {}   // CS0449  
   public void F3<T>() where T : I, class {}   // CS0449  
  
   // OK  
   public void F4<T>() where T : class {}  
   public void F5<T>() where T : struct {}  
   public void F6<T>() where T : I {}  
}  
```
