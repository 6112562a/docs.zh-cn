# <a name="compiler-warning-level-3-cs0219"></a><span data-ttu-id="27622-102">编译器警告（等级 3） CS0219</span><span class="sxs-lookup"><span data-stu-id="27622-102">Compiler Warning (level 3) CS0219</span></span>
<span data-ttu-id="27622-103">变量 “variable” 已赋值，但其值从未使用过</span><span class="sxs-lookup"><span data-stu-id="27622-103">The variable 'variable' is assigned but its value is never used</span></span>

 <span data-ttu-id="27622-104">当你声明并给一个变量赋值却不使用该变量时，编译器会发出一个 3 级警告。</span><span class="sxs-lookup"><span data-stu-id="27622-104">The compiler issues a level-three warning, when you declare and assign a variable, but do not use it.</span></span>

 > [!NOTE]
 > <span data-ttu-id="27622-105">仅当变量值是编译时间常量时，编译器才会生成此警告。</span><span class="sxs-lookup"><span data-stu-id="27622-105">The compiler generates this warning only when the variable value is a compile-time constant.</span></span> <span data-ttu-id="27622-106">将非常量表达式或方法结果分配给局部变量可以更轻松地在调试器中观察这些表达式。</span><span class="sxs-lookup"><span data-stu-id="27622-106">Assigning a non-constant expression or method result to a local variable makes it easier to observe those expressions in the debugger.</span></span> <span data-ttu-id="27622-107">它还使结果可到达，防止垃圾回收，而该变量是可访问的。</span><span class="sxs-lookup"><span data-stu-id="27622-107">It also makes the result reachable, preventing garbage collection while that variable is reachable.</span></span>

 <span data-ttu-id="27622-108">以下示例显示未生成警告时和何时生成警告的情况：</span><span class="sxs-lookup"><span data-stu-id="27622-108">The following sample shows the cases when and when not the warning is generated:</span></span>

```csharp
// CS0219.cs
// compile with: /W:3
public class MyClass
{
   public static void Main()
   {
      var interpolated = "Interpolated";
      var a = 0; // CS0219
      int b = GetZero(); // Doesn't generate a warning.
      var c = "Regular string"; // CS0219
      var d = $"Constant interpolated string"; // Doesn't generate a warning.
      var e = $"{interpolated} string"; // Doesn't generate a warning.
   }

   private static int GetZero()
   {
       return 0;
   }
}  
```
