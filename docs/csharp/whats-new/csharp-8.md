## <a name="readonly-members"></a><span data-ttu-id="5c3a6-127">Readonly 成员</span><span class="sxs-lookup"><span data-stu-id="5c3a6-127">Readonly members</span></span>

<span data-ttu-id="5c3a6-128">可将 `readonly` 修饰符应用于结构的任何成员。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-128">You can apply the `readonly` modifier to any member of a struct.</span></span> <span data-ttu-id="5c3a6-129">它指示该成员不会修改状态。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-129">It indicates that the member does not modify state.</span></span> <span data-ttu-id="5c3a6-130">这比将 `readonly` 修饰符应用于 `struct` 声明更精细。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-130">It's more granular than applying the `readonly` modifier to a `struct` declaration.</span></span>  <span data-ttu-id="5c3a6-131">请考虑以下可变结构：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-131">Consider the following mutable struct:</span></span>

```csharp
public struct Point
{
    public double X { get; set; }
    public double Y { get; set; }
    public double Distance => Math.Sqrt(X * X + Y * Y);

    public override string ToString() =>
        $"({X}, {Y}) is {Distance} from the origin";
}
```

<span data-ttu-id="5c3a6-132">像大多数结构一样， `ToString()` 方法不会修改状态。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-132">Like most structs, the `ToString()` method does not modify state.</span></span> <span data-ttu-id="5c3a6-133">可以通过将 `readonly` 修饰符添加到 `ToString()` 的声明来对此进行指示：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-133">You could indicate that by adding the `readonly` modifier to the declaration of `ToString()`:</span></span>

```csharp
public readonly override string ToString() =>
    $"({X}, {Y}) is {Distance} from the origin";
```

<span data-ttu-id="5c3a6-134">上述更改会生成编译器警告，因为 `ToString` 访问 `Distance` 属性，该属性未标记为 `readonly`：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-134">The preceding change generates a compiler warning, because `ToString` accesses the `Distance` property, which is not marked `readonly`:</span></span>

```console
warning CS8656: Call to non-readonly member 'Point.Distance.get' from a 'readonly' member results in an implicit copy of 'this'
```

<span data-ttu-id="5c3a6-135">需要创建防御性副本时，编译器会发出警告。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-135">The compiler warns you when it needs to create a defensive copy.</span></span>  <span data-ttu-id="5c3a6-136">`Distance` 属性不会更改状态，因此可以通过将 `readonly` 修饰符添加到声明来修复此警告：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-136">The `Distance` property does not change state, so you can fix this warning by adding the `readonly` modifier to the declaration:</span></span>

```csharp

```

<span data-ttu-id="5c3a6-137">请注意，`readonly` 修饰符对于只读属性是必需的。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-137">Notice that the `readonly` modifier is necessary on a read only property.</span></span> <span data-ttu-id="5c3a6-138">编译器不会假设 `get` 访问器不修改状态；必须明确声明 `readonly`。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-138">The compiler doesn't assume `get` accessors do not modify state; you must declare `readonly` explicitly.</span></span> <span data-ttu-id="5c3a6-139">编译器会强制实施以下规则：`readonly` 成员不修改状态。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-139">The compiler does enforce the rule that `readonly` members do not modify state.</span></span> <span data-ttu-id="5c3a6-140">除非删除 `readonly` 修饰符，否则不会编译以下方法：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-140">The following method will not compile unless you remove the `readonly` modifier:</span></span>

```csharp
public readonly void Translate(int xOffset, int yOffset)
{
    X += xOffset;
    Y += yOffset;
}
```

<span data-ttu-id="5c3a6-141">通过此功能，可以指定设计意图，使编译器可以强制执行该意图，并基于该意图进行优化。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-141">This feature lets you specify your design intent so the compiler can enforce it, and make optimizations based on that intent.</span></span>

### <a name="switch-expressions"></a><span data-ttu-id="5c3a6-163">Switch 表达式</span><span class="sxs-lookup"><span data-stu-id="5c3a6-163">switch expressions</span></span>

<span data-ttu-id="5c3a6-164">通常情况下，[`switch`](../language-reference/keywords/switch.md) 语句在其每个 `case` 块中生成一个值。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-164">Often, a [`switch`](../language-reference/keywords/switch.md) statement produces a value in each of its `case` blocks.</span></span> <span data-ttu-id="5c3a6-165">借助 Switch 表达式  ，可以使用更简洁的表达式语法。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-165">**Switch expressions** enable you to use more concise expression syntax.</span></span> <span data-ttu-id="5c3a6-166">只有些许重复的 `case` 和 `break` 关键字和大括号。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-166">There are fewer repetitive `case` and `break` keywords, and fewer curly braces.</span></span>  <span data-ttu-id="5c3a6-167">以下面列出彩虹颜色的枚举为例：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-167">As an example, consider the following enum that lists the colors of the rainbow:</span></span>

```csharp
public enum Rainbow
{
    Red,
    Orange,
    Yellow,
    Green,
    Blue,
    Indigo,
    Violet
}
```

<span data-ttu-id="5c3a6-168">如果应用定义了通过 `R`、`G` 和 `B` 组件构造而成的 `RGBColor` 类型，可使用以下包含 switch 表达式的方法，将 `Rainbow` 转换为 RGB 值：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-168">If your application defined an `RGBColor` type that is constructed from the `R`, `G` and `B` components, you could convert a `Rainbow` value to its RGB values using the following method containing a switch expression:</span></span>

```csharp
public static RGBColor FromRainbow(Rainbow colorBand) =>
    colorBand switch
    {
        Rainbow.Red    => new RGBColor(0xFF, 0x00, 0x00),
        Rainbow.Orange => new RGBColor(0xFF, 0x7F, 0x00),
        Rainbow.Yellow => new RGBColor(0xFF, 0xFF, 0x00),
        Rainbow.Green  => new RGBColor(0x00, 0xFF, 0x00),
        Rainbow.Blue   => new RGBColor(0x00, 0x00, 0xFF),
        Rainbow.Indigo => new RGBColor(0x4B, 0x00, 0x82),
        Rainbow.Violet => new RGBColor(0x94, 0x00, 0xD3),
        _              => throw new ArgumentException(message: "invalid enum value", paramName: nameof(colorBand)),
    };
```

<span data-ttu-id="5c3a6-169">这里有几个语法改进：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-169">There are several syntax improvements here:</span></span>

- <span data-ttu-id="5c3a6-170">变量位于 `switch` 关键字之前。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-170">The variable comes before the `switch` keyword.</span></span> <span data-ttu-id="5c3a6-171">不同的顺序使得在视觉上可以很轻松地区分 switch 表达式和 switch 语句。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-171">The different order makes it visually easy to distinguish the switch expression from the switch statement.</span></span>
- <span data-ttu-id="5c3a6-172">将 `case` 和 `:` 元素替换为 `=>`。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-172">The `case` and `:` elements are replaced with `=>`.</span></span> <span data-ttu-id="5c3a6-173">它更简洁，更直观。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-173">It's more concise and intuitive.</span></span>
- <span data-ttu-id="5c3a6-174">将 `default` 事例替换为 `_` 弃元。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-174">The `default` case is replaced with a `_` discard.</span></span>
- <span data-ttu-id="5c3a6-175">正文是表达式，不是语句。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-175">The bodies are expressions, not statements.</span></span>

<span data-ttu-id="5c3a6-176">将其与使用经典 `switch` 语句的等效代码进行对比：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-176">Contrast that with the equivalent code using the classic `switch` statement:</span></span>

```csharp

    {
        case Rainbow.Red:
            return new RGBColor(0xFF, 0x00, 0x00);
        case Rainbow.Orange:
            return new RGBColor(0xFF, 0x7F, 0x00);
        case Rainbow.Yellow:
            return new RGBColor(0xFF, 0xFF, 0x00);
        case Rainbow.Green:
            return new RGBColor(0x00, 0xFF, 0x00);
        case Rainbow.Blue:
            return new RGBColor(0x00, 0x00, 0xFF);
        case Rainbow.Indigo:
            return new RGBColor(0x4B, 0x00, 0x82);
        case Rainbow.Violet:
            return new RGBColor(0x94, 0x00, 0xD3);
        default:
            throw new ArgumentException(message: "invalid enum value", paramName: nameof(colorBand));
    };
}

    for (int i = 0; i < 20; i++)
    {
        await Task.Delay(100);
        yield return i;
    }
}
```

    Console.WriteLine(number);

## <a name="indices-and-ranges"></a><span data-ttu-id="5c3a6-251">索引和范围</span><span class="sxs-lookup"><span data-stu-id="5c3a6-251">Indices and ranges</span></span>

<span data-ttu-id="5c3a6-252">范围和索引为在数组中指定子范围（<xref:System.Span%601> 或 <xref:System.ReadOnlySpan%601>）提供了简洁语法。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-252">Ranges and indices provide a succinct syntax for specifying subranges in an array, <xref:System.Span%601>, or <xref:System.ReadOnlySpan%601>.</span></span>

<span data-ttu-id="5c3a6-253">此语言支持依赖于两个新类型和两个新运算符。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-253">This language support relies on two new types, and two new operators.</span></span>
- <span data-ttu-id="5c3a6-254"><xref:System.Index?displayProperty=nameWithType> 表示一个序列索引。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-254"><xref:System.Index?displayProperty=nameWithType> represents an index into a sequence.</span></span>
- <span data-ttu-id="5c3a6-255">`^` 运算符，指定一个索引与序列末尾相关。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-255">The `^` operator, which specifies that an index is relative to the end of the sequence.</span></span>
- <span data-ttu-id="5c3a6-256"><xref:System.Range?displayProperty=nameWithType> 表示序列的子范围。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-256"><xref:System.Range?displayProperty=nameWithType> represents a sub range of a sequence.</span></span>
- <span data-ttu-id="5c3a6-257">范围运算符 (`..`)，用于指定范围的开始和末尾，就像操作数一样。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-257">The Range operator (`..`), which specifies the start and end of a range as its operands.</span></span>

<span data-ttu-id="5c3a6-258">让我们从索引规则开始。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-258">Let's start with the rules for indexes.</span></span> <span data-ttu-id="5c3a6-259">请考虑数组 `sequence`。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-259">Consider an array `sequence`.</span></span> <span data-ttu-id="5c3a6-260">`0` 索引与 `sequence[0]` 相同。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-260">The `0` index is the same as `sequence[0]`.</span></span> <span data-ttu-id="5c3a6-261">`^0` 索引与 `sequence[sequence.Length]` 相同。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-261">The `^0` index is the same as `sequence[sequence.Length]`.</span></span> <span data-ttu-id="5c3a6-262">请注意，`sequence[^0]` 不会引发异常，就像 `sequence[sequence.Length]` 一样。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-262">Note that `sequence[^0]` does throw an exception, just as `sequence[sequence.Length]` does.</span></span> <span data-ttu-id="5c3a6-263">对于任何数字 `n`，索引 `^n` 与 `sequence.Length - n` 相同。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-263">For any number `n`, the index `^n` is the same as `sequence.Length - n`.</span></span>

<span data-ttu-id="5c3a6-264">范围指定范围的开始和末尾   。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-264">A range specifies the *start* and *end* of a range.</span></span> <span data-ttu-id="5c3a6-265">包括此范围的开始，但不包括此范围的末尾，这表示此范围包含开始但不包含末尾。  </span><span class="sxs-lookup"><span data-stu-id="5c3a6-265">The start of the range is inclusive, but the end of the range is exclusive, meaning the *start* is included in the range but the *end* is not included in the range.</span></span> <span data-ttu-id="5c3a6-266">范围 `[0..^0]` 表示整个范围，就像 `[0..sequence.Length]` 表示整个范围。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-266">The range `[0..^0]` represents the entire range, just as `[0..sequence.Length]` represents the entire range.</span></span> 

<span data-ttu-id="5c3a6-267">请看以下几个示例。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-267">Let's look at a few examples.</span></span> <span data-ttu-id="5c3a6-268">请考虑以下数组，用其顺数索引和倒数索引进行注释：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-268">Consider the following array, annotated with its index from the start and from the end:</span></span>

<span data-ttu-id="5c3a6-269">可以使用 `^1` 索引检索最后一个词：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-269">You can retrieve the last word with the `^1` index:</span></span>

<span data-ttu-id="5c3a6-270">以下代码创建了一个包含单词“quick”、“brown”和“fox”的子范围。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-270">The following code creates a subrange with the words "quick", "brown", and "fox".</span></span> <span data-ttu-id="5c3a6-271">它包括 `words[1]` 到 `words[3]`。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-271">It includes `words[1]` through `words[3]`.</span></span> <span data-ttu-id="5c3a6-272">元素 `words[4]` 不在此范围内。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-272">The element `words[4]` is not in the range.</span></span>

<span data-ttu-id="5c3a6-273">以下代码使用“lazy”和“dog”创建一个子范围。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-273">The following code creates a subrange with "lazy" and "dog".</span></span> <span data-ttu-id="5c3a6-274">它包括 `words[^2]` 和 `words[^1]`。</span><span class="sxs-lookup"><span data-stu-id="5c3a6-274">It includes `words[^2]` and `words[^1]`.</span></span> <span data-ttu-id="5c3a6-275">不包括结束索引 `words[^0]`：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-275">The end index `words[^0]` is not included:</span></span>

<span data-ttu-id="5c3a6-276">下面的示例为开始和/或结束创建了开放范围：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-276">The following examples create ranges that are open ended for the start, end, or both:</span></span>

<span data-ttu-id="5c3a6-277">此外可以将范围声明为变量：</span><span class="sxs-lookup"><span data-stu-id="5c3a6-277">You can also declare ranges as variables:</span></span>
