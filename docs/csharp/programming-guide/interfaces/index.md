<span data-ttu-id="5c7f1-103">接口包含[类](../../language-reference/keywords/class.md)或[结构](../../language-reference/keywords/struct.md)可以实现的一组相关功能的定义。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-103">An interface contains definitions for a group of related functionalities that a [class](../../language-reference/keywords/class.md) or a [struct](../../language-reference/keywords/struct.md) can implement.</span></span>
  
<span data-ttu-id="5c7f1-104">例如，使用接口可以在类中包括来自多个源的行为。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-104">By using interfaces, you can, for example, include behavior from multiple sources in a class.</span></span> <span data-ttu-id="5c7f1-105">该功能在 C# 中十分重要，因为该语言不支持类的多重继承。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-105">That capability is important in C# because the language doesn't support multiple inheritance of classes.</span></span> <span data-ttu-id="5c7f1-106">此外，如果要模拟结构的继承，也必须使用接口，因为它们无法实际从另一个结构或类继承。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-106">In addition, you must use an interface if you want to simulate inheritance for structs, because they can't actually inherit from another struct or class.</span></span>

<span data-ttu-id="5c7f1-111">实现 <xref:System.IEquatable%601> 接口的任何类或结构都必须包含与该接口指定的签名匹配的 <xref:System.IEquatable%601.Equals%2A> 方法的定义。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-111">Any class or struct that implements the <xref:System.IEquatable%601> interface must contain a definition for an <xref:System.IEquatable%601.Equals%2A> method that matches the signature that the interface specifies.</span></span> <span data-ttu-id="5c7f1-112">因此，可以依靠实现 `IEquatable<T>` 的类来包含 `Equals` 方法，类的实例可以通过该方法确定它是否等于相同类的另一个实例。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-112">As a result, you can count on a class that implements `IEquatable<T>` to contain an `Equals` method with which an instance of the class can determine whether it's equal to another instance of the same class.</span></span>  
  
<span data-ttu-id="5c7f1-113">`IEquatable<T>` 的定义不为 `Equals` 提供实现。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-113">The definition of `IEquatable<T>` doesn’t provide an implementation for `Equals`.</span></span> <span data-ttu-id="5c7f1-114">该接口仅定义签名。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-114">The interface defines only the signature.</span></span> <span data-ttu-id="5c7f1-115">这样，C# 中的接口便类似于其中所有方法都是抽象方法的抽象类。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-115">In that way, an interface in C# is similar to an abstract class in which all the methods are abstract.</span></span> <span data-ttu-id="5c7f1-116">但是，类或结构可以实现多个接口，但是类只能继承单个类（抽象或不抽象）。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-116">However, a class or struct can implement multiple interfaces, but a class can inherit only a single class, abstract or not.</span></span> <span data-ttu-id="5c7f1-117">因此，使用接口可以在类中包括来自多个源的行为。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-117">Therefore, by using interfaces, you can include behavior from multiple sources in a class.</span></span>  
  
<span data-ttu-id="5c7f1-118">有关抽象类的详细信息，请参阅[抽象类、密封类及类成员](../classes-and-structs/abstract-and-sealed-classes-and-class-members.md)。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-118">For more information about abstract classes, see [Abstract and Sealed Classes and Class Members](../classes-and-structs/abstract-and-sealed-classes-and-class-members.md).</span></span>  
  
<span data-ttu-id="5c7f1-119">接口可以包含方法、属性、事件、索引器或这四种成员类型的任意组合。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-119">Interfaces can contain methods, properties, events, indexers, or any combination of those four member types.</span></span> <span data-ttu-id="5c7f1-120">有关示例的链接，请参阅[相关部分](../interfaces/index.md#BKMK_RelatedSections)。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-120">For links to examples, see [Related Sections](../interfaces/index.md#BKMK_RelatedSections).</span></span> <span data-ttu-id="5c7f1-121">接口不能包含常量、字段、运算符、实例构造函数、终结器或类型。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-121">An interface can't contain constants, fields, operators, instance constructors, finalizers, or types.</span></span> <span data-ttu-id="5c7f1-122">接口成员会自动成为公共成员，不能包含任何访问修饰符。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-122">Interface members are automatically public, and they can't include any access modifiers.</span></span> <span data-ttu-id="5c7f1-123">成员也不能是[静态](../../language-reference/keywords/static.md)成员。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-123">Members also can't be [static](../../language-reference/keywords/static.md).</span></span>  
  
<span data-ttu-id="5c7f1-124">若要实现接口成员，实现类的对应成员必须是公共、非静态，并且具有与接口成员相同的名称和签名。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-124">To implement an interface member, the corresponding member of the implementing class must be public, non-static, and have the same name and signature as the interface member.</span></span>  
  
<span data-ttu-id="5c7f1-125">当类或结构实现接口时，类或结构必须为该接口定义的所有成员提供实现。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-125">When a class or struct implements an interface, the class or struct must provide an implementation for all of the members that the interface defines.</span></span> <span data-ttu-id="5c7f1-126">接口本身不提供类或结构可以通过继承基类功能的方式来继承的任何功能。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-126">The interface itself provides no functionality that a class or struct can inherit in the way that it can inherit base class functionality.</span></span> <span data-ttu-id="5c7f1-127">但是，如果基类实现接口，则从基类派生的任何类都会继承该实现。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-127">However, if a base class implements an interface, any class that's derived from the base class inherits that implementation.</span></span>

<span data-ttu-id="5c7f1-130">类的属性和索引器可以为接口中定义的属性或索引器定义额外的访问器。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-130">Properties and indexers of a class can define extra accessors for a property or indexer that's defined in an interface.</span></span> <span data-ttu-id="5c7f1-131">例如，接口可能会声明包含 [get](../../language-reference/keywords/get.md) 取值函数的属性。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-131">For example, an interface might declare a property that has a [get](../../language-reference/keywords/get.md) accessor.</span></span> <span data-ttu-id="5c7f1-132">实现此接口的类可以声明包含 `get` 和 [set](../../language-reference/keywords/set.md) 取值函数的同一属性。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-132">The class that implements the interface can declare the same property with both a `get` and [set](../../language-reference/keywords/set.md) accessor.</span></span> <span data-ttu-id="5c7f1-133">但是，如果属性或索引器使用显式实现，则访问器必须匹配。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-133">However, if the property or indexer uses explicit implementation, the accessors must match.</span></span> <span data-ttu-id="5c7f1-134">有关显式实现的详细信息，请参阅[显式接口实现](explicit-interface-implementation.md)和[接口属性](../classes-and-structs/interface-properties.md)。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-134">For more information about explicit implementation, see [Explicit Interface Implementation](explicit-interface-implementation.md) and [Interface Properties](../classes-and-structs/interface-properties.md).</span></span>

<span data-ttu-id="5c7f1-140">基类还可以使用虚拟成员实现接口成员。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-140">A base class can also implement interface members by using virtual members.</span></span> <span data-ttu-id="5c7f1-141">在这种情况下，派生类可以通过重写虚拟成员来更改接口行为。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-141">In that case, a derived class can change the interface behavior by overriding the virtual members.</span></span> <span data-ttu-id="5c7f1-142">有关虚拟成员的详细信息，请参阅[多态性](../classes-and-structs/polymorphism.md)。</span><span class="sxs-lookup"><span data-stu-id="5c7f1-142">For more information about virtual members, see [Polymorphism](../classes-and-structs/polymorphism.md).</span></span>
