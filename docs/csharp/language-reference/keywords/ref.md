<span data-ttu-id="39888-177">将 `ref` 修饰符添加到 `struct` 声明定义了该类型的实例必须为堆栈分配。</span><span class="sxs-lookup"><span data-stu-id="39888-177">Adding the `ref` modifier to a `struct` declaration defines that instances of that type must be stack allocated.</span></span> <span data-ttu-id="39888-178">换言之，永远不能在作为另一类的成员的堆上创建这些类型的实例。</span><span class="sxs-lookup"><span data-stu-id="39888-178">In other words, instances of these types can never be created on the heap as a member of another class.</span></span> <span data-ttu-id="39888-179">此功能的主要动机是 <xref:System.Span%601> 和相关结构。</span><span class="sxs-lookup"><span data-stu-id="39888-179">The primary motivation for this feature was <xref:System.Span%601> and related structures.</span></span>

<span data-ttu-id="39888-180">保持 `ref struct` 类型作为堆栈分配的变量的目标引入了几条编译器针对所有 `ref struct` 类型强制执行的规则。</span><span class="sxs-lookup"><span data-stu-id="39888-180">The goal of keeping a `ref struct` type as a stack-allocated variable introduces several rules that the compiler enforces for all `ref struct` types.</span></span>

- <span data-ttu-id="39888-181">不能对 `ref struct` 装箱。</span><span class="sxs-lookup"><span data-stu-id="39888-181">You can't box a `ref struct`.</span></span> <span data-ttu-id="39888-182">无法向属于 `object`、`dynamic` 或任何接口类型的变量分配 `ref struct` 类型。</span><span class="sxs-lookup"><span data-stu-id="39888-182">You cannot assign a `ref struct` type to a variable of type `object`, `dynamic`, or any interface type.</span></span>
- <span data-ttu-id="39888-183">`ref struct` 类型不能实现接口。</span><span class="sxs-lookup"><span data-stu-id="39888-183">`ref struct` types cannot implement interfaces.</span></span>
- <span data-ttu-id="39888-184">不能将 `ref struct` 声明为类或常规结构的字段成员。</span><span class="sxs-lookup"><span data-stu-id="39888-184">You can't declare a `ref struct` as a field member of a class or a normal struct.</span></span> <span data-ttu-id="39888-185">这包括声明自动实现的属性，后者会创建一个由编译器生成的支持字段。</span><span class="sxs-lookup"><span data-stu-id="39888-185">This includes declaring an auto-implemented property, which creates a compiler generated backing field.</span></span> 
- <span data-ttu-id="39888-186">不能声明异步方法中属于 `ref struct` 类型的本地变量。</span><span class="sxs-lookup"><span data-stu-id="39888-186">You cannot declare local variables that are `ref struct` types in async methods.</span></span> <span data-ttu-id="39888-187">不能在返回类似 <xref:System.Threading.Tasks.Task>、<xref:System.Threading.Tasks.Task%601> 或 `Task` 类型的同步方法中声明它们。</span><span class="sxs-lookup"><span data-stu-id="39888-187">You can declare them in synchronous methods that return <xref:System.Threading.Tasks.Task>, <xref:System.Threading.Tasks.Task%601> or `Task`-like types.</span></span>
- <span data-ttu-id="39888-188">无法在迭代器中声明 `ref struct` 本地变量。</span><span class="sxs-lookup"><span data-stu-id="39888-188">You cannot declare `ref struct` local variables in iterators.</span></span>
- <span data-ttu-id="39888-189">无法捕获 Lambda 表达式或本地函数中的 `ref struct` 变量。</span><span class="sxs-lookup"><span data-stu-id="39888-189">You cannot capture `ref struct` variables in lambda expressions or local functions.</span></span>

<span data-ttu-id="39888-190">这些限制可确保不会以可提升至托管堆的方式意外地使用 `ref struct`。</span><span class="sxs-lookup"><span data-stu-id="39888-190">These restrictions ensure you don't accidentally use a `ref struct` in a manner that could promote it to the managed heap.</span></span>

<span data-ttu-id="39888-191">可以组合修饰符以将结构声明为 `readonly ref`。</span><span class="sxs-lookup"><span data-stu-id="39888-191">You can combine modifiers to declare a struct as `readonly ref`.</span></span> <span data-ttu-id="39888-192">`readonly ref struct` 兼具 `ref struct` 和 `readonly struct` 声明的优点和限制。</span><span class="sxs-lookup"><span data-stu-id="39888-192">A `readonly ref struct` combines the benefits and restrictions of `ref struct` and `readonly struct` declarations.</span></span>
