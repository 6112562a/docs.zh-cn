 <span data-ttu-id="772dc-110">作为 `in` 参数传递的变量在方法调用中传递之前必须进行初始化。</span><span class="sxs-lookup"><span data-stu-id="772dc-110">Variables passed as `in` arguments must be initialized before being passed in a method call.</span></span> <span data-ttu-id="772dc-111">但是，所调用的方法可能不会分配值或修改参数。</span><span class="sxs-lookup"><span data-stu-id="772dc-111">However, the called method may not assign a value or modify the argument.</span></span>  
  
 <span data-ttu-id="772dc-112">尽管 `in`、`ref` 和 `out` 关键字会导致不同的运行时行为，它们并不被视为编译时方法签名的一部分。</span><span class="sxs-lookup"><span data-stu-id="772dc-112">Although the `in`, `ref`, and `out` keywords cause different run-time behavior, they are not considered part of the method signature at compile time.</span></span> <span data-ttu-id="772dc-113">因此，如果唯一的不同是一个方法采用 `ref` 或 `in` 参数，而另一个方法采用 `out` 参数，则无法重载这两个方法。</span><span class="sxs-lookup"><span data-stu-id="772dc-113">Therefore, methods cannot be overloaded if the only difference is that one method takes a `ref` or `in` argument and the other takes an `out` argument.</span></span> <span data-ttu-id="772dc-114">例如，以下代码将不会编译：</span><span class="sxs-lookup"><span data-stu-id="772dc-114">The following code, for example, will not compile:</span></span>

<span data-ttu-id="772dc-115">存在 `in` 时可以重载：</span><span class="sxs-lookup"><span data-stu-id="772dc-115">Overloading based on the presence of `in` is allowed:</span></span>

## <a name="overload-resolution-rules"></a><span data-ttu-id="772dc-116">重载决策规则</span><span class="sxs-lookup"><span data-stu-id="772dc-116">Overload resolution rules</span></span>

<span data-ttu-id="772dc-117">通过理解使用 `in` 参数的动机，可以理解使用按值方法和使用 `in` 参数方法的重载决策规则。</span><span class="sxs-lookup"><span data-stu-id="772dc-117">You can understand the overload resolution rules for methods with by value vs. `in` arguments by understanding the motivation for `in` arguments.</span></span> <span data-ttu-id="772dc-118">定义使用 `in` 参数的方法是一项潜在的性能优化。</span><span class="sxs-lookup"><span data-stu-id="772dc-118">Defining methods using `in` parameters is a potential performance optimization.</span></span> <span data-ttu-id="772dc-119">某些 `struct` 类型参数可能很大，在紧凑的循环或关键代码路径中调用方法时，复制这些结构的成本就很高。</span><span class="sxs-lookup"><span data-stu-id="772dc-119">Some `struct` type arguments may be large in size, and when methods are called in tight loops or critical code paths, the cost of copying those structures is critical.</span></span> <span data-ttu-id="772dc-120">方法声明 `in` 参数以指定参数可能按引用安全传递，因为所调用的方法不修改该参数的状态。</span><span class="sxs-lookup"><span data-stu-id="772dc-120">Methods declare `in` parameters to specify that arguments may be passed by reference safely because the called method does not modify the state of that argument.</span></span> <span data-ttu-id="772dc-121">按引用传递这些参数可以避免（可能产生的）高昂的复制成本。</span><span class="sxs-lookup"><span data-stu-id="772dc-121">Passing those arguments by reference avoids the (potentially) expensive copy.</span></span> 

<span data-ttu-id="772dc-122">为调用站点上的参数指定 `in` 通常为可选。</span><span class="sxs-lookup"><span data-stu-id="772dc-122">Specifying `in` for arguments at the call site is typically optional.</span></span> <span data-ttu-id="772dc-123">按值传递参数和使用 `in` 修饰符按引用传递参数这两种方法并没有语义差异。</span><span class="sxs-lookup"><span data-stu-id="772dc-123">There is no semantic difference between passing arguments by value and passing them by reference using the `in` modifier.</span></span> <span data-ttu-id="772dc-124">可以在调用站点选择 `in` 修饰符，因为你不需要指出参数值可能会改变。</span><span class="sxs-lookup"><span data-stu-id="772dc-124">The `in` modifier at the call site is optional because you don't need to indicate that the argument's value might be changed.</span></span> <span data-ttu-id="772dc-125">在调用站点显式添加 `in` 修饰符以确保参数是按引用传递，而不是按值传递。</span><span class="sxs-lookup"><span data-stu-id="772dc-125">You explicitly add the `in` modifier at the call site to ensure the argument is passed by reference, not by value.</span></span> <span data-ttu-id="772dc-126">显式使用 `in` 有以下两个效果：</span><span class="sxs-lookup"><span data-stu-id="772dc-126">Explicitly using `in` has the following two effects:</span></span>

<span data-ttu-id="772dc-127">首先，在调用站点指定 `in` 会强制编译器选择使用匹配的 `in` 参数定义的方法。</span><span class="sxs-lookup"><span data-stu-id="772dc-127">First, specifying `in` at the call site forces the compiler to select a method defined with a matching `in` parameter.</span></span> <span data-ttu-id="772dc-128">否则，如果两种方法唯一的区别在于是否存在 `in`，则按值重载的匹配度会更高。</span><span class="sxs-lookup"><span data-stu-id="772dc-128">Otherwise, when two methods differ only in the presence of `in`, the by value overload is a better match.</span></span>

<span data-ttu-id="772dc-129">第二点，指定 `in` 会声明你想按引用传递参数。</span><span class="sxs-lookup"><span data-stu-id="772dc-129">Second, specifying `in` declares your intent to pass an argument by reference.</span></span> <span data-ttu-id="772dc-130">结合 `in` 使用的参数必须代表一个可以直接引用的位置。</span><span class="sxs-lookup"><span data-stu-id="772dc-130">The argument used with `in` must represent a location that can be directly referred to.</span></span> <span data-ttu-id="772dc-131">`out` 和 `ref` 参数的相同常规规则适用：不得使用常量、普通属性或其他生成值的表达式。</span><span class="sxs-lookup"><span data-stu-id="772dc-131">The same general rules for `out` and `ref` arguments apply: You cannot use constants, ordinary properties, or other expressions that produce values.</span></span> <span data-ttu-id="772dc-132">否则，在调用站点省略 `in` 就会通知编译器你将允许它创建临时变量，并按只读引用传递至方法。</span><span class="sxs-lookup"><span data-stu-id="772dc-132">Otherwise, omitting `in` at the call site informs the compiler that you will allow it to create a temporary variable to pass by read-only reference to the method.</span></span> <span data-ttu-id="772dc-133">编译器创建临时变量以克服一些 `in` 参数的限制：</span><span class="sxs-lookup"><span data-stu-id="772dc-133">The compiler creates a temporary variable to overcome several restrictions with `in` arguments:</span></span>

- <span data-ttu-id="772dc-134">临时变量允许将编译时常数作为 `in` 参数。</span><span class="sxs-lookup"><span data-stu-id="772dc-134">A temporary variable allows compile-time constants as `in` parameters.</span></span>
- <span data-ttu-id="772dc-135">临时变量允许使用属性或 `in` 参数的其他表达式。</span><span class="sxs-lookup"><span data-stu-id="772dc-135">A temporary variable allows properties, or other expressions for `in` parameters.</span></span>
- <span data-ttu-id="772dc-136">存在从实参类型到形参类型的隐式转换时，临时变量允许使用实参。</span><span class="sxs-lookup"><span data-stu-id="772dc-136">A temporary variable allows arguments where there is an implicit conversion from the argument type to the parameter type.</span></span>

<span data-ttu-id="772dc-137">在前面的所有实例中，编译器创建了临时变量，用于存储常数、属性或其他表达式的值。</span><span class="sxs-lookup"><span data-stu-id="772dc-137">In all the preceding instances, the compiler creates a temporary variable that stores the value of the constant, property, or other expression.</span></span>

<span data-ttu-id="772dc-138">以下代码阐释了这些规则：</span><span class="sxs-lookup"><span data-stu-id="772dc-138">The following code illustrates these rules:</span></span>

```csharp

{
    // implementation removed

```

<span data-ttu-id="772dc-139">现在，假设可以使用另一种使用按值参数的方法。</span><span class="sxs-lookup"><span data-stu-id="772dc-139">Now, suppose another method using by value arguments was available.</span></span> <span data-ttu-id="772dc-140">结果的变化如以下代码所示：</span><span class="sxs-lookup"><span data-stu-id="772dc-140">The results change as shown in the following code:</span></span>

```csharp

    // implementation removed

{
    // implementation removed
}

Method(5); // Calls overload passed by value

Method(s); // Calls overload passed by value.
Method(in s); // CS1503: cannot convert from in short to in int

Method(i); // Calls overload passed by value

```

<span data-ttu-id="772dc-141">最后一个是按引用传递参数的唯一方法调用。</span><span class="sxs-lookup"><span data-stu-id="772dc-141">The only method call where the argument is passed by reference is the final one.</span></span>

> [!NOTE]
> <span data-ttu-id="772dc-142">为了简化操作，前面的代码将 `int` 用作参数类型。</span><span class="sxs-lookup"><span data-stu-id="772dc-142">The preceding code uses `int` as the argument type for simplicity.</span></span> <span data-ttu-id="772dc-143">因为大多数新式计算机中的引用都比 `int` 大，所以将单个 `int` 作为只读引用传递没有任何好处。</span><span class="sxs-lookup"><span data-stu-id="772dc-143">Because `int` is no larger than a reference in most modern machines, there is no benefit to passing a single `int` as a readonly reference.</span></span>
