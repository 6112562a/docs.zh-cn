# <a name="compiler-error-cs0826"></a><span data-ttu-id="669f7-102">编译器错误 CS0826</span><span class="sxs-lookup"><span data-stu-id="669f7-102">Compiler Error CS0826</span></span>
<span data-ttu-id="669f7-103">未找到隐式类型的数组的最佳类型。</span><span class="sxs-lookup"><span data-stu-id="669f7-103">No best type found for implicitly typed array.</span></span>  
  
 <span data-ttu-id="669f7-104">根据编译器使用的类型推理规则，数组元素必须都采用相同的类型或隐式转换为相同的类型。</span><span class="sxs-lookup"><span data-stu-id="669f7-104">Array elements must all be the same type or implicitly convertible to the same type according to the type inference rules used by the compiler.</span></span> <span data-ttu-id="669f7-105">最佳类型必须是数组表达式中存在的其中一个类型。</span><span class="sxs-lookup"><span data-stu-id="669f7-105">The best type must be one of the types present in the array expression.</span></span> <span data-ttu-id="669f7-106">元素不会转换为新类型，如 `object`。</span><span class="sxs-lookup"><span data-stu-id="669f7-106">Elements will not be converted to a new type such as `object`.</span></span> <span data-ttu-id="669f7-107">对于隐式类型的数组，编译器必须基于分配给它的元素类型推断数组类型。</span><span class="sxs-lookup"><span data-stu-id="669f7-107">For an implicitly typed array, the compiler must infer the array type based on the type of elements assigned to it.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="669f7-108">更正此错误</span><span class="sxs-lookup"><span data-stu-id="669f7-108">To correct this error</span></span>

- <span data-ttu-id="669f7-110">为所有数组元素指定相同的类型。</span><span class="sxs-lookup"><span data-stu-id="669f7-110">Give all array elements the same type.</span></span>

## <a name="example"></a><span data-ttu-id="669f7-112">示例</span><span class="sxs-lookup"><span data-stu-id="669f7-112">Example</span></span>  
 <span data-ttu-id="669f7-113">下面的代码生成 CS0826，因为数组元素并不都采用相同的类型，并且编译器的类型推理逻辑未找到一个最佳类型：</span><span class="sxs-lookup"><span data-stu-id="669f7-113">The following code generates CS0826 because the array elements are not all the same type, and the compiler's type inference logic does not find a single best type:</span></span>  
  
```csharp  
// cs0826.cs  
public class C  
{  
    public static int Main()  
    {  
        var x = new[] { 1, "str" }; // CS0826  
  
        char c = 'c';  
        short s1 = 0;  
        short s2 = -0;  
        short s3 = 1;  
        short s4 = -1;  
  
        var array1 = new[] { s1, s2, s3, s4, c, '1' }; // CS0826  
        return 1;  
    }  
}  
```
