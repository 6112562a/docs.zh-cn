# <a name="native-wpf-browser-hosting-support-apis"></a>承载支持 API 的本机 WPF 浏览器
从 WPF 主机注册的 Active Document 服务器（也称为 DocObject）有助于在 Web 浏览器中托管 WPF 应用程序。 Internet Explorer 可以直接激活活动文档并与其集成。 对于 Mozilla 浏览器中 XBAP 和松散 XAML 文档的托管，WPF 提供了一个 NPAPI 插件，它提供了与 Internet Explorer 类似的 WPF 活动文档服务器的托管环境。 但是，在其他浏览器和独立应用程序中托管 XBAPs 和 XAML 文档的最简单实用方法是通过 Internet 资源管理器 Web 浏览器控件。 Web 浏览器控件提供复杂的活动文档服务器托管环境，但它允许自己的主机自定义和扩展该环境，并直接与当前活动文档对象通信。  
  
 WPF 活动文档服务器实现了几个常见的托管接口，包括 IOleObject、IOleDocument、IOleInPlaceActiveobject、IPersistMoniker、IOleCommandTarget。 [IOleObject](/windows/win32/api/oleidl/nn-oleidl-ioleobject) [IOleDocument](/windows/win32/api/docobj/nn-docobj-ioledocument) [IOleInPlaceActiveObject](/windows/win32/api/oleidl/nn-oleidl-ioleinplaceactiveobject) [IPersistMoniker](https://docs.microsoft.com/previous-versions/windows/internet-explorer/ie-developer/platform-apis/ms775042(v=vs.85)) [IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget) 在 Web 浏览器控件中托管时，这些接口可以是[IWebBrowser2：:Document](https://docs.microsoft.com/previous-versions/aa752116(v=vs.85))属性返回的对象中的查询。  
  
## <a name="iolecommandtarget"></a>IOleCommand目标  
 WPF 活动文档服务器对[IOleCommandTarget](/windows/win32/api/docobj/nn-docobj-iolecommandtarget)的实现支持标准 OLE 命令组（具有空命令组 GUID）的许多与导航相关的浏览器特定命令。 此外，它识别名为CGID_PresentationHost的自定义命令组。 目前，此组中仅定义了一个命令。  
  
```cpp  
DEFINE_GUID(CGID_PresentationHost, 0xd0288c55, 0xd6, 0x4f5e, 0xa8, 0x51, 0x79, 0xde, 0xc5, 0x1b, 0x10, 0xec);  
enum PresentationHostCommands {
   PHCMDID_TABINTO = 1
};  
```  
  
 PHCMDID_TABINTO指示演示主机根据 Shift 键的状态将焦点切换到其内容中的第一个或最后一个可聚焦元素。
