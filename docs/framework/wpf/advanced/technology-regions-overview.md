## <a name="regions"></a><span data-ttu-id="fa7d9-105">区域</span><span class="sxs-lookup"><span data-stu-id="fa7d9-105">Regions</span></span>  
 <span data-ttu-id="fa7d9-106">在顶级窗口内，可以这么想：每个包含互操作应用程序某个技术的 HWND 都具有自己的区域（也称为“空域”）。</span><span class="sxs-lookup"><span data-stu-id="fa7d9-106">Within a top-level window, you can conceptualize that [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] 区域，但为了便于讨论，可以假定只有一个区域。）</span><span class="sxs-lookup"><span data-stu-id="fa7d9-108">(Strictly speaking, there is more than one [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] region if there is more than one [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] HWND, but for purposes of this discussion, you can assume there is only one).</span></span> <span 并通过互操作 api 尽可能多地禁用。</span><span class="sxs-lookup"><span data-stu-id="fa7d9-110">Attempting to render [!INCLUDE[TLA2#tla_winclient](../../../../includes/tla2sharptla-winclient-md.md)] pixels over [!INCLUDE[TLA2#tla_win32](../../../../includes/tla2sharptla-win32-md.md)] leads to undesirable results, and is disallowed as much as possible through the interoperation APIs.</span></span>
