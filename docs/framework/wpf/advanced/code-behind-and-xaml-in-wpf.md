- <span data-ttu-id="3a2d3-120">请注意，在标记编译生成操作的默认行为下，可以将派生留空于代码后面的部分类定义中。</span><span class="sxs-lookup"><span data-stu-id="3a2d3-120">Note that under the default behavior of the markup compile build actions, you can leave the derivation blank in the partial class definition on the code-behind side.</span></span> <span data-ttu-id="3a2d3-121">编译的结果将假定页面根的背衬类型是部分类的基础，即使未指定。</span><span class="sxs-lookup"><span data-stu-id="3a2d3-121">The compiled result will assume the page root's backing type to be the basis for the partial class, even if it not specified.</span></span> <span data-ttu-id="3a2d3-122">但是，依赖此行为不是最佳做法。</span><span class="sxs-lookup"><span data-stu-id="3a2d3-122">However, relying on this behavior is not a best practice.</span></span>

- <span data-ttu-id="3a2d3-126">处理程序必须匹配支持类型系统中相应事件的委托。</span><span class="sxs-lookup"><span data-stu-id="3a2d3-126">The handler must match the delegate for the appropriate event in the backing type system.</span></span>
