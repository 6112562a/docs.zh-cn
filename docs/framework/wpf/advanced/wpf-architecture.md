## <a name="systemthreadingdispatcherobject"></a><span data-ttu-id="6cd7d-120">System.Threading.DispatcherObject</span><span class="sxs-lookup"><span data-stu-id="6cd7d-120">System.Threading.DispatcherObject</span></span>  
 <span data-ttu-id="6cd7d-121">WPF 中的大多数对象派生<xref:System.Windows.Threading.DispatcherObject>自，它提供了处理并发和线程的基本构造。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-121">Most objects in WPF derive from <xref:System.Windows.Threading.DispatcherObject>, which provides the basic constructs for dealing with concurrency and threading.</span></span> <span data-ttu-id="6cd7d-122">WPF 基于调度程序实现的消息传递系统。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-122">WPF is based on a messaging system implemented by the dispatcher.</span></span> <span data-ttu-id="6cd7d-123">这的工作原理很像熟悉的Win32消息泵;事实上，WPF 调度程序使用 User32 消息执行交叉线程调用。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-123">This works much like the familiar Win32 message pump; in fact, the WPF dispatcher uses User32 messages for performing cross thread calls.</span></span>  
  
 <span data-ttu-id="6cd7d-124">在 WPF 中讨论并发性时，实际上需要了解两个核心概念 - 调度器和线程相关性。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-124">There are really two core concepts to understand when discussing concurrency in WPF – the dispatcher and thread affinity.</span></span>  
  
 <span data-ttu-id="6cd7d-125">在 WPF 的设计阶段，目标是移动到单个执行线程，但采用非线程"关联化"模型。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-125">During the design phase of WPF, the goal was to move to a single thread of execution, but a non-thread "affinitized" model.</span></span> <span data-ttu-id="6cd7d-126">当一个组件使用执行线程的标识来存储某种类型的状态时，将发生线程关联。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-126">Thread affinity happens when a component uses the identity of the executing thread to store some type of state.</span></span> <span data-ttu-id="6cd7d-127">最常见的形式是使用线程本地存储 (TLS) 来存储状态。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-127">The most common form of this is to use the thread local store (TLS) to store state.</span></span> <span data-ttu-id="6cd7d-128">线程关联要求执行的每个逻辑线程仅由操作系统中的一个物理线程所拥有，这将占用大量内存。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-128">Thread affinity requires that each logical thread of execution be owned by only one physical thread in the operating system, which can become memory intensive.</span></span> <span data-ttu-id="6cd7d-129">最后，WPF 的线程处理模型通过线程关联与单一线程执行的现有 User32 线程处理模型保持同步。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-129">In the end, WPF’s threading model was kept in sync with the existing User32 threading model of single threaded execution with thread affinity.</span></span> <span data-ttu-id="6cd7d-130">这样做的主要原因是互操作性 – OLE 2.0、剪贴板和 Internet 资源管理器等系统都需要单线程关联 （STA） 执行。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-130">The primary reason for this was interoperability – systems like OLE 2.0, the clipboard, and Internet Explorer all require single thread affinity (STA) execution.</span></span>  
  
 <span data-ttu-id="6cd7d-131">假设你具有带有 STA 线程的对象，则需要在线程之间通信并验证你是否位于正确的线程上的一种方法。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-131">Given that you have objects with STA threading, you need a way to communicate between threads, and validate that you are on the correct thread.</span></span> <span data-ttu-id="6cd7d-132">调度程序的作用就在于此。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-132">Herein lies the role of the dispatcher.</span></span> <span data-ttu-id="6cd7d-133">调度程序是一个基本的消息调度系统，具有多个按优先顺序排列的队列。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-133">The dispatcher is a basic message dispatching system, with multiple prioritized queues.</span></span> <span data-ttu-id="6cd7d-134">消息的示例包括原始输入通知（鼠标移动）、框架函数（布局）或用户命令（执行此方法）。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-134">Examples of messages include raw input notifications (mouse moved), framework functions (layout), or user commands (execute this method).</span></span> <span data-ttu-id="6cd7d-135">通过派生从<xref:System.Windows.Threading.DispatcherObject>，您将创建具有 STA 行为的 CLR 对象，并在创建时为调度程序提供指向调度程序的指针。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-135">By deriving from <xref:System.Windows.Threading.DispatcherObject>, you create a CLR object that has STA behavior, and will be given a pointer to a dispatcher at creation time.</span></span>

## <a name="systemwindowsdependencyobject"></a><span data-ttu-id="6cd7d-136">System.Windows.DependencyObject</span><span class="sxs-lookup"><span data-stu-id="6cd7d-136">System.Windows.DependencyObject</span></span>

 <span data-ttu-id="6cd7d-141">为了使更多的系统由属性驱动，需要比 CLR 提供的更丰富的属性系统。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-141">In order to have more of the system driven by properties, a richer property system than what the CLR provides was needed.</span></span> <span data-ttu-id="6cd7d-142">这种丰富性的一个简单示例是更改通知。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-142">A simple example of this richness is change notifications.</span></span> <span data-ttu-id="6cd7d-143">若要实现双向绑定，需要绑定的双方支持更改通知。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-143">In order to enable two way binding, you need both sides of the bind to support change notification.</span></span> <span data-ttu-id="6cd7d-144">若要使行为与属性值相关联，需要在属性值更改时收到通知。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-144">In order to have behavior tied to property values, you need to be notified when the property value changes.</span></span> <span data-ttu-id="6cd7d-145">微软 .NET 框架有一个接口 **，NotifyPropertyChange**，它允许对象发布更改通知，但它是可选的。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-145">The Microsoft .NET Framework has an interface, **INotifyPropertyChange**, which allows an object to publish change notifications, however it is optional.</span></span>  
  
 <span data-ttu-id="6cd7d-146">WPF 提供了一个更丰富的属性系统，派生<xref:System.Windows.DependencyObject>自类型。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-146">WPF provides a richer property system, derived from the <xref:System.Windows.DependencyObject> type.</span></span> <span data-ttu-id="6cd7d-147">该属性系统实际是一个“依赖”属性系统，因为它会跟踪属性表达式之间的依赖关系，并在依赖关系更改时自动重新验证属性值。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-147">The property system is truly a "dependency" property system in that it tracks dependencies between property expressions and automatically revalidates property values when dependencies change.</span></span> <span data-ttu-id="6cd7d-148">例如，如果属性继承 （如<xref:System.Windows.Controls.Control.FontSize%2A>），则如果属性在继承该值的元素的父项上发生更改，则系统将自动更新。</span><span class="sxs-lookup"><span data-stu-id="6cd7d-148">For example, if you have a property that inherits (like <xref:System.Windows.Controls.Control.FontSize%2A>), the system is automatically updated if the property changes on a parent of an element that inherits the value.</span></span>
