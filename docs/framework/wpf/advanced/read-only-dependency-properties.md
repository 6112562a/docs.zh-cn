 <span data-ttu-id="60e91-108">在 [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] 框架中定义的某些依赖属性是只读的。</span><span class="sxs-lookup"><span data-stu-id="60e91-108">Some of the dependency properties defined in the [!INCLUDE[TLA#tla_winclient](../../../../includes/tlasharptla-winclient-md.md)] framework are read-only.</span></span> <span data-ttu-id="60e91-109">指定只读依赖属性的一般原因如下：这些属性应该用于状态确定，但是有多种因素影响该状态，从用户界面设计的角度看，仅将属性设置为该状态并不能达到预期的效果。</span><span class="sxs-lookup"><span data-stu-id="60e91-109">The typical reason for specifying a read-only dependency property is that these are properties that should be used for state determination, but where that state is influenced by a multitude of factors, but just setting the property to that state isn't desirable from a user interface design perspective.</span></span> <span data-ttu-id="60e91-110">例如，属性<xref:System.Windows.UIElement.IsMouseOver%2A>实际上只是从鼠标输入确定的曲面状态。</span><span class="sxs-lookup"><span data-stu-id="60e91-110">For example, the property <xref:System.Windows.UIElement.IsMouseOver%2A> is really just surfacing state as determined from the mouse input.</span></span> <span data-ttu-id="60e91-111">任何通过避开实际的鼠标输入以编程方式设置此值的尝试都是不可预期的，并将导致不一致。</span><span class="sxs-lookup"><span data-stu-id="60e91-111">Any attempt to set this value programmatically by circumventing the true mouse input would be unpredictable and would cause inconsistency.</span></span>  
  
 <span data-ttu-id="60e91-112">由于其不可设置性，只读依赖属性不适用于依赖属性通常为其提供一个解决方案（即：数据绑定，直接样式化为某个值、验证、动画和继承）的多种方案。</span><span class="sxs-lookup"><span data-stu-id="60e91-112">By virtue of not being settable, read-only dependency properties aren't appropriate for many of the scenarios for which dependency properties normally offer a solution (namely: data binding, directly stylable to a value, validation, animation, inheritance).</span></span> <span data-ttu-id="60e91-113">尽管不可设置，但只读依赖属性仍具有一些由属性系统中的依赖属性支持的其他功能。</span><span class="sxs-lookup"><span data-stu-id="60e91-113">Despite not being settable, read-only dependency properties still have some of the additional capabilities supported by dependency properties in the property system.</span></span> <span data-ttu-id="60e91-114">只读依赖属性仍可以用作样式中的属性触发器，这是其他功能中最重要的功能。</span><span class="sxs-lookup"><span data-stu-id="60e91-114">The most important remaining capability is that the read-only dependency property can still be used as a property trigger in a style.</span></span> <span data-ttu-id="60e91-115">不能启用具有普通通用语言运行时 （CLR） 属性的触发器;因此，无法启用触发器。它需要是一个依赖项属性。</span><span class="sxs-lookup"><span data-stu-id="60e91-115">You can't enable triggers with a normal common language runtime (CLR) property;
