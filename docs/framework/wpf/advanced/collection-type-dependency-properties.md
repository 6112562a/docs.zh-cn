## <a name="implementing-a-collection-type-dependency-property"></a><span data-ttu-id="bfa39-104">实现集合类型依赖属性</span><span class="sxs-lookup"><span data-stu-id="bfa39-104">Implementing a Collection-Type Dependency Property</span></span>  
 <span data-ttu-id="bfa39-105">通常, 对于依赖属性, 你遵循的实现模式是定义 CLR 属性包装器, 其中该属性是由<xref:System.Windows.DependencyProperty>标识符而不是字段或其他构造提供支持的。</span><span class="sxs-lookup"><span data-stu-id="bfa39-105">For a dependency property in general, the implementation pattern that you follow is that you define a CLR property wrapper, where that property is backed by a <xref:System.Windows.DependencyProperty> identifier rather than a field or other construct.</span></span> <span data-ttu-id="bfa39-106">实现集合类型属性时也应按照此相同模式。</span><span class="sxs-lookup"><span data-stu-id="bfa39-106">You follow this same pattern when you implement a collection-type property.</span></span> <span data-ttu-id="bfa39-107">但是, 只要集合中包含的类型本身<xref:System.Windows.DependencyObject>就是或<xref:System.Windows.Freezable>派生类, 集合类型属性就会给模式带来一些复杂性。</span><span class="sxs-lookup"><span data-stu-id="bfa39-107">However, a collection-type property introduces some complexity to the pattern whenever the type that is contained within the collection is itself a <xref:System.Windows.DependencyObject> or <xref:System.Windows.Freezable> derived class.</span></span>

## <a name="initializing-the-collection-beyond-the-default-value"></a><span data-ttu-id="bfa39-108">不使用默认值初始化集合</span><span class="sxs-lookup"><span data-stu-id="bfa39-108">Initializing the Collection Beyond the Default Value</span></span>  
 <span data-ttu-id="bfa39-109">创建依赖属性时，不要将属性默认值指定为初始字段值。</span><span class="sxs-lookup"><span data-stu-id="bfa39-109">When you create a dependency property, you do not specify the property default value as the initial field value.</span></span> <span data-ttu-id="bfa39-110">相反，应通过依赖属性元数据指定默认值。</span><span class="sxs-lookup"><span data-stu-id="bfa39-110">Instead, you specify the default value through the dependency property metadata.</span></span> <span data-ttu-id="bfa39-111">如果属性为引用类型，则依赖属性元数据中指定的默认值不是每个实例分别的默认值，而是应用到类型的所有实例的默认值。</span><span class="sxs-lookup"><span data-stu-id="bfa39-111">If your property is a reference type, the default value specified in dependency property metadata is not a default value per instance; instead it is a default value that applies to all instances of the type.</span></span> <span data-ttu-id="bfa39-112">因此，对于类型的新创建实例，必须小心，不要将集合属性元数据定义的单个静态集合用作工作默认值。</span><span class="sxs-lookup"><span data-stu-id="bfa39-112">Therefore you must be careful to not use the singular static collection defined by the collection property metadata as the working default value for newly created instances of your type.</span></span> <span data-ttu-id="bfa39-113">相反，必须确保有意将集合值设置为唯一（实例）集合，作为类构造函数逻辑的一部分。</span><span class="sxs-lookup"><span data-stu-id="bfa39-113">Instead, you must make sure that you deliberately set the collection value to a unique (instance) collection as part of your class constructor logic.</span></span> <span data-ttu-id="bfa39-114">否则，你会创建一个不需要的单例类。</span><span class="sxs-lookup"><span data-stu-id="bfa39-114">Otherwise you will have created an unintentional singleton class.</span></span>

## <a name="reporting-binding-value-changes-from-collection-properties"></a><span data-ttu-id="bfa39-129">报告集合属性中的绑定值更改</span><span class="sxs-lookup"><span data-stu-id="bfa39-129">Reporting Binding Value Changes from Collection Properties</span></span>  
 <span data-ttu-id="bfa39-130">本身为依赖属性的集合属性不会自动将更改报告给其子属性。</span><span class="sxs-lookup"><span data-stu-id="bfa39-130">A collection property that is itself a dependency property does not automatically report changes to its subproperties.</span></span> <span data-ttu-id="bfa39-131">如果将绑定创建到集合，这可阻止绑定报告更改，从而使某些数据绑定方案无效。</span><span class="sxs-lookup"><span data-stu-id="bfa39-131">If you are creating bindings into a collection, this can prevent the binding from reporting changes, thus invalidating some data binding scenarios.</span></span> <span data-ttu-id="bfa39-132">但是, 如果使用集合类型<xref:System.Windows.FreezableCollection%601>作为集合类型, 则会正确报告集合中包含的元素的子属性更改, 并且绑定将按预期方式工作。</span><span class="sxs-lookup"><span data-stu-id="bfa39-132">However, if you use the collection type <xref:System.Windows.FreezableCollection%601> as your collection type, then subproperty changes to contained elements in the collection are properly reported, and binding works as expected.</span></span>  
  
 <span data-ttu-id="bfa39-133">若要启用依赖项对象集合中的子属性绑定, 请创建类型<xref:System.Windows.FreezableCollection%601>为的集合属性, 并将该集合的<xref:System.Windows.DependencyObject>类型约束为任何派生类。</span><span class="sxs-lookup"><span data-stu-id="bfa39-133">To enable subproperty binding in a dependency object collection, create the collection property as type <xref:System.Windows.FreezableCollection%601>, with a type constraint for that collection to any <xref:System.Windows.DependencyObject> derived class.</span></span>
