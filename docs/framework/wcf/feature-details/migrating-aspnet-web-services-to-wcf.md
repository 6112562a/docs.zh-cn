<span data-ttu-id="20cfa-103">ASP.NET 提供 .NET Framework 类库和工具以用于生成 Web 服务，并提供用于在 Internet Information Services (IIS) 内承载服务的功能。</span><span class="sxs-lookup"><span data-stu-id="20cfa-103">ASP.NET provides .NET Framework class libraries and tools for building Web services, as well as facilities for hosting services within Internet Information Services (IIS).</span></span> <span data-ttu-id="20cfa-104">Windows Communication Foundation (WCF) 提供了.NET Framework 类库、 工具和承载功能，使软件实体，可以使用任何协议，包括 Web 服务所使用的协议进行通信。</span><span class="sxs-lookup"><span data-stu-id="20cfa-104">Windows Communication Foundation (WCF) provides .NET Framework class libraries, tools and hosting facilities for enabling software entities to communicate using any protocols, including those used by Web services.</span></span>  <span data-ttu-id="20cfa-105">ASP.NET Web 服务迁移到 WCF 允许应用程序以充分利用新功能和改进功能，是唯一的 WCF。</span><span class="sxs-lookup"><span data-stu-id="20cfa-105">Migrating ASP.NET Web Services to WCF allows your applications to take advantage of new features and improvements that are unique to WCF.</span></span>  
  
 <span data-ttu-id="20cfa-106">WCF 具有 ASP.NET Web 服务的多个重要优势。</span><span class="sxs-lookup"><span data-stu-id="20cfa-106">WCF has several important advantages relative to ASP.NET Web services.</span></span> <span data-ttu-id="20cfa-107">尽管 ASP.NET Web 服务工具仅用于生成 Web 服务，WCF 提供了软件实体必须进行相互通信时，可以使用的工具。</span><span class="sxs-lookup"><span data-stu-id="20cfa-107">While ASP.NET Web services tools are solely for building Web services, WCF provides tools that can be used when software entities must be made to communicate with one another.</span></span> <span data-ttu-id="20cfa-108">这会减少开发人员为适应不同软件通信方案而必需了解的技术的数量，从而降低软件开发资源的成本及完成软件开发项目所需的时间。</span><span class="sxs-lookup"><span data-stu-id="20cfa-108">This will reduce the number of technologies that developers are required to know in order to accommodate different software communication scenarios, which in turn will reduce the cost of software development resources, as well as the time to complete software development projects.</span></span>  
  
 <span data-ttu-id="20cfa-109">甚至对于 Web 服务开发项目，WCF 支持多个 Web 服务协议比 ASP.NET Web 服务支持。</span><span class="sxs-lookup"><span data-stu-id="20cfa-109">Even for Web service development projects, WCF supports more Web service protocols than ASP.NET Web services support.</span></span> <span data-ttu-id="20cfa-110">这些附加协议提供更完善的解决方案，涉及可靠会话和事务等其他内容。</span><span class="sxs-lookup"><span data-stu-id="20cfa-110">These additional protocols provide for more sophisticated solutions involving, amongst other things, reliable sessions and transactions.</span></span>  
  
 <span data-ttu-id="20cfa-111">WCF 支持多个协议比 ASP.NET Web 服务的消息传输。</span><span class="sxs-lookup"><span data-stu-id="20cfa-111">WCF supports more protocols for transporting messages than ASP.NET Web services.</span></span> <span data-ttu-id="20cfa-112">ASP.NET Web 服务仅支持使用超文本传输协议 (HTTP) 发送消息。</span><span class="sxs-lookup"><span data-stu-id="20cfa-112">ASP.NET Web services only support sending messages by using the Hypertext Transfer Protocol (HTTP).</span></span> <span data-ttu-id="20cfa-113">WCF 支持通过使用 HTTP，传输控制协议 (TCP)，命名管道和 Microsoft 消息队列 (MSMQ) 发送的消息。</span><span class="sxs-lookup"><span data-stu-id="20cfa-113">WCF supports sending messages by using HTTP, as well as the Transmission Control Protocol (TCP), named pipes, and Microsoft Message Queuing (MSMQ).</span></span> <span data-ttu-id="20cfa-114">更重要的是，WCF 可以进行扩展以支持其他传输协议。</span><span class="sxs-lookup"><span data-stu-id="20cfa-114">More important, WCF can be extended to support additional transport protocols.</span></span> <span data-ttu-id="20cfa-115">因此，使用 WCF 开发的软件可以进行适配以便与更广泛的其他软件，从而提高了潜在的投资回报一起工作。</span><span class="sxs-lookup"><span data-stu-id="20cfa-115">Therefore, software developed using WCF can be adapted to work together with a wider variety of other software, thereby increasing the potential return on the investment.</span></span>  
  
 <span data-ttu-id="20cfa-116">WCF 提供用于部署多更丰富的工具和管理应用程序比 ASP.NET Web 服务提供。</span><span class="sxs-lookup"><span data-stu-id="20cfa-116">WCF provides much richer facilities for deploying and managing applications than ASP.NET Web services provides.</span></span> <span data-ttu-id="20cfa-117">除配置系统，ASP.NET 也具有，WCF 还提供配置编辑器中，来自发件人到接收方并返回通过任意数目的中间方、 跟踪查看器、 消息日志记录、 大量性能计数器的活动跟踪和对 Windows Management Instrumentation 的支持。</span><span class="sxs-lookup"><span data-stu-id="20cfa-117">In addition to a configuration system, which ASP.NET also has, WCF offers a configuration editor, activity tracing from senders to receivers and back through any number of intermediaries, a trace viewer, message logging, a vast number of performance counters, and support for Windows Management Instrumentation.</span></span>  
  
 <span data-ttu-id="20cfa-118">给定这些潜在的好处的相对于 ASP.NET Web 的 WCF 服务，如果使用的，或考虑使用 ASP.NET Web 服务有多个选项：</span><span class="sxs-lookup"><span data-stu-id="20cfa-118">Given these potential benefits of WCF relative to ASP.NET Web services, if you are using, or are considering using ASP.NET Web services you have several options:</span></span>  
  
- <span data-ttu-id="20cfa-119">继续使用 ASP.NET Web 服务，并放弃由 WCF 提供的好处。</span><span class="sxs-lookup"><span data-stu-id="20cfa-119">Continue to use ASP.NET Web services, and forego the benefits proffered by WCF.</span></span>  
  
- <span data-ttu-id="20cfa-120">继续使用 ASP.NET Web 服务，并打算在将来的某个时间采用 WCF。</span><span class="sxs-lookup"><span data-stu-id="20cfa-120">Keep using ASP.NET Web services with the intention of adopting WCF at some time in the future.</span></span> <span data-ttu-id="20cfa-121">在本部分中的主题介绍如何充分利用能够使用新 ASP.NET Web 服务的应用程序以及未来的 WCF 应用程序的前景。</span><span class="sxs-lookup"><span data-stu-id="20cfa-121">The topics in this section explain how to maximize the prospects for being able to use new ASP.NET Web service applications together with future WCF applications.</span></span> <span data-ttu-id="20cfa-122">在本部分中的主题还说明如何生成新的 ASP.NET Web 服务，以使其更轻松地将它们迁移到 WCF。</span><span class="sxs-lookup"><span data-stu-id="20cfa-122">The topics in this section also explain how to build new ASP.NET Web services so as to make it easier to migrate them to WCF.</span></span> <span data-ttu-id="20cfa-123">但是，如果服务的安全很重要，或可靠性或事务保证是必需的或者如果自定义管理工具将需要构造，则它是更好的选择采用 WCF。</span><span class="sxs-lookup"><span data-stu-id="20cfa-123">However, if securing the services is important, or reliability or transaction assurances are required, or if custom management facilities will have to be constructed, then it is a better option to adopt WCF.</span></span> <span data-ttu-id="20cfa-124">WCF 专为这样的方案。</span><span class="sxs-lookup"><span data-stu-id="20cfa-124">WCF is designed for precisely such scenarios.</span></span>  
  
- <span data-ttu-id="20cfa-125">对于新开发，同时继续保持现有的 ASP.NET Web 服务应用程序采用 WCF。</span><span class="sxs-lookup"><span data-stu-id="20cfa-125">Adopt WCF for new development, while continuing to maintain your existing ASP.NET Web service applications.</span></span> <span data-ttu-id="20cfa-126">这种选择很可能是最佳选择。</span><span class="sxs-lookup"><span data-stu-id="20cfa-126">This choice is very likely the optimal one.</span></span> <span data-ttu-id="20cfa-127">它的好处 WCF，同时可节省修改现有的应用程序以使用它的成本。</span><span class="sxs-lookup"><span data-stu-id="20cfa-127">It yields the benefits of WCF, while sparing the cost of modifying the existing applications to use it.</span></span> <span data-ttu-id="20cfa-128">在此方案中，在现有 ASP.NET 应用程序中新的 WCF 应用程序可以共存。</span><span class="sxs-lookup"><span data-stu-id="20cfa-128">In this scenario, new WCF applications can co-exist with existing ASP.NET applications.</span></span> <span data-ttu-id="20cfa-129">新的 WCF 应用程序将能够使用现有的 ASP.NET Web 服务，并可以使用 WCF 进行编程到现有 ASP.NET 应用程序通过 WCF ASP.NET 兼容模式的新操作功能。</span><span class="sxs-lookup"><span data-stu-id="20cfa-129">New WCF applications will be able to use existing ASP.NET Web services, and WCF can be used to program new operational capabilities into existing ASP.NET applications by virtue of WCF ASP.NET compatibility mode.</span></span>  
  
- <span data-ttu-id="20cfa-130">采用 WCF 和现有 ASP.NET Web 服务应用程序迁移到 WCF。</span><span class="sxs-lookup"><span data-stu-id="20cfa-130">Adopt WCF and migrate existing ASP.NET Web service applications to WCF.</span></span> <span data-ttu-id="20cfa-131">可以选择此选项可以使用 WCF，提供的功能来增强现有应用程序，或者重新生成现有的 ASP.NET Web 服务在新的功能、 功能更强大的 WCF 应用程序。</span><span class="sxs-lookup"><span data-stu-id="20cfa-131">You may choose this option to enhance the existing applications with features provided by WCF, or to reproduce the functionality of existing ASP.NET Web services within new, more powerful WCF applications.</span></span>
