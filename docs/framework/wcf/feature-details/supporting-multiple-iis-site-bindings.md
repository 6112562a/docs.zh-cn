## <a name="multiple-base-addresses"></a><span data-ttu-id="a03ac-109">多个基址</span><span class="sxs-lookup"><span data-stu-id="a03ac-109">Multiple Base Addresses</span></span>  
 <span data-ttu-id="a03ac-110">此功能才可用于在 IIS 下承载的 WCF 服务。</span><span class="sxs-lookup"><span data-stu-id="a03ac-110">This feature is only available to WCF services that are hosted under IIS.</span></span> <span data-ttu-id="a03ac-111">默认情况下不启用此功能。</span><span class="sxs-lookup"><span data-stu-id="a03ac-111">This feature is not enabled by default.</span></span> <span data-ttu-id="a03ac-112">若要启用它必须添加`multipleSiteBindingsEnabled`属性为 <`serviceHostingEnvironment`> 元素在 Web.config 中的文件，并将其设置为`true`，如以下示例所示。</span><span class="sxs-lookup"><span data-stu-id="a03ac-112">To enable it you must add the `multipleSiteBindingsEnabled` attribute to the <`serviceHostingEnvironment`> element in your Web.config file and set it to `true`, as shown in the following example.</span></span>  
  
```xml  
<serviceHostingEnvironment multipleSiteBindingsEnabled="true"/>  
```  
  
 <span data-ttu-id="a03ac-113">当承载在 IIS 下的 WCF 服务时，IIS 可基于虚拟目录包含的应用程序的 URI 创建一个基址。</span><span class="sxs-lookup"><span data-stu-id="a03ac-113">When hosting a WCF service under IIS, IIS creates one base address for you based on the URI to the virtual directory that contains the application.</span></span> <span data-ttu-id="a03ac-114">您可以通过 Internet 信息服务管理器添加使用相同协议的其他基址，从而向网站添加一个或多个绑定。</span><span class="sxs-lookup"><span data-stu-id="a03ac-114">You can add additional base addresses that use the same protocol by using Internet Information Services Manager to add one or more bindings to your Web site.</span></span> <span data-ttu-id="a03ac-115">对于每个绑定，请指定协议（HTTP 或 HTTPS）、IP 地址、端口和主机名。</span><span class="sxs-lookup"><span data-stu-id="a03ac-115">For each binding specify a protocol (HTTP or HTTPS), an IP address, a port, and a host name.</span></span> <span data-ttu-id="a03ac-116">有关使用 Internet 信息服务管理器的详细信息，请参阅[IIS 管理器 (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164057)。</span><span class="sxs-lookup"><span data-stu-id="a03ac-116">For more information about using Internet Information Services Manager, see [IIS Manager (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164057).</span></span> <span data-ttu-id="a03ac-117">有关向站点添加绑定的详细信息，请参阅[创建网站 (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164060)</span><span class="sxs-lookup"><span data-stu-id="a03ac-117">For more information about adding bindings to a site, see [Create a Web Site (IIS 7)](https://go.microsoft.com/fwlink/?LinkId=164060)</span></span>  
  
 <span data-ttu-id="a03ac-118">指定的同一站点的多个基址会影响 WCF 帮助页，导入架构，并由服务生成的 WSDL/MEX 信息的内容。</span><span class="sxs-lookup"><span data-stu-id="a03ac-118">Specifying multiple base addresses for the same site affects the content of the WCF Help page, importing schema, and the WSDL/MEX information generated by the service.</span></span> <span data-ttu-id="a03ac-119">WCF 帮助页显示要用于生成 WCF 客户端可以与服务进行通信的命令行。</span><span class="sxs-lookup"><span data-stu-id="a03ac-119">The WCF Help page displays the command line to use to generate a WCF client that can communicate with the service.</span></span> <span data-ttu-id="a03ac-120">此命令行只包含在 IIS 绑定中为网站指定的第一个地址。</span><span class="sxs-lookup"><span data-stu-id="a03ac-120">This command line contains only the first address specified in the IIS binding for the Web site.</span></span> <span data-ttu-id="a03ac-121">与导入方案时相似，只使用在 IIS 绑定中指定的第一个基址。</span><span class="sxs-lookup"><span data-stu-id="a03ac-121">Similarly when importing schema, only the first base address specified in the IIS binding is used.</span></span> <span data-ttu-id="a03ac-122">WSDL 和 MEX 数据包含在 IIS 绑定中指定的所有基址。</span><span class="sxs-lookup"><span data-stu-id="a03ac-122">WSDL and MEX data contain all the base addresses specified in the IIS bindings.</span></span>
