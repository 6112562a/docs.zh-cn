#### <a name="specifying-an-endpoint-to-use-a-binding-type"></a><span data-ttu-id="1afcc-125">指定终结点以使用绑定类型</span><span class="sxs-lookup"><span data-stu-id="1afcc-125">Specifying an Endpoint to Use a Binding Type</span></span>

```xml  
  <!-- This section is optional with the default configuration introduced  
       in .NET Framework 4. -->

      address="/HelloWorld2/"
```  

 <span data-ttu-id="1afcc-127">在本示例中，`name` 属性表示使用该配置的服务类型。</span><span class="sxs-lookup"><span data-stu-id="1afcc-127">In this example, the `name` attribute indicates which service type the configuration is for.</span></span> <span data-ttu-id="1afcc-128">当您用 `HelloWorld` 协定通过代码中创建服务时，它将以示例配置中定义的所有终结点进行初始化。</span><span class="sxs-lookup"><span data-stu-id="1afcc-128">When you create a service in your code with the `HelloWorld` contract, it is initialized with all of the endpoints defined in the example configuration.</span></span> <span data-ttu-id="1afcc-129">如果此程序集实现只有一个服务协定，`name`属性可以省略，因为服务使用唯一可用的类型。</span><span class="sxs-lookup"><span data-stu-id="1afcc-129">If the assembly implements only one service contract, the `name` attribute can be omitted because the service uses the only available type.</span></span> <span data-ttu-id="1afcc-130">该属性采用字符串，该字符串必须遵循以下格式：`Namespace.Class, AssemblyName, Version=0.0.0.0, Culture=neutral, PublicKeyToken=null`</span><span class="sxs-lookup"><span data-stu-id="1afcc-130">The attribute takes a string, which must be in the format `Namespace.Class, AssemblyName, Version=0.0.0.0, Culture=neutral, PublicKeyToken=null`</span></span>  
  
 <span data-ttu-id="1afcc-131">`address` 属性指定其他终结点用于与该服务通信的 URI。</span><span class="sxs-lookup"><span data-stu-id="1afcc-131">The `address` attribute specifies the URI that other endpoints use to communicate to the service.</span></span> <span data-ttu-id="1afcc-132">该 URI 可以是绝对路径，也可以是相对路径。</span><span class="sxs-lookup"><span data-stu-id="1afcc-132">The URI can be either an absolute or relative path.</span></span> <span data-ttu-id="1afcc-133">如果提供的是相对地址，则需要主机提供适合于绑定中所使用的传输方案的基址。</span><span class="sxs-lookup"><span data-stu-id="1afcc-133">If a relative address is provided, the host is expected to provide a base address that is appropriate for the transport scheme used in the binding.</span></span> <span data-ttu-id="1afcc-134">如果未配置地址，则假定基址为该终结点的地址。</span><span class="sxs-lookup"><span data-stu-id="1afcc-134">If an address is not configured, the base address is assumed to be the address for that endpoint.</span></span>

#### <a name="modifying-a-predefined-binding"></a><span data-ttu-id="1afcc-140">修改预定义绑定</span><span class="sxs-lookup"><span data-stu-id="1afcc-140">Modifying a Predefined Binding</span></span>  
 <span data-ttu-id="1afcc-141">在下面的示例中，修改了预定义绑定。</span><span class="sxs-lookup"><span data-stu-id="1afcc-141">In the following example, a predefined binding is modified.</span></span> <span data-ttu-id="1afcc-142">然后可以将该绑定用于配置服务中的任意终结点。</span><span class="sxs-lookup"><span data-stu-id="1afcc-142">It can then be used to configure any endpoint in the service.</span></span> <span data-ttu-id="1afcc-143">可通过将 <xref:System.ServiceModel.Configuration.IBindingConfigurationElement.ReceiveTimeout%2A> 值设置为 1 秒的方式来修改绑定。</span><span class="sxs-lookup"><span data-stu-id="1afcc-143">The binding is modified by setting the <xref:System.ServiceModel.Configuration.IBindingConfigurationElement.ReceiveTimeout%2A> value to 1 second.</span></span> <span data-ttu-id="1afcc-144">请注意，该属性返回一个 <xref:System.TimeSpan> 对象。</span><span class="sxs-lookup"><span data-stu-id="1afcc-144">Note that the property returns a <xref:System.TimeSpan> object.</span></span>  
  
 <span data-ttu-id="1afcc-145">在绑定部分中可以找到该更改的绑定。</span><span class="sxs-lookup"><span data-stu-id="1afcc-145">That altered binding is found in the bindings section.</span></span> <span data-ttu-id="1afcc-146">现在，通过在 `binding` 元素中设置 `endpoint` 特性，可以在创建任何终结点时使用该更改的绑定。</span><span class="sxs-lookup"><span data-stu-id="1afcc-146">This altered binding can now be used when creating any endpoint by setting the `binding` attribute in the `endpoint` element.</span></span>

## <a name="configuring-a-behavior-to-apply-to-a-service"></a><span data-ttu-id="1afcc-148">配置要应用于服务的行为</span><span class="sxs-lookup"><span data-stu-id="1afcc-148">Configuring a Behavior to Apply to a Service</span></span>  
 <span data-ttu-id="1afcc-149">在下面的示例中，为该服务类型配置了一个特定的行为。</span><span class="sxs-lookup"><span data-stu-id="1afcc-149">In the following example, a specific behavior is configured for the service type.</span></span> <span data-ttu-id="1afcc-150">`ServiceMetadataBehavior`元素用于使[ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md)查询服务并从元数据生成 Web 服务描述语言 (WSDL) 文档。</span><span class="sxs-lookup"><span data-stu-id="1afcc-150">The `ServiceMetadataBehavior` element is used to enable the [ServiceModel Metadata Utility Tool (Svcutil.exe)](../../../docs/framework/wcf/servicemodel-metadata-utility-tool-svcutil-exe.md) to query the service and generate Web Services Description Language (WSDL) documents from the metadata.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="1afcc-151">如果为该行为指定了特定名称，则服务或终结点部分中指定的 `behaviorConfiguration` 必须与其相匹配。</span><span class="sxs-lookup"><span data-stu-id="1afcc-151">If you give a particular name to the behavior, the `behaviorConfiguration` specified in the service or endpoint section must match it.</span></span>

```xml  
        <ServiceMetadata httpGetEnabled="true" />   
    </behavior>  
</behaviors>  
<services>  
    <service   
       name="HelloWorld, IndigoConfig, Version=2.0.0.0, Culture=neutral, PublicKeyToken=null">
       <endpoint   
          address="http://computer:8080/Hello"  
          contract="HelloWorld, IndigoConfig, Version=2.0.0.0, Culture=neutral, PublicKeyToken=null"  
          binding="basicHttpBinding" />
    </service>  
</services>  
```  
  
 <span data-ttu-id="1afcc-152">上面的配置使客户端可以调用并获取“HelloWorld”类型的服务的元数据。</span><span class="sxs-lookup"><span data-stu-id="1afcc-152">The preceding configuration enables a client to call and get the metadata of the "HelloWorld" typed service.</span></span>  
  
 `svcutil /config:Client.exe.config http://computer:8080/Hello?wsdl`  
  
## <a name="specifying-a-service-with-two-endpoints-using-different-binding-values"></a><span data-ttu-id="1afcc-153">使用不同的绑定值指定具有两个终结点的服务</span><span class="sxs-lookup"><span data-stu-id="1afcc-153">Specifying a Service with Two Endpoints Using Different Binding Values</span></span>  
 <span data-ttu-id="1afcc-154">这是最后一个示例，在本示例中，为 `HelloWorld` 服务类型配置了两个终结点。</span><span class="sxs-lookup"><span data-stu-id="1afcc-154">In this last example, two endpoints are configured for the `HelloWorld` service type.</span></span> <span data-ttu-id="1afcc-155">每个终结点使用不同的自定义`bindingConfiguration`相同的绑定类型的属性 (每个修改`basicHttpBinding`)。</span><span class="sxs-lookup"><span data-stu-id="1afcc-155">Each endpoint uses a different customized  `bindingConfiguration` attribute of the same binding type (each modifies the `basicHttpBinding`).</span></span>  
  
```xml  
<service name="HelloWorld, IndigoConfig, Version=2.0.0.0, Culture=neutral, PublicKeyToken=null">  
    <endpoint  
        address="http://computer:8080/Hello1"  
        contract="HelloWorld, IndigoConfig, Version=2.0.0.0, Culture=neutral, PublicKeyToken=null"  
        binding="basicHttpBinding"  
        bindingConfiguration="shortTimeout" />
    <endpoint  
        address="http://computer:8080/Hello2"  
        contract="HelloWorld, IndigoConfig, Version=2.0.0.0, Culture=neutral, PublicKeyToken=null"  
        binding="basicHttpBinding"  
        bindingConfiguration="Secure" />
</service>  
<bindings>  
    <basicHttpBinding   
        name="shortTimeout"  
        timeout="00:00:00:01"   
     />  
     <basicHttpBinding   
        name="Secure">  
        <Security mode="Transport" />  
     </basicHttpBinding>
</bindings>  
```  
  
 <span data-ttu-id="1afcc-156">通过添加 `protocolMapping` 部分并按下面的示例所示配置绑定，可以使用默认配置获得相同的行为。</span><span class="sxs-lookup"><span data-stu-id="1afcc-156">You can get the same behavior using the default configuration by adding a `protocolMapping` section and configuring the bindings as demonstrated in the following example.</span></span>  
  
```xml  
<protocolMapping>  
    <add scheme="http" binding="basicHttpBinding" bindingConfiguration="shortTimeout" />  
    <add scheme="https" binding="basicHttpBinding" bindingConfiguration="Secure" />  
</protocolMapping>  
<bindings>  
    <basicHttpBinding   
        name="shortTimeout"  
        timeout="00:00:00:01"   
     />  
     <basicHttpBinding   
        name="Secure" />  
        <Security mode="Transport" />  
</bindings>  
```
