 <span data-ttu-id="2e638-108">如果未`ActivityId`标头消息中存在时 (即`propagateActivity` = `false`客户端上)，或当`propagateActivity` = `false`的服务器上，服务器会生成一个新的活动 id。</span><span class="sxs-lookup"><span data-stu-id="2e638-108">When no `ActivityId` header is present in the message (that is, `propagateActivity`=`false` on the client), or when `propagateActivity`=`false` on the server, the server generates a new activity ID.</span></span>

## <a name="servicemodel-activity-tracing-is-disabled"></a><span data-ttu-id="2e638-121">ServiceModel 活动跟踪禁用</span><span class="sxs-lookup"><span data-stu-id="2e638-121">ServiceModel Activity Tracing is Disabled</span></span>

 <span data-ttu-id="2e638-124">如果`propagateActivity` = `true`并`ActivityTracing` = `off` ServiceModel 跟踪侦听器 （无论 ServiceModel 上是否启用跟踪），在客户端或服务器上将发生以下：</span><span class="sxs-lookup"><span data-stu-id="2e638-124">If `propagateActivity`=`true` and `ActivityTracing`=`off` for a ServiceModel trace listener (regardless of whether tracing is enabled on ServiceModel), the following happen on either the client or server:</span></span>  
  
- <span data-ttu-id="2e638-125">在操作请求或发送响应时，TLS 中的活动 ID 从用户代码传播出去，直至构成消息。</span><span class="sxs-lookup"><span data-stu-id="2e638-125">On operation request or sending response, the activity ID in TLS is propagated out of user code until a message is formed.</span></span> <span data-ttu-id="2e638-126">活动 ID 标头也在消息发送之前插入消息中。</span><span class="sxs-lookup"><span data-stu-id="2e638-126">An activity ID header is also inserted into the message before it is sent.</span></span>
