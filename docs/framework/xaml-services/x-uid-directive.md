## <a name="remarks"></a><span data-ttu-id="10f90-108">备注</span><span class="sxs-lookup"><span data-stu-id="10f90-108">Remarks</span></span>  
 <span data-ttu-id="10f90-109">在 [MS-CHAP] 中，`x:Uid` 定义为指令。</span><span class="sxs-lookup"><span data-stu-id="10f90-109">In [MS-XAML], `x:Uid` is defined as a directive.</span></span> <span data-ttu-id="10f90-110">有关详细信息，请参阅[\[MS-XAML\] 部分 5.3.6](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10))。</span><span class="sxs-lookup"><span data-stu-id="10f90-110">For more information, see [\[MS-XAML\] Section 5.3.6](https://docs.microsoft.com/previous-versions/msp-n-p/ff650760(v=pandp.10)).</span></span>  
  
 <span data-ttu-id="10f90-111">由于所述的 XAML 本地化方案，`x:Uid` 与 `x:Name` 是离散的，因此用于本地化的标识符不依赖于 `x:Name`的编程模型含义。</span><span class="sxs-lookup"><span data-stu-id="10f90-111">`x:Uid` is discrete from `x:Name` both because of the stated XAML localization scenario and so that identifiers that are used for localization have no dependencies on the programming model implications of `x:Name`.</span></span> <span data-ttu-id="10f90-112">此外，`x:Name` 由 XAML 名称范围控制;但 `x:Uid` 不受任何 XAML 语言定义的唯一性强制概念的约束。</span><span class="sxs-lookup"><span data-stu-id="10f90-112">Also, `x:Name` is governed by the XAML namescope; however, `x:Uid` is not governed by any XAML language defined concept of uniqueness enforcement.</span></span> <span data-ttu-id="10f90-113">一般意义上的 XAML 处理器（不是本地化过程的一部分的处理器）不应强制实施 `x:Uid` 值的唯一性。</span><span class="sxs-lookup"><span data-stu-id="10f90-113">XAML processors in a broad sense (processors that are not part of the localization process) are not expected to enforce uniqueness of `x:Uid` values.</span></span> <span data-ttu-id="10f90-114">从概念上讲，这种责任在值的发起方。</span><span class="sxs-lookup"><span data-stu-id="10f90-114">That responsibility is conceptually on the originator of the values.</span></span> <span data-ttu-id="10f90-115">单个 XAML 源中 `x:Uid` 值的唯一性假定对于值的使用者（如专用的全球化过程或工具）是合理的。</span><span class="sxs-lookup"><span data-stu-id="10f90-115">The expectation of uniqueness of `x:Uid` values within a single XAML source is reasonable for consumers of the values, such as dedicated globalization processes or tools.</span></span> <span data-ttu-id="10f90-116">典型的唯一性模型是，`x:Uid` 值在表示 XAML 的 XML 编码文件中是唯一的。</span><span class="sxs-lookup"><span data-stu-id="10f90-116">The typical uniqueness model is that `x:Uid` values are unique within an XML-encoded file that represents XAML.</span></span>  
  
 <span data-ttu-id="10f90-117">对于特定 XAML 架构具有重要了解的工具，可以选择仅将 `x:Uid` 应用于真正可本地化的字符串，而不是用于标记中出现文本字符串值的所有情况。</span><span class="sxs-lookup"><span data-stu-id="10f90-117">Tools that have significant knowledge of a particular XAML schema can choose to apply `x:Uid` only for true localizable strings, instead of for all cases where a text string value is encountered in markup.</span></span>  
  
 <span data-ttu-id="10f90-118">框架可以通过将特性 <xref:System.Windows.Markup.UidPropertyAttribute> 应用于定义类型，将其对象模型中的特定属性指定为 `x:Uid` 的别名。</span><span class="sxs-lookup"><span data-stu-id="10f90-118">Frameworks can specify a particular property in their object model to be an alias for `x:Uid` by applying the attribute <xref:System.Windows.Markup.UidPropertyAttribute> to the defining type.</span></span> <span data-ttu-id="10f90-119">如果框架指定了特定属性，则在同一对象上同时指定 `x:Uid` 和化名成员是无效的。</span><span class="sxs-lookup"><span data-stu-id="10f90-119">If a framework specifies a particular property, it is not valid to specify both `x:Uid` and the aliased member on the same object.</span></span> <span data-ttu-id="10f90-120">如果同时指定 `x:Uid` 和化名成员，则 .NET Framework XAML 服务 API 通常会在这种情况下引发 <xref:System.Xaml.XamlDuplicateMemberException>。</span><span class="sxs-lookup"><span data-stu-id="10f90-120">If both `x:Uid` and the aliased member are specified, the .NET Framework XAML Services API typically throws <xref:System.Xaml.XamlDuplicateMemberException> for this case.</span></span>
