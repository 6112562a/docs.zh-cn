# <a name="compiled-queries--linq-to-entities"></a><span data-ttu-id="e50d0-102">已编译的查询 (LINQ to Entities)</span><span class="sxs-lookup"><span data-stu-id="e50d0-102">Compiled Queries  (LINQ to Entities)</span></span>
<span data-ttu-id="e50d0-103">如果应用程序需要在实体框架中多次执行结构类似的查询，通常可以通过仅编译查询一次并在每次执行时使用不同参数的方法来提高性能。</span><span class="sxs-lookup"><span data-stu-id="e50d0-103">When you have an application that executes structurally similar queries many times in the Entity Framework, you can frequently increase performance by compiling the query one time and executing it several times with different parameters.</span></span> <span data-ttu-id="e50d0-104">例如，某应用程序要检索特定城市的所有客户，而该城市是运行时由用户在窗体中指定的。</span><span class="sxs-lookup"><span data-stu-id="e50d0-104">For example, an application might have to retrieve all the customers in a particular city; the city is specified at runtime by the user in a form.</span></span> <span data-ttu-id="e50d0-105">LINQ to Entities 支持将已编译的查询用于此目的。</span><span class="sxs-lookup"><span data-stu-id="e50d0-105">LINQ to Entities supports using compiled queries for this purpose.</span></span>  
  
 <span data-ttu-id="e50d0-106">从 .NET Framework 4.5 开始，LINQ 查询是自动缓存的。</span><span class="sxs-lookup"><span data-stu-id="e50d0-106">Starting with the .NET Framework 4.5, LINQ queries are cached automatically.</span></span> <span data-ttu-id="e50d0-107">但是，您仍可以使用已编译的 LINQ 查询来降低后续执行中的这一开销，编译的查询比自动缓存的 LINQ 查询效率更高。</span><span class="sxs-lookup"><span data-stu-id="e50d0-107">However, you can still use compiled LINQ queries to reduce this cost in later executions and compiled queries can be more efficient than LINQ queries that are automatically cached.</span></span> <span data-ttu-id="e50d0-108">请注意，不自动缓存将 `Enumerable.Contains` 运算符应用到内存中集合的 LINQ to Entities 查询。</span><span class="sxs-lookup"><span data-stu-id="e50d0-108">Note that LINQ to Entities queries that apply the `Enumerable.Contains` operator to in-memory collections are not automatically cached.</span></span> <span data-ttu-id="e50d0-109">此外，不允许在已编译的 LINQ 查询中参数化内存中的集合。</span><span class="sxs-lookup"><span data-stu-id="e50d0-109">Also parameterizing in-memory collections in compiled LINQ queries is not allowed.</span></span>

query.</span></span> <span data-ttu-id="e50d0-113">给定 `Compile` 及其参数值，<xref:System.Data.Objects.ObjectContext> 方法将返回生成某个结果的委托（例如 <xref:System.Linq.IQueryable%601> 实例）。</span><span class="sxs-lookup"><span data-stu-id="e50d0-113">The `Compile` methods, provided with a <xref:System.Data.Objects.ObjectContext> and parameter values, return a delegate that produces some result (such as an <xref:System.Linq.IQueryable%601> instance).</span></span> <span data-ttu-id="e50d0-114">只在第一次执行的过程中查询才编译一次。</span><span class="sxs-lookup"><span data-stu-id="e50d0-114">The query compiles once during only the first execution.</span></span> <span data-ttu-id="e50d0-115">编译时为查询设置的合并选项在以后无法更改。</span><span class="sxs-lookup"><span data-stu-id="e50d0-115">The merge options set for the query at the time of the compilation cannot be changed later.</span></span> <span data-ttu-id="e50d0-116">对查询进行编译后，您可以仅提供基元类型的参数，但不能替换将更改生成的 SQL 的查询部分。</span><span class="sxs-lookup"><span  <span data-ttu-id="e50d0-118">[!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)]查询表达式的<xref:System.Data.Objects.CompiledQuery>的`Compile`由一个泛型方法编译表示`Func`委托，如<xref:System.Func%605>。</span><span class="sxs-lookup"><span data-stu-id="e50d0-118">The [!INCLUDE[linq_entities](../../../../../../includes/linq-entities-md.md)] query expression that the <xref:System.Data.Objects.CompiledQuery>'s `Compile` method compiles is represented by one of the generic `Func` delegates, such as <xref:System.Func%605>.</span></span> <span data-ttu-id="e50d0-119">查询表达式最多可以包装一个 `ObjectContext` 参数、一个返回参数和 16 个查询参数。</span><span class="sxs-lookup"><span data-stu-id="e50d0-119">At most, the query expression can encapsulate an `ObjectContext` parameter, a return parameter, and 16 query parameters.</span></span> <span data-ttu-id="e50d0-120">如果需要的查询参数不止 16 个，则可以创建一个结构并用其属性表示查询参数。</span><span class="sxs-lookup"><span data-stu-id="e50d0-120">If more than 16 query parameters are required, you can create a structure whose properties represent query parameters.</span></span> <span data-ttu-id="e50d0-121">然后，该结构上的这些属性在经过设置之后就可以用于查询表达式中。</span><span class="sxs-lookup"><span data-stu-id="e50d0-121">You can then use the properties on the structure in the query expression after you set the properties.</span></span>
