# <a name="entity-data-model-inheritance"></a><span data-ttu-id="5b139-102">实体数据模型：继承</span><span class="sxs-lookup"><span data-stu-id="5b139-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="5b139-103">实体数据模型 (EDM) 支持的继承[实体类型](../../../../docs/framework/data/adonet/entity-type.md)。</span><span class="sxs-lookup"><span data-stu-id="5b139-103">The Entity Data Model (EDM) supports inheritance for [entity types](../../../../docs/framework/data/adonet/entity-type.md).</span></span> <span data-ttu-id="5b139-104">EDM 中的继承与面向对象的编程语言中的类的继承类似。</span><span class="sxs-lookup"><span data-stu-id="5b139-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="5b139-105">像使用面向对象语言中的类，概念模型中您可以定义一个实体类型 (*派生类型*) 的继承自另一个实体类型 (*基类型*)。</span><span class="sxs-lookup"><span data-stu-id="5b139-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="5b139-106">但是，与面向对象的编程中的类，不同的概念模型中的派生的类型始终会继承所有[属性](../../../../docs/framework/data/adonet/property.md)并[导航属性](../../../../docs/framework/data/adonet/navigation-property.md)的基类型。</span><span class="sxs-lookup"><span data-stu-id="5b139-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](../../../../docs/framework/data/adonet/property.md) and [navigation properties](../../../../docs/framework/data/adonet/navigation-property.md) of the base type.</span></span> <span data-ttu-id="5b139-107">不能重写派生类型中的继承属性。</span><span class="sxs-lookup"><span data-stu-id="5b139-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="5b139-108">在概念模型中，可以构建继承层次结构，其中一个派生类型将继承自另一个派生类型。</span><span class="sxs-lookup"><span data-stu-id="5b139-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="5b139-109">在层次结构 （不是派生的类型的层次结构中的一个类型） 的顶部的类型称为*根类型*。</span><span class="sxs-lookup"><span data-stu-id="5b139-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="5b139-110">在继承层次结构，[实体键](../../../../docs/framework/data/adonet/entity-key.md)必须在根类型上定义。</span><span class="sxs-lookup"><span data-stu-id="5b139-110">In an inheritance hierarchy, the [entity key](../../../../docs/framework/data/adonet/entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="5b139-111">不能构建这样的继承层次结构，即一个派生类型继承自多个类型。</span><span class="sxs-lookup"><span data-stu-id="5b139-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="5b139-112">例如，在包含 `Book` 实体类型的概念模型中，可以定义都是继承自 `FictionBook` 的派生类型 `NonFictionBook` 和 `Book`。</span><span class="sxs-lookup"><span data-stu-id="5b139-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="5b139-113">但是，随后不能定义同时继承自 `FictionBook` 和 `NonFictionBook` 类型的类型。</span><span class="sxs-lookup"><span data-stu-id="5b139-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
