 <span data-ttu-id="4a8ee-110">生成的代码[!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)]默认情况下提供延迟的加载功能。</span><span class="sxs-lookup"><span data-stu-id="4a8ee-110">The code generated by [!INCLUDE[vbtecdlinq](../../../../../../includes/vbtecdlinq-md.md)] provides deferred loading capabilities by default.</span></span> <span data-ttu-id="4a8ee-111">延迟加载对在中间层以透明方式根据需要加载数据而言非常方便。</span><span class="sxs-lookup"><span data-stu-id="4a8ee-111">Deferred loading is very convenient on the mid-tier for transparent loading of data on demand.</span></span> <span data-ttu-id="4a8ee-112">但是，它会给序列化过程带来问题，原因是不论是否需要进行延迟加载，序列化程序都会触发延迟加载。</span><span class="sxs-lookup"><span data-stu-id="4a8ee-112">However, it is problematic for serialization, because the serializer triggers deferred loading whether deferred loading is intended or not.</span></span> <span data-ttu-id="4a8ee-113">实际上，对对象进行序列化时，会对其在所有出站延迟加载引用下的可传递闭包进行序列化。</span><span class="sxs-lookup"><span data-stu-id="4a8ee-113">In effect, when an object is serialized, its transitive closure under all outbound defer-loaded references is serialized.</span></span>
