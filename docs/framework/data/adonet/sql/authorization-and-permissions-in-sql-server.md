 <span data-ttu-id="186c2-118">向角色而不是用户授予权限可简化安全管理。</span><span class="sxs-lookup"><span data-stu-id="186c2-118">Granting permissions to roles rather than to users simplifies security administration.</span></span> <span data-ttu-id="186c2-119">分配给角色的权限集由该角色的所有成员继承。</span><span class="sxs-lookup"><span data-stu-id="186c2-119">Permission sets that are assigned to roles are inherited by all members of the role.</span></span> <span data-ttu-id="186c2-120">在角色中添加或移除用户要比为单个用户重新创建单独的权限集更为简便。</span><span class="sxs-lookup"><span data-stu-id="186c2-120">It is easier to add or remove users from a role than it is to recreate separate permission sets for individual users.</span></span> <span data-ttu-id="186c2-121">角色可以嵌套；但是，嵌套的级别太多会降低性能。</span><span class="sxs-lookup"><span data-stu-id="186c2-121">Roles can be nested; however, too many levels of nesting can degrade performance.</span></span> <span data-ttu-id="186c2-122">您也可以将用户添加到固定数据库角色来简化权限的分配。</span><span class="sxs-lookup"><span data-stu-id="186c2-122">You can also add users to fixed database roles to simplify assigning permissions.</span></span>

## <a name="ownership-chains"></a><span data-ttu-id="186c2-147">所有权链</span><span class="sxs-lookup"><span data-stu-id="186c2-147">Ownership Chains</span></span>  
 <span data-ttu-id="186c2-148">SQL Server 确保只有已被授予权限的主体才能访问对象。</span><span class="sxs-lookup"><span data-stu-id="186c2-148">SQL Server ensures that only principals that have been granted permission can access objects.</span></span> <span data-ttu-id="186c2-149">当多个数据库对象互相访问时，该序列称为链。</span><span class="sxs-lookup"><span data-stu-id="186c2-149">When multiple database objects access each other, the sequence is known as a chain.</span></span> <span data-ttu-id="186c2-150">当 SQL Server 遍历链中的链接时，它计算权限的方式不同于单独访问各项时计算权限的方式。</span><span class="sxs-lookup"><span data-stu-id="186c2-150">When SQL Server is traversing the links in the chain, it evaluates permissions differently than it would if it were accessing each item separately.</span></span> <span data-ttu-id="186c2-151">当通过链访问一个对象时，SQL Server 首选将对象的所有者与调用对象的所有者（链中的上一个链接）进行比较。</span><span class="sxs-lookup"><span data-stu-id="186c2-151">When an object is accessed through a chain, SQL Server first compares the object's owner to the owner of the calling object (the previous link in the chain).</span></span> <span data-ttu-id="186c2-152">如果这两个对象具有相同所有者，则不检查被引用对象的权限。</span><span class="sxs-lookup"><span data-stu-id="186c2-152">If both objects have the same owner, permissions on the referenced object are not checked.</span></span> <span data-ttu-id="186c2-153">每当一个对象访问另一个具有不同所有者的对象时，所有权链就会断开，因此 SQL Server 必须检查调用方的安全上下文。</span><span class="sxs-lookup"><span data-stu-id="186c2-153">Whenever an object accesses another object that has a different owner, the ownership chain is broken and SQL Server must check the caller's security context.</span></span>
