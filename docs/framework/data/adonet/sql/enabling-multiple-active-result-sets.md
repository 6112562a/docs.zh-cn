## <a name="special-considerations-when-using-mars"></a><span data-ttu-id="f925d-117">使用 MARS 时的特殊注意事项</span><span class="sxs-lookup"><span data-stu-id="f925d-117">Special Considerations When Using MARS</span></span>

### <a name="statement-interleaving"></a><span data-ttu-id="f925d-120">语句交替</span><span class="sxs-lookup"><span data-stu-id="f925d-120">Statement Interleaving</span></span>  
 <span data-ttu-id="f925d-121">MARS 操作在服务器上同步执行。</span><span class="sxs-lookup"><span data-stu-id="f925d-121">MARS operations execute synchronously on the server.</span></span> <span data-ttu-id="f925d-122">允许 SELECT 和 BULK INSERT 语句的语句交替。</span><span class="sxs-lookup"><span data-stu-id="f925d-122">Statement interleaving of SELECT and BULK INSERT statements is allowed.</span></span> <span data-ttu-id="f925d-123">但是，数据操作语言 (DML) 和数据定义语言 (DDL) 语句会自动执行。</span><span class="sxs-lookup"><span data-stu-id="f925d-123">However, data manipulation language (DML) and data definition language (DDL) statements execute atomically.</span></span> <span data-ttu-id="f925d-124">将阻止任何在执行原子批处理时尝试执行的语句。</span><span class="sxs-lookup"><span data-stu-id="f925d-124">Any statements attempting to execute while an atomic batch is executing are blocked.</span></span> <span data-ttu-id="f925d-125">服务器上的并行执行不是 MARS 功能。</span><span class="sxs-lookup"><span data-stu-id="f925d-125">Parallel execution at the server is not a MARS feature.</span></span>  
  
 <span data-ttu-id="f925d-126">如果在 MARS 连接下提交两个批处理，其中一个批处理包含 SELECT 语句，另一个包含 DML 语句，DML 可以在 SELECT 语句执行过程中开始执行。</span><span class="sxs-lookup"><span data-stu-id="f925d-126">If two batches are submitted under a MARS connection, one of them containing a SELECT statement, the other containing a DML statement, the DML can begin execution within execution of the SELECT statement.</span></span> <span data-ttu-id="f925d-127">但是，DML 语句必须运行完成，SELECT 语句才可以继续执行。</span><span class="sxs-lookup"><span data-stu-id="f925d-127">However, the DML statement must run to completion before the SELECT statement can make progress.</span></span> <span data-ttu-id="f925d-128">如果两个语句在相同事务下运行，读取操作将看不到 DML 语句在 SELECT 语句开始执行后所作的任何更改。</span><span class="sxs-lookup"><span data-stu-id="f925d-128">If both statements are running under the same transaction, any changes made by a DML statement after the SELECT statement has started execution are not visible to the read operation.</span></span>  
  
 <span data-ttu-id="f925d-129">SELECT 语句中的 WAITFOR 语句在等待时不生成事务，即直到生成第一行时才生成事务。</span><span class="sxs-lookup"><span data-stu-id="f925d-129">A WAITFOR statement inside a SELECT statement does not yield the transaction while it is waiting, that is, until the first row is produced.</span></span> <span data-ttu-id="f925d-130">这意味着在 WAITFOR 语句等待时，无法在相同连接内执行任何其他批处理。</span><span class="sxs-lookup"><span data-stu-id="f925d-130">This implies that no other batches can execute within the same connection while a WAITFOR statement is waiting.</span></span>  
  
### <a name="mars-session-cache"></a><span data-ttu-id="f925d-131">MARS 会话缓存</span><span class="sxs-lookup"><span data-stu-id="f925d-131">MARS Session Cache</span></span>
