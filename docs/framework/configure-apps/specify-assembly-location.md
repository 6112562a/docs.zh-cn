## <a name="using-the-codebase-element"></a><span data-ttu-id="47849-107">使用\<b a s e > 元素</span><span class="sxs-lookup"><span data-stu-id="47849-107">Using the \<codeBase> Element</span></span>  
 <span data-ttu-id="47849-108">可以使用 **\<b a s e >** 只能机配置或发布服务器策略在文件中还将程序集版本重定向的元素。</span><span class="sxs-lookup"><span data-stu-id="47849-108">You can use the **\<codeBase>** element only in machine configuration or publisher policy files that also redirect the assembly version.</span></span> <span data-ttu-id="47849-109">当运行时确定要使用的程序集版本时，则会应用确定版本的文件的基本代码设置。</span><span class="sxs-lookup"><span data-stu-id="47849-109">When the runtime determines which assembly version to use, it applies the code base setting from the file that determines the version.</span></span> <span data-ttu-id="47849-110">如果指示没有基本代码，运行时探测程序集以正常方式。</span><span class="sxs-lookup"><span data-stu-id="47849-110">If no code base is indicated, the runtime probes for the assembly in the normal way.</span></span> <span data-ttu-id="47849-111">有关详细信息，请参阅[运行时如何定位程序集](../../../docs/framework/deployment/how-the-runtime-locates-assemblies.md)。</span><span class="sxs-lookup"><span data-stu-id="47849-111">For details, see [How the Runtime Locates Assemblies](../../../docs/framework/deployment/how-the-runtime-locates-assemblies.md).</span></span>  
  
 <span data-ttu-id="47849-112">下面的示例演示如何指定程序集的位置。</span><span class="sxs-lookup"><span data-stu-id="47849-112">The following example shows how to specify an assembly's location.</span></span>  
  
```xml  
<configuration>  
   <runtime>  
      <assemblyBinding xmlns="urn:schemas-microsoft-com:asm.v1">  
       <dependentAssembly>  
         <assemblyIdentity name="myAssembly"  
                           publicKeyToken="32ab4ba45e0a69a1"  
                           culture="en-us" />  
         <codeBase version="2.0.0.0"  
                   href="http://www.litwareinc.com/myAssembly.dll"/>  
       </dependentAssembly>  
      </assemblyBinding>  
   </runtime>  
</configuration>  
```  
  
 <span data-ttu-id="47849-113">**版本**属性是必需的所有强名称的程序集，但不是具有强名称的程序集，应省略。</span><span class="sxs-lookup"><span data-stu-id="47849-113">The **version** attribute is required for all strong-named assemblies but should be omitted for assemblies that are not strong-named.</span></span> <span data-ttu-id="47849-114"> *\*\<B a s e >** 元素需要*\*href*\*属性。</span><span class="sxs-lookup"><span data-stu-id="47849-114">The **\<codeBase>** element requires the **href** attribute.</span></span> <span data-ttu-id="47849-115">不能指定版本范围 **\<b a s e >** 元素。</span><span class="sxs-lookup"><span data-stu-id="47849-115">You cannot specify version ranges in the **\<codeBase>** element.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="47849-116">如果你所提供的不是强名称的程序集的基本代码的提示，提示必须指向应用程序基控件或应用程序基目录的子目录。</span><span class="sxs-lookup"><span data-stu-id="47849-116">If you are supplying a code base hint for an assembly that is not strong-named, the hint must point to the application base or a subdirectory of the application base directory.</span></span>  
  
## <a name="using-the-probing-element"></a><span data-ttu-id="47849-117">使用\<探测 > 元素</span><span class="sxs-lookup"><span data-stu-id="47849-117">Using the \<probing> Element</span></span>  
 <span data-ttu-id="47849-118">运行时定位程序集不具有通过探测的基本代码。</span><span class="sxs-lookup"><span data-stu-id="47849-118">The runtime locates assemblies that do not have a code base by probing.</span></span> <span data-ttu-id="47849-119">探测的详细信息，请参阅[运行时如何定位程序集](../../../docs/framework/deployment/how-the-runtime-locates-assemblies.md)。</span><span class="sxs-lookup"><span data-stu-id="47849-119">For more information about probing, see [How the Runtime Locates Assemblies](../../../docs/framework/deployment/how-the-runtime-locates-assemblies.md).</span></span>  
  
 <span data-ttu-id="47849-120">可以使用[\<探测 >](../../../docs/framework/configure-apps/file-schema/runtime/probing-element.md)应用程序配置文件来指定运行时查找程序集时应搜索的子目录中的元素。</span><span class="sxs-lookup"><span data-stu-id="47849-120">You can use the [\<probing>](../../../docs/framework/configure-apps/file-schema/runtime/probing-element.md) element in the application configuration file to specify subdirectories the runtime should search when locating an assembly.</span></span> <span data-ttu-id="47849-121">下面的示例演示如何指定运行时应搜索的目录。</span><span class="sxs-lookup"><span data-stu-id="47849-121">The following example shows how to specify directories the runtime should search.</span></span>
