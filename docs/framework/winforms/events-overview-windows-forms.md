## <a name="delegates-and-their-role"></a><span data-ttu-id="3ef52-114">대리자 및 해당 역할</span><span class="sxs-lookup"><span data-stu-id="3ef52-114">Delegates and Their Role</span></span>
 <span data-ttu-id="3ef52-115">委托是在 .NET Framework 中通常用于生成事件处理机制的类。</span><span class="sxs-lookup"><span data-stu-id="3ef52-115">Delegates are classes commonly used within the .NET Framework to build event-handling mechanisms.</span></span> <span data-ttu-id="3ef52-116">委托大致等同于函数指针，通常用于视觉C++对象和其他面向对象的语言。</span><span class="sxs-lookup"><span data-stu-id="3ef52-116">Delegates roughly equate to function pointers, commonly used in Visual C++ and other object-oriented languages.</span></span> <span data-ttu-id="3ef52-117">그러나 함수 포인터와는 달리 대리자는 개체 지향적이고 형식이 안전하며 보안이 유지됩니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-117">Unlike function pointers however, delegates are object-oriented, type-safe, and secure.</span></span> <span data-ttu-id="3ef52-118">또한 함수 포인터는 특정 함수에 대한 참조만을 포함하지만 대리자는 개체 참조와 해당 개체 내에 있는 하나 이상의 메서드에 대한 참조로 구성됩니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-118">In addition, where a function pointer contains only a reference to a particular function, a delegate consists of a reference to an object, and references to one or more methods within the object.</span></span>

 <span data-ttu-id="3ef52-119">此事件模型使用*委托*将事件绑定到用于处理这些事件的方法。</span><span class="sxs-lookup"><span data-stu-id="3ef52-119">This event model uses *delegates* to bind events to the methods that are used to handle them.</span></span> <span data-ttu-id="3ef52-120">대리자는 처리기 메서드를 지정하여 다른 클래스가 이벤트 알림을 등록할 수 있도록 설정합니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-120">The delegate enables other classes to register for event notification by specifying a handler method.</span></span> <span data-ttu-id="3ef52-121">이벤트가 발생하면 대리자가 bound 메서드를 호출합니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-121">When the event occurs, the delegate calls the bound method.</span></span> <span data-ttu-id="3ef52-122">有关如何定义委托的详细信息，请参阅[事件](../../standard/events/index.md)。</span><span class="sxs-lookup"><span data-stu-id="3ef52-122">For more information about how to define delegates, see [Events](../../standard/events/index.md).</span></span>

<span data-ttu-id="3ef52-123">대리자는 단일 메서드에 바인딩될 수도 있고 여러 메서드에 바인딩(멀티캐스트)될 수도 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-123">Delegates can be bound to a single method or to multiple methods, referred to as multicasting.</span></span> <span data-ttu-id="3ef52-124">为事件创建委托时，您（或 Windows）通常会创建多路广播事件。</span><span class="sxs-lookup"><span data-stu-id="3ef52-124">When creating a delegate for an event, you (or the Windows) typically create a multicast event.</span></span> <span data-ttu-id="3ef52-125">이때 논리적으로 이벤트당 여러 번 반복되지 않는 특정 절차(예: 대화 상자 표시)를 수행하는 이벤트가 드물지만 예외적으로 발생할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-125">A rare exception might be an event that results in a specific procedure (such as displaying a dialog box) that would not logically repeat multiple times per event.</span></span> <span data-ttu-id="3ef52-126">有关如何创建多播委托的信息，请参阅[如何合并委托（多路广播委托）](../../csharp/programming-guide/delegates/how-to-combine-delegates-multicast-delegates.md)。</span><span class="sxs-lookup"><span data-stu-id="3ef52-126">For information about how to create a multicast delegate, see [How to combine delegates (Multicast Delegates)](../../csharp/programming-guide/delegates/how-to-combine-delegates-multicast-delegates.md).</span></span>

 <span data-ttu-id="3ef52-127">멀티캐스트 대리자는 바인딩 대상 메서드의 호출 목록을 유지 관리하며,</span><span class="sxs-lookup"><span data-stu-id="3ef52-127">A multicast delegate maintains an invocation list of the methods it is bound to.</span></span> <span data-ttu-id="3ef52-128">호출 목록에 메서드를 추가하기 위한 <xref:System.Delegate.Combine%2A> 메서드와 해당 메서드를 제거하기 위한 <xref:System.Delegate.Remove%2A> 메서드를 지원합니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-128">The multicast delegate supports a <xref:System.Delegate.Combine%2A> method to add a method to the invocation list and a <xref:System.Delegate.Remove%2A> method to remove it.</span></span>

 <span data-ttu-id="3ef52-129">애플리케이션이 이벤트를 기록하면 컨트롤은 해당 이벤트의 대리자를 호출하여 이벤트를 발생시킵니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-129">When an event is recorded by the application, the control raises the event by invoking the delegate for that event.</span></span> <span data-ttu-id="3ef52-130">그러면 대리자는 bound 메서드를 호출합니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-130">The delegate in turn calls the bound method.</span></span> <span data-ttu-id="3ef52-131">가장 일반적인 경우(멀티캐스트 대리자)에서 대리자는 호출 목록의 각 bound 메서드를 차례로 호출하므로 일대다 알림이 제공됩니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-131">In the most common case (a multicast delegate) the delegate calls each bound method in the invocation list in turn, which provides a one-to-many notification.</span></span> <span data-ttu-id="3ef52-132">이러한 전략이 사용되므로 컨트롤이 이벤트 알림을 위해 대상 개체 목록을 유지하지 않아도 됩니다. 대리자가 모든 등록과 알림을 처리하기 때문입니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-132">This strategy means that the control does not need to maintain a list of target objects for event notification—the delegate handles all registration and notification.</span></span>

 <span data-ttu-id="3ef52-133">또한 대리자는 여러 이벤트를 같은 메서드에 바인딩할 수 있도록 함으로써 다대일 알림도 허용합니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-133">Delegates also enable multiple events to be bound to the same method, allowing a many-to-one notification.</span></span> <span data-ttu-id="3ef52-134">예를 들어 단추 클릭 이벤트와 메뉴 명령 클릭 이벤트 모두가 같은 대리자를 호출할 수 있으며, 이 대리자는 단일 메서드를 호출하여 두 개별 이벤트를 같은 방식으로 처리합니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-134">For example, a button-click event and a menu-command–click event can both invoke the same delegate, which then calls a single method to handle these separate events the same way.</span></span>

 <span data-ttu-id="3ef52-135">대리자에는 동적 바인딩 메커니즘이 사용되므로 대리자는 런타임에 서명이 이벤트 처리기의 서명과 일치하는 모든 메서드에 바인딩될 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-135">The binding mechanism used with delegates is dynamic: a delegate can be bound at run time to any method whose signature matches that of the event handler.</span></span> <span data-ttu-id="3ef52-136">이 기능을 사용하면 조건에 따라 bound 메서드를 설정하거나 변경하고 이벤트 처리기를 컨트롤에 동적으로 연결할 수 있습니다.</span><span class="sxs-lookup"><span data-stu-id="3ef52-136">With this feature, you can set up or change the bound method depending on a condition and to dynamically attach an event handler to a control.</span></span>
