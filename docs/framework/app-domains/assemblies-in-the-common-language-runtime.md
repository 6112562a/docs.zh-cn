# <a name="assemblies-in-the-common-language-runtime"></a><span data-ttu-id="65044-102">公共语言运行时中的程序集</span><span class="sxs-lookup"><span data-stu-id="65044-102">Assemblies in the Common Language Runtime</span></span>
<span data-ttu-id="65044-103">程序集是 .NET Framework 应用程序的构造块；它们形成了部署、版本控制、重复使用、激活范围和安全权限的基本单元。</span><span class="sxs-lookup"><span data-stu-id="65044-103">Assemblies are the building blocks of .NET Framework applications; they form the fundamental unit of deployment, version control, reuse, activation scoping, and security permissions.</span></span> <span data-ttu-id="65044-104">程序集是为协同工作而生成的类型和资源的集合，这些类型和资源构成了一个逻辑功能单元。</span><span class="sxs-lookup"><span data-stu-id="65044-104">An assembly is a collection of types and resources that are built to work together and form a logical unit of functionality.</span></span> <span data-ttu-id="65044-105">程序集向公共语言运行时提供了解类型实现所需要的信息。</span><span class="sxs-lookup"><span data-stu-id="65044-105">An assembly provides the common language runtime with the information it needs to be aware of type implementations.</span></span> <span data-ttu-id="65044-106">对于运行时，类型不存在于程序集上下文之外。</span><span class="sxs-lookup"><span data-stu-id="65044-106">To the runtime, a type does not exist outside the context of an assembly.</span></span>  
  
 <span data-ttu-id="65044-107">程序集执行以下功能：</span><span class="sxs-lookup"><span data-stu-id="65044-107">An assembly performs the following functions:</span></span>  
  
- <span data-ttu-id="65044-108">包含公共语言运行时执行的代码。</span><span class="sxs-lookup"><span data-stu-id="65044-108">It contains code that the common language runtime executes.</span></span> <span data-ttu-id="65044-109">如果可迁移可执行 (PE) 文件没有相关联的程序集清单，则将不执行该文件中的 Microsoft 中间语言 (MSIL) 代码。</span><span class="sxs-lookup"><span data-stu-id="65044-109">Microsoft intermediate language (MSIL) code in a portable executable (PE) file will not be executed if it does not have an associated assembly manifest.</span></span> <span data-ttu-id="65044-110">请注意，每个程序集只能有一个入口点（即 `DllMain`、`WinMain` 或 `Main`）。</span><span class="sxs-lookup"><span data-stu-id="65044-110">Note that each assembly can have only one entry point (that is, `DllMain`, `WinMain`, or `Main`).</span></span>  
  
- <span data-ttu-id="65044-111">程序集形成安全边界。</span><span class="sxs-lookup"><span data-stu-id="65044-111">It forms a security boundary.</span></span> <span data-ttu-id="65044-112">程序集就是在其中请求和授予权限的单元。</span><span class="sxs-lookup"><span data-stu-id="65044-112">An assembly is the unit at which permissions are requested and granted.</span></span> <span data-ttu-id="65044-113">有关应用于程序集的安全边界的更多信息，请参见[程序集安全注意事项](../../../docs/framework/app-domains/assembly-security-considerations.md)。</span><span class="sxs-lookup"><span data-stu-id="65044-113">For more information about security boundaries as they apply to assemblies, see [Assembly Security Considerations](../../../docs/framework/app-domains/assembly-security-considerations.md).</span></span>  
  
- <span data-ttu-id="65044-114">程序集形成类型边界。</span><span class="sxs-lookup"><span data-stu-id="65044-114">It forms a type boundary.</span></span> <span data-ttu-id="65044-115">每一类型的标识均包括该类型所驻留的程序集的名称。</span><span class="sxs-lookup"><span data-stu-id="65044-115">Every type's identity includes the name of the assembly in which it resides.</span></span> <span data-ttu-id="65044-116">在一个程序集的范围中加载的称为 `MyType` 的类型不同于在另一个程序集范围中加载的称为 `MyType` 的类型。</span><span class="sxs-lookup"><span data-stu-id="65044-116">A type called `MyType` that is loaded in the scope of one assembly is not the same as a type called `MyType` that is loaded in the scope of another assembly.</span></span>  
  
- <span data-ttu-id="65044-117">程序集形成引用范围边界。</span><span class="sxs-lookup"><span data-stu-id="65044-117">It forms a reference scope boundary.</span></span> <span data-ttu-id="65044-118">程序集的清单包含用于解析类型和满足资源请求的程序集元数据。</span><span class="sxs-lookup"><span data-stu-id="65044-118">The assembly's manifest contains assembly metadata that is used for resolving types and satisfying resource requests.</span></span> <span data-ttu-id="65044-119">它指定在该程序集之外公开的类型和资源。</span><span class="sxs-lookup"><span data-stu-id="65044-119">It specifies the types and resources that are exposed outside the assembly.</span></span> <span data-ttu-id="65044-120">该清单还枚举它所依赖的其他程序集。</span><span class="sxs-lookup"><span data-stu-id="65044-120">The manifest also enumerates other assemblies on which it depends.</span></span>  
  
- <span data-ttu-id="65044-121">程序集形成版本边界。</span><span class="sxs-lookup"><span data-stu-id="65044-121">It forms a version boundary.</span></span> <span data-ttu-id="65044-122">程序集是公共语言运行时中最小的可版本化单元，同一程序集中的所有类型和资源均会被版本化为一个单元。</span><span class="sxs-lookup"><span data-stu-id="65044-122">The assembly is the smallest versionable unit in the common language runtime; all types and resources in the same assembly are versioned as a unit.</span></span> <span data-ttu-id="65044-123">程序集的清单描述你为任何依赖项程序集所指定的版本依赖性。</span><span class="sxs-lookup"><span data-stu-id="65044-123">The assembly's manifest describes the version dependencies you specify for any dependent assemblies.</span></span> <span data-ttu-id="65044-124">有关版本控制的更多信息，请参见[程序集版本控制](../../../docs/framework/app-domains/assembly-versioning.md)。</span><span class="sxs-lookup"><span data-stu-id="65044-124">For more information about versioning, see [Assembly Versioning](../../../docs/framework/app-domains/assembly-versioning.md).</span></span>  
  
- <span data-ttu-id="65044-125">程序集形成部署单元。</span><span class="sxs-lookup"><span data-stu-id="65044-125">It forms a deployment unit.</span></span> <span data-ttu-id="65044-126">当一个应用程序启动时，只有该应用程序最初调用的程序集必须存在。</span><span class="sxs-lookup"><span data-stu-id="65044-126">When an application starts, only the assemblies that the application initially calls must be present.</span></span> <span data-ttu-id="65044-127">其他程序集（例如本地化资源和包含实用工具类的程序集）可以按需检索。</span><span class="sxs-lookup"><span data-stu-id="65044-127">Other assemblies, such as localization resources or assemblies containing utility classes, can be retrieved on demand.</span></span> <span data-ttu-id="65044-128">这就使应用程序在第一次下载时保持精简。</span><span class="sxs-lookup"><span data-stu-id="65044-128">This allows applications to be kept simple and thin when first downloaded.</span></span> <span data-ttu-id="65044-129">有关部署程序集的更多信息，请参见[部署应用程序](../../../docs/framework/deployment/index.md)。</span><span class="sxs-lookup"><span data-stu-id="65044-129">For more information about deploying assemblies, see [Deploying Applications](../../../docs/framework/deployment/index.md).</span></span>  
  
- <span data-ttu-id="65044-130">程序集是支持并行执行的单元。</span><span class="sxs-lookup"><span data-stu-id="65044-130">It is the unit at which side-by-side execution is supported.</span></span> <span data-ttu-id="65044-131">有关运行多个程序集版本的更多信息，请参见[程序集和并行执行](../../../docs/framework/app-domains/assemblies-and-side-by-side-execution.md)。</span><span class="sxs-lookup"><span data-stu-id="65044-131">For more information about running multiple versions of an assembly, see [Assemblies and Side-by-Side Execution](../../../docs/framework/app-domains/assemblies-and-side-by-side-execution.md).</span></span>  
  
 <span data-ttu-id="65044-132">程序集可以为静态或动态。</span><span class="sxs-lookup"><span data-stu-id="65044-132">Assemblies can be static or dynamic.</span></span> <span data-ttu-id="65044-133">静态程序集可以包括 .NET Framework 类型（接口和类），以及该程序集的资源（位图、JPEG 文件、资源文件等）。</span><span class="sxs-lookup"><span data-stu-id="65044-133">Static assemblies can include .NET Framework types (interfaces and classes), as well as resources for the assembly (bitmaps, JPEG files, resource files, and so on).</span></span> <span data-ttu-id="65044-134">静态程序集存储在磁盘上的可迁移可执行 (PE) 文件中。</span><span class="sxs-lookup"><span data-stu-id="65044-134">Static assemblies are stored on disk in portable executable (PE) files.</span></span> <span data-ttu-id="65044-135">你还可以使用 .NET Framework 来创建动态程序集，动态程序集直接从内存运行并且在执行前不存储到磁盘上。</span><span class="sxs-lookup"><span data-stu-id="65044-135">You can also use the .NET Framework to create dynamic assemblies, which are run directly from memory and are not saved to disk before execution.</span></span> <span data-ttu-id="65044-136">你可以在执行动态程序集后将它们保存在磁盘上。</span><span class="sxs-lookup"><span data-stu-id="65044-136">You can save dynamic assemblies to disk after they have executed.</span></span>  
  
 <span data-ttu-id="65044-137">有几种创建程序集的方法。</span><span class="sxs-lookup"><span data-stu-id="65044-137">There are several ways to create assemblies.</span></span> <span data-ttu-id="65044-138">你可以使用过去用来创建 .dll 或 .exe 文件的开发工具，例如 Visual Studio。</span><span class="sxs-lookup"><span data-stu-id="65044-138">You can use development tools, such as Visual Studio, that you have used in the past to create .dll or .exe files.</span></span> <span data-ttu-id="65044-139">你可以使用 Windows SDK 中提供的工具创建带有在其他开发环境中创建的模块的程序集。</span><span class="sxs-lookup"><span data-stu-id="65044-139">You can use tools provided in the Windows SDK to create assemblies with modules created in other development environments.</span></span> <span data-ttu-id="65044-140">还可以使用公共语言运行时 API（例如 <xref:System.Reflection.Emit?displayProperty=nameWithType>）来创建动态程序集。</span><span class="sxs-lookup"><span data-stu-id="65044-140">You can also use common language runtime APIs, such as <xref:System.Reflection.Emit?displayProperty=nameWithType>, to create dynamic assemblies.</span></span>
