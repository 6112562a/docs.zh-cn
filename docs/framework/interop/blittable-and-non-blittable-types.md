# <a name="blittable-and-non-blittable-types"></a><span data-ttu-id="5723e-102">可直接复制到本机结构中的类型和非直接复制到本机结构中的类型</span><span class="sxs-lookup"><span data-stu-id="5723e-102">Blittable and Non-Blittable Types</span></span>
<span data-ttu-id="5723e-103">大多数数据类型在托管和非托管内存中具有共同的表示形式，而且不需要互操作封送处理程序进行特殊处理。</span><span class="sxs-lookup"><span data-stu-id="5723e-103">Most data types have a common representation in both managed and unmanaged memory and do not require special handling by the interop marshaler.</span></span> <span data-ttu-id="5723e-104">这些类型称为 blittable 类型，因为它们在托管和非托管代码之间传递时不需要进行转换。</span><span class="sxs-lookup"><span data-stu-id="5723e-104">These types are called *blittable types* because they do not require conversion when they are passed between managed and unmanaged code.</span></span>  
  
 <span data-ttu-id="5723e-105">从平台调用返回的结构必须是 blittable 类型。</span><span class="sxs-lookup"><span data-stu-id="5723e-105">Structures that are returned from platform invoke calls must be blittable types.</span></span> <span data-ttu-id="5723e-106">平台调用不支持返回类型为 non-blittable 结构。</span><span class="sxs-lookup"><span data-stu-id="5723e-106">Platform invoke does not support non-blittable structures as return types.</span></span>

 <span data-ttu-id="5723e-108">下面的复杂类型也是 blittable 类型：</span><span class="sxs-lookup"><span data-stu-id="5723e-108">The following complex types are also blittable types:</span></span>  
  
- <span data-ttu-id="5723e-109">所有 blittable 类型的一维数组，如整数数组。</span><span class="sxs-lookup"><span data-stu-id="5723e-109">One-dimensional arrays of blittable types, such as an array of integers.</span></span> <span data-ttu-id="5723e-110">但是，包含 blittable 类型变量数组的类型本身不是 blittable 类型。</span><span class="sxs-lookup"><span data-stu-id="5723e-110">However, a type that contains a variable array of blittable types is not itself blittable.</span></span>  
  
- <span data-ttu-id="5723e-111">所有只包含 blittable 类型（和作为格式化类型进行封送的类）的格式化的值类型。</span><span class="sxs-lookup"><span data-stu-id="5723e-111">Formatted value types that contain only blittable types (and classes if they are marshaled as formatted types).</span></span> <span data-ttu-id="5723e-112">有关格式化的值类型的详细信息，请参阅[值类型的默认封送处理](default-marshaling-behavior.md#default-marshaling-for-value-types)。</span><span class="sxs-lookup"><span data-stu-id="5723e-112">For more information about formatted value types, see [Default marshaling for value types](default-marshaling-behavior.md#default-marshaling-for-value-types).</span></span>  
  
 <span data-ttu-id="5723e-113">对象引用不是 blittable 类型。</span><span class="sxs-lookup"><span data-stu-id="5723e-113">Object references are not blittable.</span></span> <span data-ttu-id="5723e-114">这包括本身是 blittable 的对象的引用数组。</span><span class="sxs-lookup"><span data-stu-id="5723e-114">This includes an array of references to objects that are blittable by themselves.</span></span> <span data-ttu-id="5723e-115">例如，可以定义一个属于 blittable 类型的结构，但不能定义包含这些结构的引用数组的 blittable 类型。</span><span class="sxs-lookup"><span data-stu-id="5723e-115">For example, you can define a structure that is blittable, but you cannot define a blittable type that contains an array of references to those structures.</span></span>  
  
 <span data-ttu-id="5723e-116">作为一种优化方式，所有 blittable 类型数组和只包含 blittable 成员的类都是[固定的](copying-and-pinning.md)，因此无需在封送处理期间进行复制。</span><span class="sxs-lookup"><span data-stu-id="5723e-116">As an optimization, arrays of blittable types and classes that contain only blittable members are [pinned](copying-and-pinning.md) instead of copied during marshaling.</span></span> <span data-ttu-id="5723e-117">若调用方和被调用方位于同一单元中，这些类型会显示为作为 In/Out 参数被封送。</span><span class="sxs-lookup"><span data-stu-id="5723e-117">These types can appear to be marshaled as In/Out parameters when the caller and callee are in the same apartment.</span></span> <span data-ttu-id="5723e-118">但是，这些类型实际上是作为 In 形参进行封送的，而且，如果要将实参作为 In/Out 形参进行封送，则必须应用 <xref:System.Runtime.InteropServices.InAttribute> 和 <xref:System.Runtime.InteropServices.OutAttribute> 属性。</span><span class="sxs-lookup"><span data-stu-id="5723e-118">However, these types are actually marshaled as In parameters, and you must apply the <xref:System.Runtime.InteropServices.InAttribute> and <xref:System.Runtime.InteropServices.OutAttribute> attributes if you want to marshal the argument as an In/Out parameter.</span></span>  
  
 <span data-ttu-id="5723e-119">在非托管环境中，某些托管数据类型要求具有不同的表示形式。</span><span class="sxs-lookup"><span data-stu-id="5723e-119">Some managed data types require a different representation in an unmanaged environment.</span></span> <span data-ttu-id="5723e-120">必须将这些 non-blittable 数据类型转换为可以封送的形式。</span><span class="sxs-lookup"><span data-stu-id="5723e-120">These non-blittable data types must be converted into a form that can be marshaled.</span></span> <span data-ttu-id="5723e-121">例如，托管字符串就是 non-blittable 类型，因为这些字符串必须转换为字符串对象后才能进行封送。</span><span class="sxs-lookup"><span data-stu-id="5723e-121">For example, managed strings are non-blittable types because they must be converted into string objects before they can be marshaled.</span></span>
