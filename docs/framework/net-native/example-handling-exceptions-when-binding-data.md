## <a name="what-was-the-app-doing"></a><span data-ttu-id="2a3b7-108">应用过去在执行什么操作？</span><span class="sxs-lookup"><span data-stu-id="2a3b7-108">What was the app doing?</span></span>  
 <span data-ttu-id="2a3b7-109">在堆栈底部帧从<xref:Windows.UI.Xaml?displayProperty=nameWithType>命名空间指示的 XAML 呈现引擎正在运行。</span><span class="sxs-lookup"><span data-stu-id="2a3b7-109">At the base of the stack, frames from the <xref:Windows.UI.Xaml?displayProperty=nameWithType> namespace indicate that the XAML rendering engine was running.</span></span>   <span data-ttu-id="2a3b7-110">对 <xref:System.Reflection.PropertyInfo.GetValue%2A?displayProperty=nameWithType> 方法的显示了属性值的基于反射的查找，该值位于元数据遭到删除的类型上。</span><span class="sxs-lookup"><span data-stu-id="2a3b7-110">The use of the <xref:System.Reflection.PropertyInfo.GetValue%2A?displayProperty=nameWithType> method indicates a reflection-based lookup of a property’s value on the type whose metadata was removed.</span></span>  
  
 <span data-ttu-id="2a3b7-111">第一步是提供一个元数据指令，将其添加到该类型的 `serialize` 元数据，使其所有属性都可访问：</span><span class="sxs-lookup"><span data-stu-id="2a3b7-111">The first step in providing a metadata directive would be to add `serialize` metadata for the type so that its properties are all accessible:</span></span>

 <span data-ttu-id="2a3b7-113">在此情况下，如果数据绑定拥有一个 `ViewModel` 的不完整元数据，它对于其他模型可能也是如此。</span><span class="sxs-lookup"><span data-stu-id="2a3b7-113">In this scenario, if data binding has incomplete metadata for one `ViewModel`, it may for others, too.</span></span>  <span data-ttu-id="2a3b7-114">如果代码的结构方式使得该应用的查看模型都位于 `App.ViewModels` 命名空间，你可以使用一个更一般的运行时指令：</span><span class="sxs-lookup"><span data-stu-id="2a3b7-114">If the code is structured in a way that the app’s view models are all in the `App.ViewModels` namespace, you could use a more general runtime directive:</span></span>

  
## <a name="could-the-code-be-rewritten-to-not-use-reflection"></a><span data-ttu-id="2a3b7-115">代码能够重新，改为不使用反射吗？</span><span class="sxs-lookup"><span data-stu-id="2a3b7-115">Could the code be rewritten to not use reflection?</span></span>

 <span data-ttu-id="2a3b7-117">然而，有几种方法可以指定 `ViewModel` 到 XAML 页面，从而让工具链在汇编时间可以将属性绑定与正确的类型关联起来并保存元数据，而不必使用运行时指令。</span><span class="sxs-lookup"><span data-stu-id="2a3b7-117">However, there are ways to specify the `ViewModel` to the XAML page so that the tool chain can associate property bindings with the correct type at compile time and keep the metadata without using a runtime directive.</span></span>  <span data-ttu-id="2a3b7-118">例如，可以应用<xref:Windows.UI.Xaml.Data.BindableAttribute?displayProperty=nameWithType>属性上的属性。</span><span class="sxs-lookup"><span data-stu-id="2a3b7-118">For example, you could apply the <xref:Windows.UI.Xaml.Data.BindableAttribute?displayProperty=nameWithType> attribute on properties.</span></span> <span data-ttu-id="2a3b7-119">这会使得 XAML 编译器生成所需的查找信息并避免在 Default.rd.xml 文件中要求一个运行时指令。</span><span class="sxs-lookup"><span data-stu-id="2a3b7-119">This causes the XAML compiler to generate the required lookup information and avoids requiring a runtime directive in the Default.rd.xml file.</span></span>
