# <a name="ihosttaskalert-method"></a>IHostTask::Alert 方法
请求宿主唤醒当前[IHostTask](../../../../docs/framework/unmanaged-api/hosting/ihosttask-interface.md)实例表示的任务, 以便可以中止该任务。  
  
## <a name="syntax"></a>语法  
  
```cpp  
HRESULT Alert ();  
```  
  
## <a name="return-value"></a>返回值  
  
|HRESULT|描述|  
|-------------|-----------------|  
|S_OK|方法已成功返回。|  
|HOST_E_CLRNOTAVAILABLE|公共语言运行时 (CLR) 未加载到进程中, 或 CLR 处于无法运行托管代码或成功处理调用的状态。|  
|HOST_E_TIMEOUT|调用超时。|  
|HOST_E_NOT_OWNER|调用方不拥有该锁。|  
|HOST_E_ABANDONED|已阻止的线程或纤程正在等待某个事件时, 该事件被取消。|  
|E_FAIL|发生未知的灾难性故障。 当方法返回 E_FAIL 时, CLR 在该进程内将不再可用。 对宿主方法的后续调用会返回 HOST_E_CLRNOTAVAILABLE。|  
  
## <a name="remarks"></a>备注  
 当从用户代码`Alert`调用时<xref:System.Threading.Thread.Abort%2A?displayProperty=nameWithType> , 或当与当前<xref:System.Threading.Thread>的<xref:System.AppDomain>关联关闭时, CLR 将调用方法。 由于调用是异步进行的, 因此主机必须立即返回。 如果主机无法立即对任务发出警报, 则必须在下一次进入状态时唤醒该任务。  
  
> [!NOTE]
> `Alert`仅影响运行时将[WAIT_OPTION](../../../../docs/framework/unmanaged-api/hosting/wait-option-enumeration.md)值 WAIT_ALERTABLE 传递给方法 (如[Join](../../../../docs/framework/unmanaged-api/hosting/ihosttask-join-method.md)) 的那些任务。
