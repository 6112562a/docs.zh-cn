## <a name="value-restriction"></a><span data-ttu-id="d27c2-122">值限制</span><span class="sxs-lookup"><span data-stu-id="d27c2-122">Value Restriction</span></span>

<span data-ttu-id="d27c2-123">编译器会执行自动泛化仅在具有显式参数的完整函数定义和简单的不可变值。</span><span class="sxs-lookup"><span data-stu-id="d27c2-123">The compiler performs automatic generalization only on complete function definitions that have explicit arguments, and on simple immutable values.</span></span>

<span data-ttu-id="d27c2-124">这意味着编译器发出错误，如果您尝试编译代码未充分约束为特定的类型，但也不是可归纳。</span><span class="sxs-lookup"><span data-stu-id="d27c2-124">This means that the compiler issues an error if you try to compile code that is not sufficiently constrained to be a specific type, but is also not generalizable.</span></span> <span data-ttu-id="d27c2-125">此问题的错误消息是指此限制的值作为自动泛化*值限制*。</span><span class="sxs-lookup"><span data-stu-id="d27c2-125">The error message for this problem refers to this restriction on automatic generalization for values as the *value restriction*.</span></span>

<span data-ttu-id="d27c2-126">通常情况下，值限制错误发生时所需的构造是通用，但编译器没有足够的信息进行通用化处理，或在无意中省略足够在非泛型构造函数中的类型信息时。</span><span class="sxs-lookup"><span data-stu-id="d27c2-126">Typically, the value restriction error occurs either when you want a construct to be generic but the compiler has insufficient information to generalize it, or when you unintentionally omit sufficient type information in a nongeneric construct.</span></span> <span data-ttu-id="d27c2-127">值限制错误的解决方案是提供更明确的信息，以更全面地约束类型推理问题，通过以下方式之一：</span><span class="sxs-lookup"><span data-stu-id="d27c2-127">The solution to the value restriction error is to provide more explicit information to more fully constrain the type inference problem, in one of the following ways:</span></span>

- <span data-ttu-id="d27c2-128">将通过将显式类型批注添加到值或参数为非泛型类型约束。</span><span class="sxs-lookup"><span data-stu-id="d27c2-128">Constrain a type to be nongeneric by adding an explicit type annotation to a value or parameter.</span></span>

- <span data-ttu-id="d27c2-129">如果使用问题不可泛化构造来定义如函数组合或不完整的泛型函数应用的扩充的函数自变量，请尝试重写为普通函数定义的函数。</span><span class="sxs-lookup"><span data-stu-id="d27c2-129">If the problem is using a nongeneralizable construct to define a generic function, such as a function composition or incompletely applied curried function arguments, try to rewrite the function as an ordinary function definition.</span></span>

- <span data-ttu-id="d27c2-130">如果问题是太复杂，无法进行一般化，通过添加额外的未使用参数来使到函数的表达式。</span><span class="sxs-lookup"><span data-stu-id="d27c2-130">If the problem is an expression that is too complex to be generalized, make it into a function by adding an extra, unused parameter.</span></span>

- <span data-ttu-id="d27c2-131">添加显式泛型类型参数。</span><span class="sxs-lookup"><span data-stu-id="d27c2-131">Add explicit generic type parameters.</span></span> <span data-ttu-id="d27c2-132">很少使用此选项。</span><span class="sxs-lookup"><span data-stu-id="d27c2-132">This option is rarely used.</span></span>

- <span data-ttu-id="d27c2-133">下面的代码示例说明了每种方案。</span><span class="sxs-lookup"><span data-stu-id="d27c2-133">The following code examples illustrate each of these scenarios.</span></span>

<span data-ttu-id="d27c2-134">案例 1： 太复杂的表达式。</span><span class="sxs-lookup"><span data-stu-id="d27c2-134">Case 1: Too complex an expression.</span></span> <span data-ttu-id="d27c2-135">在此示例中，列表`counter`用于`int option ref`，但它不定义为一个简单的不可变值。</span><span class="sxs-lookup"><span data-stu-id="d27c2-135">In this example, the list `counter` is intended to be `int option ref`, but it is not defined as a simple immutable value.</span></span>

<span data-ttu-id="d27c2-136">案例 2： 使用不可泛化构造定义的泛型函数。</span><span class="sxs-lookup"><span data-stu-id="d27c2-136">Case 2: Using a nongeneralizable construct to define a generic function.</span></span> <span data-ttu-id="d27c2-137">在此示例中，构造是不可泛化，因为它涉及到部分应用函数自变量。</span><span class="sxs-lookup"><span data-stu-id="d27c2-137">In this example, the construct is nongeneralizable because it involves partial application of function arguments.</span></span>


<span data-ttu-id="d27c2-138">案例 3： 添加一个额外的、 未使用的参数。</span><span class="sxs-lookup"><span data-stu-id="d27c2-138">Case 3: Adding an extra, unused parameter.</span></span> <span data-ttu-id="d27c2-139">此表达式不是简单的泛化，因为编译器会发出值限制错误。</span><span class="sxs-lookup"><span data-stu-id="d27c2-139">Because this expression is not simple enough for generalization, the compiler issues the value restriction error.</span></span>

```fsharp

// Adding an extra (unused) parameter makes it a function, which is generalizable.

```

<span data-ttu-id="d27c2-140">情形 4： 添加的类型参数。</span><span class="sxs-lookup"><span data-stu-id="d27c2-140">Case 4: Adding type parameters.</span></span>

```fsharp

// Adding a type parameter and type annotation lets you write a generic value.
```

<span data-ttu-id="d27c2-141">在最后一种情况下，值将成为类型函数，可用于创建许多不同类型的值为例，如下：</span><span class="sxs-lookup"><span data-stu-id="d27c2-141">In the last case, the value becomes a type function, which may be used to create values of many different types, for example as follows:</span></span>
