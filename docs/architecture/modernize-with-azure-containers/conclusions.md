# <a name="conclusions"></a>结论

- 基于容器的解决方案最终提供成本节约优势。 容器是部署问题的解决方案, 因为它们消除了在生产环境中缺乏依赖项而导致的矛盾。 通过消除这些问题, 它可以显著提高开发/测试、DevOps 和生产操作。

- Docker 容器正逐渐成为任何基于服务器的应用程序或服务的标准部署单位。

- 对于生产环境, 应使用 orchestrator (如 Kubernetes) 来托管基于的可缩放容器的应用程序。

- Azure Vm 托管容器是一种快速而简单的方法, 可用于在云中创建小规模的开发/测试环境。

- 默认情况下, 在将关系数据库从现有应用程序迁移到 Azure 时, 建议使用 Azure SQL 数据库托管实例。

- Visual Studio 2017 和 Image2Docker 是基本工具, 通过加速入门学习曲线, 你可以通过 Windows 容器开始现代化现有的 .NET 应用程序。

- 在生产环境中放置容器化应用程序时, 将始终创建或采用 DevOps 区域性, 并使用 DevOps 工具创建 CI/CD 管道 (如 Azure DevOps Services 或 Jenkins)。

- Microsoft Azure 提供最全面、最完整的环境, 通过 Windows 容器、云基础结构和 PaaS 服务使现有 .NET Framework 应用程序实现现代化。
