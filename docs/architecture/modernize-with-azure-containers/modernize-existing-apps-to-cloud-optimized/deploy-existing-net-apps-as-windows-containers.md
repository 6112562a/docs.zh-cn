## <a name="windows-container-types"></a><span data-ttu-id="b52df-211">Windows 容器类型</span><span class="sxs-lookup"><span data-stu-id="b52df-211">Windows container types</span></span>

<span data-ttu-id="b52df-212">与 Linux 容器类似，Windows Server 容器使用 Docker 引擎进行管理。</span><span class="sxs-lookup"><span data-stu-id="b52df-212">Like Linux containers, Windows Server containers are managed by using Docker Engine.</span></span> <span data-ttu-id="b52df-213">与 Linux 容器不同，Windows 容器包括两个不同的容器类型或运行时间-Windows Server 容器和 Hyper-v 隔离。</span><span class="sxs-lookup"><span data-stu-id="b52df-213">Unlike Linux containers, Windows containers include two different container types, or run times-Windows Server containers and Hyper-V isolation.</span></span>

<span data-ttu-id="b52df-214">**Windows Server 容器**：通过进程和命名空间隔离技术提供应用程序隔离。</span><span class="sxs-lookup"><span data-stu-id="b52df-214">**Windows Server containers**: Provides application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="b52df-215">Windows Server 容器与容器主机和在主机上运行的所有容器共享内核。</span><span class="sxs-lookup"><span data-stu-id="b52df-215">A Windows Server container shares a kernel with the container host and all containers that are running on the host.</span></span> <span data-ttu-id="b52df-216">这些容器不提供恶意安全边界，且不应用于隔离不受信任的代码。</span><span class="sxs-lookup"><span data-stu-id="b52df-216">These containers do not provide a hostile security boundary and should not be used to isolate untrusted code.</span></span> <span data-ttu-id="b52df-217">由于共享内核空间，这些容器需要相同的内核版本和配置。</span><span class="sxs-lookup"><span data-stu-id="b52df-217">Because of the shared kernel space, these containers require the same kernel version and configuration.</span></span>

<span data-ttu-id="b52df-218">**Hyper-v 隔离**：通过在高度优化的虚拟机上运行每个容器，在 Windows Server 容器提供的隔离上扩展。</span><span class="sxs-lookup"><span data-stu-id="b52df-218">**Hyper-V isolation**: Expands on the isolation provided by Windows Server Containers by running each container on a highly optimized VM.</span></span> <span data-ttu-id="b52df-219">在此配置中，容器主机的内核不与同一主机上的其他容器共享。</span><span class="sxs-lookup"><span data-stu-id="b52df-219">In this configuration, the kernel of the container host is not shared with other containers on the same host.</span></span> <span data-ttu-id="b52df-220">这些容器专用于恶意多租户托管，具有 VM 的相同安全保障。</span><span class="sxs-lookup"><span data-stu-id="b52df-220">These containers are designed for hostile multitenant hosting, with the same security assurances of a VM.</span></span> <span data-ttu-id="b52df-221">由于这些容器不与主机或主机上的其他容器共享内核，因此它们可以运行具有不同版本和配置的内核（支持的版本）。</span><span class="sxs-lookup"><span data-stu-id="b52df-221">Because these containers don't share the kernel with the host or other containers on the host, they can run kernels with different versions and configurations (with supported versions).</span></span> <span data-ttu-id="b52df-222">例如，Windows 10 上的所有 Windows 容器使用 Hyper-v 隔离来利用 Windows Server 内核版本和配置。</span><span class="sxs-lookup"><span data-stu-id="b52df-222">For example, all Windows containers on Windows 10 use Hyper-V isolation to utilize the Windows Server kernel version and configuration.</span></span>

<span data-ttu-id="b52df-223">在 Windows 上运行带有或不带 Hyper-v 隔离的容器是一个运行时决定。</span><span class="sxs-lookup"><span data-stu-id="b52df-223">Running a container on Windows with or without Hyper-V isolation is a run-time decision.</span></span> <span data-ttu-id="b52df-224">您可以选择最初使用 Hyper-v 隔离创建容器，并在运行时选择将其作为 Windows Server 容器运行。</span><span class="sxs-lookup"><span data-stu-id="b52df-224">You might choose to create the container with Hyper-V isolation initially, and at run time, choose to run it as a Windows Server container instead.</span></span>
