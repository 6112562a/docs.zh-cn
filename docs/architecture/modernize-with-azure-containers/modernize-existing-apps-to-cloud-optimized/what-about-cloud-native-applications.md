<span data-ttu-id="5f23c-127">[云本机计算基础（CNCF）](https://www.cncf.io/)是云本机原则的主要提升程序。</span><span class="sxs-lookup"><span data-stu-id="5f23c-127">The [Cloud Native Computing Foundation (CNCF)](https://www.cncf.io/) is a primary promoter of cloud-native principles.</span></span> <span data-ttu-id="5f23c-128">Microsoft 是[CNCF 的成员](https://azure.microsoft.com/blog/announcing-cncf/)。</span><span class="sxs-lookup"><span data-stu-id="5f23c-128">Microsoft is a [member of the CNCF](https://azure.microsoft.com/blog/announcing-cncf/).</span></span>

<span data-ttu-id="5f23c-131">如果将整个应用程序迁移到云本机模型，则要考虑的最重要因素是，必须重塑到基于微服务的体系结构。</span><span class="sxs-lookup"><span data-stu-id="5f23c-131">The most important factor to consider if you migrate a full application to the cloud-native model is that you must rearchitect to a microservices-based architecture.</span></span> <span data-ttu-id="5f23c-132">由于涉及大型重构过程，这显然需要大量的开发工作。</span><span class="sxs-lookup"><span data-stu-id="5f23c-132">This clearly requires a significant investment in development because of the large refactoring process involved.</span></span> <span data-ttu-id="5f23c-133">通常会为需要新级别的可伸缩性和长期灵活性的任务关键型应用程序选择此选项。</span><span class="sxs-lookup"><span data-stu-id="5f23c-133">This option usually is chosen for mission-critical applications that need new levels of scalability and long-term agility.</span></span> <span data-ttu-id="5f23c-134">但是，你可以通过为只是几个新方案添加微服务来开始向云中移动，并最终重构应用程序并将其完全视为微服务。</span><span class="sxs-lookup"><span data-stu-id="5f23c-134">But, you could start moving toward cloud-native by adding microservices for just a few new scenarios, and eventually refactor the application fully as microservices.</span></span> <span data-ttu-id="5f23c-135">这是一种在某些情况下最好的方法。</span><span class="sxs-lookup"><span data-stu-id="5f23c-135">This is an incremental approach that is the best option for some scenarios.</span></span>

## <a name="what-about-microservices"></a><span data-ttu-id="5f23c-136">微服务怎么样？</span><span class="sxs-lookup"><span data-stu-id="5f23c-136">What about microservices?</span></span>

<span data-ttu-id="5f23c-137">在考虑组织的云本机应用程序时，了解微服务及其工作原理非常重要。</span><span class="sxs-lookup"><span data-stu-id="5f23c-137">Understanding microservices and how they work is important when you are considering cloud-native applications for your organization.</span></span>

<span data-ttu-id="5f23c-138">微服务体系结构是一种高级方法，你可以将其用于从头开始创建的应用程序，也可以将现有应用程序发展为云本机应用程序。</span><span class="sxs-lookup"><span data-stu-id="5f23c-138">The microservices architecture is an advanced approach that you can use for applications that are created from scratch or when you evolve existing applications toward cloud-native applications.</span></span> <span data-ttu-id="5f23c-139">首先，可以将一些微服务添加到现有应用程序，以了解新的微服务范例。</span><span class="sxs-lookup"><span data-stu-id="5f23c-139">You can start by adding a few microservices to existing applications to learn about the new microservices paradigms.</span></span> <span data-ttu-id="5f23c-140">但很显然，您需要构建并编写代码，特别是对于这种类型的体系结构方法。</span><span class="sxs-lookup"><span data-stu-id="5f23c-140">But clearly, you need to architect and code, especially for this type of architectural approach.</span></span>

<span data-ttu-id="5f23c-141">但是，对于任何新的或新式应用程序，微服务不是必需的。</span><span class="sxs-lookup"><span data-stu-id="5f23c-141">However, microservices are not mandatory for any new or modern application.</span></span> <span data-ttu-id="5f23c-142">微服务不是 "幻项目符号"，而且它们并不是创建每个应用程序的最佳方法。</span><span class="sxs-lookup"><span data-stu-id="5f23c-142">Microservices are not a "magic bullet," and they aren't the single, best way to create every application.</span></span> <span data-ttu-id="5f23c-143">使用微服务的方式和时间取决于需要构建的应用程序的类型。</span><span class="sxs-lookup"><span data-stu-id="5f23c-143">How and when you use microservices depends on the type of application that you need to build.</span></span>

<span data-ttu-id="5f23c-144">微服务体系结构正在成为分布式和大型任务关键型应用程序的首选方法，这些应用程序基于多个独立的子系统，采用自治服务的形式。</span><span class="sxs-lookup"><span data-stu-id="5f23c-144">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications that are based on multiple, independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="5f23c-145">在基于微服务的体系结构中，应用程序构建为可独立开发、测试、版本控制、部署和缩放的服务集合。</span><span class="sxs-lookup"><span data-stu-id="5f23c-145">In a microservices-based architecture, an application is built as a collection of services that can be independently developed, tested, versioned, deployed, and scaled.</span></span> <span data-ttu-id="5f23c-146">这可以包括每个微服务的任何相关的自治数据库。</span><span class="sxs-lookup"><span data-stu-id="5f23c-146">This can include any related, autonomous database per microservice.</span></span>

<span data-ttu-id="5f23c-147">若要详细了解可以使用 .net Core 实现的微服务体系结构，请参阅可下载的 PDF 电子书[.net 微服务：适用于容器化 .NET 应用程序的体系结构](https://aka.ms/microservicesebook)。</span><span class="sxs-lookup"><span data-stu-id="5f23c-147">For a detailed look at a microservices architecture that you can implement by using .NET Core, see the downloadable PDF e-book [.NET microservices: Architecture for containerized .NET applications](https://aka.ms/microservicesebook).</span></span> <span data-ttu-id="5f23c-148">本指南还提供[在线](../../microservices/index.md)支持。</span><span class="sxs-lookup"><span data-stu-id="5f23c-148">The guide also is available [online](../../microservices/index.md).</span></span>

<span data-ttu-id="5f23c-149">但即使是在微服务提供功能强大的独立部署、强子系统边界和技术多样性的情况下，它们也会带来许多新的挑战。</span><span class="sxs-lookup"><span data-stu-id="5f23c-149">But even in scenarios in which microservices offer powerful capabilities-independent deployment, strong subsystem boundaries, and technology diversity-they also raise many new challenges.</span></span> <span data-ttu-id="5f23c-150">这些挑战与分布式应用程序开发相关，如分散和独立的数据模型;实现微服务之间的弹性通信;需要最终一致性;和操作复杂性。</span><span class="sxs-lookup"><span data-stu-id="5f23c-150">The challenges are related to distributed application development, such as fragmented and independent data models; achieving resilient communication between microservices; the need for eventual consistency; and operational complexity.</span></span> <span data-ttu-id="5f23c-151">与传统的单一应用程序相比，微服务引入了更高的复杂性。</span><span class="sxs-lookup"><span data-stu-id="5f23c-151">Microservices introduce a higher level of complexity compared to traditional monolithic applications.</span></span>

<span data-ttu-id="5f23c-152">由于微服务体系结构的复杂性，只有特定的方案和特定的应用程序类型适用于基于微服务的应用程序。</span><span class="sxs-lookup"><span data-stu-id="5f23c-152">Because of the complexity of a microservices architecture, only specific scenarios and certain application types are suitable for microservice-based applications.</span></span> <span data-ttu-id="5f23c-153">其中包括具有多个不断发展的子系统的大型复杂应用程序。</span><span class="sxs-lookup"><span data-stu-id="5f23c-153">These include large and complex applications that have multiple, evolving subsystems.</span></span> <span data-ttu-id="5f23c-154">在这些情况下，有必要投资更复杂的软件体系结构，以提高长期的灵活性和更高效的应用程序维护。</span><span class="sxs-lookup"><span data-stu-id="5f23c-154">In these cases, it's worth investing in a more complex software architecture, for increased long-term agility and more efficient application maintenance.</span></span> <span data-ttu-id="5f23c-155">但对于不太复杂的方案，使用单一应用程序方法或更简单的 N 层方法可能会更好。</span><span class="sxs-lookup"><span data-stu-id="5f23c-155">But for less complex scenarios, it might be better to continue with a monolithic application approach or simpler N-Tier approaches.</span></span>

<span data-ttu-id="5f23c-156">最后要注意的是，即使是重复此概念的风险，也不应在应用程序中将微服务作为 "所有内容" 或 "无"。</span><span class="sxs-lookup"><span data-stu-id="5f23c-156">As a final note, even at the risk of being repetitive about this concept, you shouldn't look at using microservices in your applications as "all-in or nothing at all."</span></span> <span data-ttu-id="5f23c-157">可以通过添加基于微服务的新的小型方案来扩展和改进现有的单一应用程序。</span><span class="sxs-lookup"><span data-stu-id="5f23c-157">You can extend and evolve existing monolithic applications by adding new, small scenarios based on microservices.</span></span> <span data-ttu-id="5f23c-158">无需从头开始使用微服务体系结构方法。</span><span class="sxs-lookup"><span data-stu-id="5f23c-158">You don't need to start from scratch to start working with a microservices architecture approach.</span></span> <span data-ttu-id="5f23c-159">事实上，我们建议通过添加新方案，从使用现有的单层或 N 层应用程序。</span><span class="sxs-lookup"><span data-stu-id="5f23c-159">In fact, we recommend that you evolve from using an existing monolithic or N-Tier application by adding new scenarios.</span></span> <span data-ttu-id="5f23c-160">最终，可以将应用程序分解为自治组件或微服务。</span><span class="sxs-lookup"><span data-stu-id="5f23c-160">Eventually, you can break down the application into autonomous components or microservices.</span></span> <span data-ttu-id="5f23c-161">您可以逐步微服务的方向开始改进单一应用程序。</span><span class="sxs-lookup"><span data-stu-id="5f23c-161">You can start evolving your monolithic applications in a microservices direction, step by step.</span></span>

<span data-ttu-id="5f23c-162">在任何情况下，本指南的其余部分都集中于 "没有基于微服务的应用"，因为本指南主要针对通常具有单层或 N 层体系结构的现有应用的现代化。</span><span class="sxs-lookup"><span data-stu-id="5f23c-162">In any case, the rest of this present guidance focuses most of all on "no microservices-based apps" because this guidance is mainly targeting the modernization of existing apps that usually have monolithic or N-Tier architectures.</span></span>
