<span data-ttu-id="35527-115">如果你可以灵活地开发应用程序，则你的应用程序是云优化的，它从其他本地应用程序自主开发应用程序，然后在云中发布、部署、自动缩放、监视和排查应用程序问题。</span><span class="sxs-lookup"><span data-stu-id="35527-115">Your application is Cloud-Optimized if you can-in an agile manner-develop your app autonomously from other on-premises apps, and then release, deploy, auto-scale, monitor, and troubleshoot your app in the cloud.</span></span>

<span data-ttu-id="35527-147">可以通过将现有应用程序转换为云优化的应用程序（无需重新架构或编码）来获得以下好处：</span><span class="sxs-lookup"><span data-stu-id="35527-147">You can get the following benefits by converting an existing application to a Cloud-Optimized application (without rearchitecting or coding):</span></span>

- <span data-ttu-id="35527-151">**复原应用程序和基础结构**。</span><span class="sxs-lookup"><span data-stu-id="35527-151">**Resilient application and infrastructure**.</span></span> <span data-ttu-id="35527-152">迁移到云时，需要采用暂时性故障;在云中将发生故障。</span><span class="sxs-lookup"><span data-stu-id="35527-152">When you migrate to the cloud, you need to embrace transient failures; failures will occur in the cloud.</span></span> <span data-ttu-id="35527-153">此外，云基础结构和硬件是 "可更换的"，这会增加暂时性停机的机会。</span><span class="sxs-lookup"><span data-stu-id="35527-153">Also, cloud infrastructure and hardware are "replaceable," which increases opportunities for transient downtime.</span></span> <span data-ttu-id="35527-154">同时，实现复原和自动恢复的内部云功能和某些应用程序开发技术可以更轻松地在云中发生意外故障时进行恢复。</span><span class="sxs-lookup"><span data-stu-id="35527-154">At the same time, inner cloud capabilities and certain application development techniques that implement resiliency and automate recovery make it much easier to recover from unexpected failures in the cloud.</span></span>
