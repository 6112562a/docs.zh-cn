## <a name="the-cloud"></a><span data-ttu-id="64f23-143">云...</span><span class="sxs-lookup"><span data-stu-id="64f23-143">The cloud…</span></span>

<span data-ttu-id="64f23-144">云原生系统充分利用了云服务模型。</span><span class="sxs-lookup"><span data-stu-id="64f23-144">Cloud-native systems take full advantage of the cloud service model.</span></span>

<span data-ttu-id="64f23-145">这些系统旨在在动态虚拟化的云环境中蓬勃发展，广泛使用[平台即服务 （PaaS）](https://azure.microsoft.com/overview/what-is-paas/)计算基础架构和托管服务。</span><span class="sxs-lookup"><span data-stu-id="64f23-145">Designed to thrive in a dynamic, virtualized cloud environment, these systems make extensive use of [Platform as a Service (PaaS)](https://azure.microsoft.com/overview/what-is-paas/) compute infrastructure and managed services.</span></span> <span data-ttu-id="64f23-146">他们将底层基础架构视为*一次性*基础结构 - 在几分钟内预配，并通过自动化按需调整规模、缩放、移动或销毁。</span><span class="sxs-lookup"><span data-stu-id="64f23-146">They treat the underlying infrastructure as *disposable* - provisioned in minutes and resized, scaled, moved, or destroyed on demand – via automation.</span></span>

<span data-ttu-id="64f23-147">考虑广泛接受的DevOps概念[宠物与牛](https://medium.com/@Joachim8675309/devops-concepts-pets-vs-cattle-2380b5aab313)。</span><span class="sxs-lookup"><span data-stu-id="64f23-147">Consider the widely accepted DevOps concept of [Pets vs. Cattle](https://medium.com/@Joachim8675309/devops-concepts-pets-vs-cattle-2380b5aab313).</span></span> <span data-ttu-id="64f23-148">在传统的数据中心中，服务器被视为*宠物*：物理机器，被赋予一个有意义的名称，并受到关注。</span><span class="sxs-lookup"><span data-stu-id="64f23-148">In a traditional data center, servers are treated as *Pets*: a physical machine, given a meaningful name, and cared for.</span></span> <span data-ttu-id="64f23-149">通过向同一计算机添加更多资源（向上扩展）进行扩展。</span><span class="sxs-lookup"><span data-stu-id="64f23-149">You scale by adding more resources to the same machine (scaling up).</span></span> <span data-ttu-id="64f23-150">如果服务器生病，您将将其护理回运行状况。</span><span class="sxs-lookup"><span data-stu-id="64f23-150">If the server becomes sick, you nurse it back to health.</span></span> <span data-ttu-id="64f23-151">如果服务器不可用，每个人都会注意到。</span><span class="sxs-lookup"><span data-stu-id="64f23-151">Should the server become unavailable, everyone notices.</span></span>

<span data-ttu-id="64f23-152">*牛*服务模式不同。</span><span class="sxs-lookup"><span data-stu-id="64f23-152">The *Cattle* service model is different.</span></span> <span data-ttu-id="64f23-153">将每个实例预配为虚拟机或容器。</span><span class="sxs-lookup"><span data-stu-id="64f23-153">You provision each instance as a virtual machine or container.</span></span> <span data-ttu-id="64f23-154">它们相同，并分配了系统标识符，如服务-01、服务-02等。</span><span class="sxs-lookup"><span data-stu-id="64f23-154">They're identical and assigned a system identifier such as Service-01, Service-02, and so on.</span></span> <span data-ttu-id="64f23-155">您可以通过创建更多（向外扩展）进行扩展。</span><span class="sxs-lookup"><span data-stu-id="64f23-155">You scale by creating more of them (scaling out).</span></span> <span data-ttu-id="64f23-156">当一个人变得不可用时，没有人会注意到。</span><span class="sxs-lookup"><span data-stu-id="64f23-156">When one becomes unavailable, nobody notices.</span></span>

<span data-ttu-id="64f23-157">牛模式拥抱*了不变的基础设施*。</span><span class="sxs-lookup"><span data-stu-id="64f23-157">The cattle model embraces *immutable infrastructure*.</span></span> <span data-ttu-id="64f23-158">服务器未修复或修改。</span><span class="sxs-lookup"><span data-stu-id="64f23-158">Servers aren't repaired or modified.</span></span> <span data-ttu-id="64f23-159">如果一个失败或需要更新，它将被销毁，并预配了新的更新 - 所有这些都通过自动化完成。</span><span class="sxs-lookup"><span data-stu-id="64f23-159">If one fails or requires updating, it's destroyed and a new one is provisioned – all done via automation.</span></span>

<span data-ttu-id="64f23-160">云原生系统采用牛服务模式。</span><span class="sxs-lookup"><span data-stu-id="64f23-160">Cloud-native systems embrace the Cattle service model.</span></span> <span data-ttu-id="64f23-161">它们继续随着基础结构的扩展或运行而运行，而不考虑运行的机器。</span><span class="sxs-lookup"><span data-stu-id="64f23-161">They continue to run as the infrastructure scales in or out with no regard to the machines upon which they're running.</span></span>

<span data-ttu-id="64f23-162">Azure 云平台支持这种类型的高弹性基础架构，具有自动缩放、自我修复和监视功能。</span><span class="sxs-lookup"><span data-stu-id="64f23-162">The Azure cloud platform supports this type of highly elastic infrastructure with automatic scaling, self-healing, and monitoring capabilities.</span></span>
