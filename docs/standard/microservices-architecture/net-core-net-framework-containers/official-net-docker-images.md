### <a name="during-development-and-build"></a><span data-ttu-id="f3024-116">在开发和生成过程中</span><span class="sxs-lookup"><span data-stu-id="f3024-116">During development and build</span></span>

<span data-ttu-id="f3024-122">为什么此类型的生成映像很重要？</span><span class="sxs-lookup"><span data-stu-id="f3024-122">Why is this type of build image important?</span></span> <span data-ttu-id="f3024-123">不能将此映像部署到生产中。</span><span class="sxs-lookup"><span data-stu-id="f3024-123">You do not deploy this image to production.</span></span> <span data-ttu-id="f3024-124">相反，它是用于生成放置在生产映像中的内容的映像。</span><span class="sxs-lookup"><span data-stu-id="f3024-124">Instead, it is an image you use to build the content you place into a production image.</span></span> <span data-ttu-id="f3024-125">此映像将用于持续集成 (CI) 环境，或在使用 Docker 多阶段生成时用于生成环境。</span><span class="sxs-lookup"><span data-stu-id="f3024-125">This image would be used in your continuous integration (CI) environment or build environment when using Docker Multi-stage builds.</span></span>

### <a name="in-production"></a><span data-ttu-id="f3024-126">生产中</span><span class="sxs-lookup"><span data-stu-id="f3024-126">In production</span></span>

<span data-ttu-id="f3024-127">在生产中重要的是基于生产 .NET Core 映像部署和启动容器的速度。</span><span class="sxs-lookup"><span data-stu-id="f3024-127">What is important in production is how fast you can deploy and start your containers based on a production .NET Core image.</span></span> <span data-ttu-id="f3024-128">因此，基于 microsoft/dotnet:2.2-aspnetcore-runtime 的仅运行时映像很小，以便它可以通过网络从 Docker 注册表快速传输到 Docker 主机。</span><span class="sxs-lookup"><span data-stu-id="f3024-128">Therefore, the runtime-only image based on *microsoft/dotnet:2.2-aspnetcore-runtime* is small so that it can travel quickly across the network from your Docker registry to your Docker hosts.</span></span> <span data-ttu-id="f3024-129">已准备运行内容，以此实现从启动容器到处理结果的最快时间。</span><span class="sxs-lookup"><span data-stu-id="f3024-129">The contents are ready to run, enabling the fastest time from starting the container to processing results.</span></span> <span data-ttu-id="f3024-130">在 Docker 模型中，不需要编译 C\# 代码，但在使用生成容器运行 dotnet 生成或 dotnet 发布时需要。</span><span class="sxs-lookup"><span data-stu-id="f3024-130">In the Docker model, there is no need for compilation from C\# code, as there is when you run dotnet build or dotnet publish when using the build container.</span></span>

<span data-ttu-id="f3024-131">在此优化的映像中，只放置运行应用程序所需的二进制文件和其他内容。</span><span class="sxs-lookup"><span data-stu-id="f3024-131">In this optimized image you put only the binaries and other content needed to run the application.</span></span> <span data-ttu-id="f3024-132">例如，由 dotnet 发布创建的内容仅包含已编译的.NET 二进制文件、映像、.js 和 .css 文件。</span><span class="sxs-lookup"><span data-stu-id="f3024-132">For example, the content created by dotnet publish contains only the compiled .NET binaries, images, .js, and .css files.</span></span> <span data-ttu-id="f3024-133">随着时间推移，将推出包含预实时编译（在运行时进行从中间语言到本机语言的编译）包的映像。</span><span class="sxs-lookup"><span data-stu-id="f3024-133">Over time, you will see images that contain pre-jitted (the compilation from IL to native that occurs at runtime) packages.</span></span>

<span data-ttu-id="f3024-134">虽然 .NET Core 和 ASP.NET Core 映像有多个版本，但它们全都共享一个或多个层，包括基本层。</span><span class="sxs-lookup"><span data-stu-id="f3024-134">Although there are multiple versions of the .NET Core and ASP.NET Core images, they all share one or more layers, including the base layer.</span></span> <span data-ttu-id="f3024-135">因此，存储映像所需的磁盘空间量很小；它仅包含自定义映像和其基础映像之间的增量。</span><span class="sxs-lookup"><span data-stu-id="f3024-135">Therefore, the amount of disk space needed to store an image is small; it consists only of the delta between your custom image and its base image.</span></span> <span data-ttu-id="f3024-136">结果，从注册表中提取映像速度会很快。</span><span class="sxs-lookup"><span data-stu-id="f3024-136">The result is that it is quick to pull the image from your registry.</span></span>
