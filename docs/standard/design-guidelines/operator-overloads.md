# <a name="operator-overloads"></a><span data-ttu-id="16dbf-102">运算符重载</span><span class="sxs-lookup"><span data-stu-id="16dbf-102">Operator Overloads</span></span>
<span data-ttu-id="16dbf-103">运算符重载允许显示如同内置语言基元的框架类型。</span><span class="sxs-lookup"><span data-stu-id="16dbf-103">Operator overloads allow framework types to appear as if they were built-in language primitives.</span></span>  
  
 <span data-ttu-id="16dbf-104">虽然允许，并且在某些情况下很有用，应谨慎使用运算符重载。</span><span class="sxs-lookup"><span data-stu-id="16dbf-104">Although allowed and useful in some situations, operator overloads should be used cautiously.</span></span> <span data-ttu-id="16dbf-105">有很多情况下在哪些运算符重载具有已被滥用，如框架设计器启动时若要使用的操作，应为简单的方法的运算符。</span><span class="sxs-lookup"><span data-stu-id="16dbf-105">There are many cases in which operator overloading has been abused, such as when framework designers started to use operators for operations that should be simple methods.</span></span> <span data-ttu-id="16dbf-106">以下指导原则可帮助您决定何时以及如何使用运算符重载。</span><span class="sxs-lookup"><span data-stu-id="16dbf-106">The following guidelines should help you decide when and how to use operator overloading.</span></span>  
  
 <span data-ttu-id="16dbf-107">**X AVOID** 定义运算符重载，除了应感觉基元 （内置） 类型的类型中。</span><span class="sxs-lookup"><span data-stu-id="16dbf-107">**X AVOID** defining operator overloads, except in types that should feel like primitive (built-in) types.</span></span>  
  
 <span data-ttu-id="16dbf-108">**✓ CONSIDER** 应感觉类似于基元类型的类型中定义运算符重载。</span><span class="sxs-lookup"><span data-stu-id="16dbf-108">**✓ CONSIDER** defining operator overloads in a type that should feel like a primitive type.</span></span>  
  
 <span data-ttu-id="16dbf-109">例如，<xref:System.String?displayProperty=nameWithType>已`operator==`和`operator!=`定义。</span><span class="sxs-lookup"><span data-stu-id="16dbf-109">For example, <xref:System.String?displayProperty=nameWithType> has `operator==` and `operator!=` defined.</span></span>  
  
 <span data-ttu-id="16dbf-110">**✓ DO** 中表示的数字的结构定义运算符重载 (如<xref:System.Decimal?displayProperty=nameWithType>)。</span><span class="sxs-lookup"><span data-stu-id="16dbf-110">**✓ DO** define operator overloads in structs that represent numbers (such as <xref:System.Decimal?displayProperty=nameWithType>).</span></span>  
  
 <span data-ttu-id="16dbf-111">**X DO NOT** 过于刻意定义运算符重载时。</span><span class="sxs-lookup"><span data-stu-id="16dbf-111">**X DO NOT** be cute when defining operator overloads.</span></span>  
  
 <span data-ttu-id="16dbf-112">运算符重载是会立即明显操作的结果将是的非常有用。</span><span class="sxs-lookup"><span data-stu-id="16dbf-112">Operator overloading is useful in cases in which it is immediately obvious what the result of the operation will be.</span></span> <span data-ttu-id="16dbf-113">例如，最好能够减去<xref:System.DateTime>从另一个`DateTime`，并获取<xref:System.TimeSpan>。</span><span class="sxs-lookup"><span data-stu-id="16dbf-113">For example, it makes sense to be able to subtract one <xref:System.DateTime> from another `DateTime` and get a <xref:System.TimeSpan>.</span></span> <span data-ttu-id="16dbf-114">但是，不适合使用逻辑 union 运算符来联合两个数据库查询，或使用移位运算符要写入到流。</span><span class="sxs-lookup"><span data-stu-id="16dbf-114">However, it is not appropriate to use the logical union operator to union two database queries, or to use the shift operator to write to a stream.</span></span>  
  
 <span data-ttu-id="16dbf-115">**X DO NOT** 提供运算符重载，除非至少其中一个操作数是定义重载的类型。</span><span class="sxs-lookup"><span data-stu-id="16dbf-115">**X DO NOT** provide operator overloads unless at least one of the operands is of the type defining the overload.</span></span>  
  
 <span data-ttu-id="16dbf-116">**✓ DO** 对称方式重载运算符。</span><span class="sxs-lookup"><span data-stu-id="16dbf-116">**✓ DO** overload operators in a symmetric fashion.</span></span>  
  
 <span data-ttu-id="16dbf-117">例如，如果重载`operator==`，还应该重载`operator!=`。</span><span class="sxs-lookup"><span data-stu-id="16dbf-117">For example, if you overload the `operator==`, you should also overload the `operator!=`.</span></span> <span data-ttu-id="16dbf-118">同样，如果重载`operator<`，还应该重载`operator>`，依次类推。</span><span class="sxs-lookup"><span data-stu-id="16dbf-118">Similarly, if you overload the `operator<`, you should also overload the `operator>`, and so on.</span></span>  
  
 <span data-ttu-id="16dbf-119">**✓ CONSIDER** 提供的友好名称的对应到每个重载运算符的方法。</span><span class="sxs-lookup"><span data-stu-id="16dbf-119">**✓ CONSIDER** providing methods with friendly names that correspond to each overloaded operator.</span></span>  
  
 <span data-ttu-id="16dbf-120">许多语言不支持运算符重载。</span><span class="sxs-lookup"><span data-stu-id="16dbf-120">Many languages do not support operator overloading.</span></span> <span data-ttu-id="16dbf-121">出于此原因，建议重载运算符的类型包括具有适当的特定于域的名称提供等效功能的辅助方法。</span><span class="sxs-lookup"><span data-stu-id="16dbf-121">For this reason, it is recommended that types that overload operators include a secondary method with an appropriate domain-specific name that provides equivalent functionality.</span></span>

### <a name="overloading-operator-"></a><span data-ttu-id="16dbf-126">重载运算符 = =</span><span class="sxs-lookup"><span data-stu-id="16dbf-126">Overloading Operator ==</span></span>

### <a name="conversion-operators"></a><span data-ttu-id="16dbf-129">转换运算符</span><span class="sxs-lookup"><span data-stu-id="16dbf-129">Conversion Operators</span></span>  
 <span data-ttu-id="16dbf-130">转换运算符是一元运算符，从而允许从一种类型转换到另一个。</span><span class="sxs-lookup"><span data-stu-id="16dbf-130">Conversion operators are unary operators that allow conversion from one type to another.</span></span> <span data-ttu-id="16dbf-131">必须将运算符定义为操作数或返回类型上的静态成员。</span><span class="sxs-lookup"><span data-stu-id="16dbf-131">The operators must be defined as static members on either the operand or the return type.</span></span> <span data-ttu-id="16dbf-132">有两种类型的转换运算符： 隐式和显式。</span><span class="sxs-lookup"><span data-stu-id="16dbf-132">There are two types of conversion operators: implicit and explicit.</span></span>  
  
 <span data-ttu-id="16dbf-133">**X DO NOT** 提供转换运算符，如果最终用户未明确要求这种转换。</span><span class="sxs-lookup"><span data-stu-id="16dbf-133">**X DO NOT** provide a conversion operator if such conversion is not clearly expected by the end users.</span></span>  
  
 <span data-ttu-id="16dbf-134">**X DO NOT** 定义转换运算符之外的类型的域。</span><span class="sxs-lookup"><span data-stu-id="16dbf-134">**X DO NOT** define conversion operators outside of a type’s domain.</span></span>  
  
 <span data-ttu-id="16dbf-135">例如， <xref:System.Int32>， <xref:System.Double>，并<xref:System.Decimal>都是数值类型，而<xref:System.DateTime>不是。</span><span class="sxs-lookup"><span data-stu-id="16dbf-135">For example, <xref:System.Int32>, <xref:System.Double>, and <xref:System.Decimal> are all numeric types, whereas <xref:System.DateTime> is not.</span></span> <span data-ttu-id="16dbf-136">因此，应该有没有要转换的转换运算符`Double(long)`到`DateTime`。</span><span class="sxs-lookup"><span data-stu-id="16dbf-136">Therefore, there should be no conversion operator to convert a `Double(long)` to a `DateTime`.</span></span> <span data-ttu-id="16dbf-137">在这种情况中为首选构造函数。</span><span class="sxs-lookup"><span data-stu-id="16dbf-137">A constructor is preferred in such a case.</span></span>  
  
 <span data-ttu-id="16dbf-138">**X DO NOT** 提供隐式转换运算符，如果转换，则可能丢失信息。</span><span class="sxs-lookup"><span data-stu-id="16dbf-138">**X DO NOT** provide an implicit conversion operator if the conversion is potentially lossy.</span></span>  
  
 <span data-ttu-id="16dbf-139">例如，应该不会有隐式转换`Double`到`Int32`因为`Double`具有更大范围比`Int32`。</span><span class="sxs-lookup"><span data-stu-id="16dbf-139">For example, there should not be an implicit conversion from `Double` to `Int32` because `Double` has a wider range than `Int32`.</span></span> <span data-ttu-id="16dbf-140">可以提供显式转换运算符，即使转换，则可能会丢失数据。</span><span class="sxs-lookup"><span data-stu-id="16dbf-140">An explicit conversion operator can be provided even if the conversion is potentially lossy.</span></span>  
  
 <span data-ttu-id="16dbf-141">**X DO NOT** 从隐式强制转换引发异常。</span><span class="sxs-lookup"><span data-stu-id="16dbf-141">**X DO NOT** throw exceptions from implicit casts.</span></span>  
  
 <span data-ttu-id="16dbf-142">它是很难了解发生什么情况，因为它们可能不知道转换正在进行的最终用户。</span><span class="sxs-lookup"><span data-stu-id="16dbf-142">It is very difficult for end users to understand what is happening, because they might not be aware that a conversion is taking place.</span></span>  
  
 <span data-ttu-id="16dbf-143">**✓ DO** 引发<xref:System.InvalidCastException?displayProperty=nameWithType>如果对强制转换运算符的调用导致有损转换和运算符的协定不允许丢失数据的转换。</span><span class="sxs-lookup"><span data-stu-id="16dbf-143">**✓ DO** throw <xref:System.InvalidCastException?displayProperty=nameWithType> if a call to a cast operator results in a lossy conversion and the contract of the operator does not allow lossy conversions.</span></span>
