# <a name="names-of-classes-structs-and-interfaces"></a><span data-ttu-id="430bb-102">类、结构和接口的名称</span><span class="sxs-lookup"><span data-stu-id="430bb-102">Names of Classes, Structs, and Interfaces</span></span>
<span data-ttu-id="430bb-103">以下命名准则适用于常规类型命名。</span><span class="sxs-lookup"><span data-stu-id="430bb-103">The naming guidelines that follow apply to general type naming.</span></span>  
  
 <span data-ttu-id="430bb-104">**✓ 务必**通过使用 PascalCasing，用名词或名词短语命名类和结构。</span><span class="sxs-lookup"><span data-stu-id="430bb-104">**✓ DO** name classes and structs with nouns or noun phrases, using PascalCasing.</span></span>  
  
 <span data-ttu-id="430bb-105">这将类型名称与使用谓词短语命名的方法区分开来。</span><span class="sxs-lookup"><span data-stu-id="430bb-105">This distinguishes type names from methods, which are named with verb phrases.</span></span>  
  
 <span data-ttu-id="430bb-106">**✓ 务必**使用形容词短语命名接口，或偶尔用名词或名词短语命名接口。</span><span class="sxs-lookup"><span data-stu-id="430bb-106">**✓ DO** name interfaces with adjective phrases, or occasionally with nouns or noun phrases.</span></span>  
  
 <span data-ttu-id="430bb-107">应少使用名词和名词短语，它们可能会指示类型为抽象类而非接口。</span><span class="sxs-lookup"><span data-stu-id="430bb-107">Nouns and noun phrases should be used rarely and they might indicate that the type should be an abstract class, and not an interface.</span></span>  
  
 <span data-ttu-id="430bb-108">**X 不要**给类名加前缀（例如，"C"）。</span><span class="sxs-lookup"><span data-stu-id="430bb-108">**X DO NOT** give class names a prefix (e.g., "C").</span></span>  
  
 <span data-ttu-id="430bb-109">**✓ 考虑**使用基类的名称作为派生类名称的结尾部分。</span><span class="sxs-lookup"><span data-stu-id="430bb-109">**✓ CONSIDER** ending the name of derived classes with the name of the base class.</span></span>  
  
 <span data-ttu-id="430bb-110">这样可让名称非常易读，并清楚体现了关系。</span><span class="sxs-lookup"><span data-stu-id="430bb-110">This is very readable and explains the relationship clearly.</span></span> <span data-ttu-id="430bb-111">代码中的相关例子有：`ArgumentOutOfRangeException`，这是一种 `Exception`，还有 `SerializableAttribute`，这是一种 `Attribute`。</span><span class="sxs-lookup"><span data-stu-id="430bb-111">Some examples of this in code are: `ArgumentOutOfRangeException`, which is a kind of `Exception`, and `SerializableAttribute`, which is a kind of `Attribute`.</span></span> <span data-ttu-id="430bb-112">但是，在应用此准则时，务必应进行合理判断；例如，`Button` 类是一种 `Control` 事件，尽管其名称中并未出现 `Control`。</span><span class="sxs-lookup"><span data-stu-id="430bb-112">However, it is important to use reasonable judgment in applying this guideline; for example, the `Button` class is a kind of `Control` event, although `Control` doesn’t appear in its name.</span></span>  
  
 <span data-ttu-id="430bb-113">**✓ 务必**在接口名称前加上字母 I 作为前缀，以指示该类型是接口。</span><span class="sxs-lookup"><span data-stu-id="430bb-113">**✓ DO** prefix interface names with the letter I, to indicate that the type is an interface.</span></span>  
  
 <span data-ttu-id="430bb-114">例如，`IComponent`（描述性名词），`ICustomAttributeProvider`（名词短语）和 `IPersistable`（形容词）是合适的接口名称。与其他类型名称一样，应避免使用缩略形式。</span><span class="sxs-lookup"><span data-stu-id="430bb-114">For example, `IComponent` (descriptive noun), `ICustomAttributeProvider` (noun phrase), and `IPersistable` (adjective) are appropriate interface names.</span></span> <span data-ttu-id="430bb-115">与其他类型名称，避免缩写。</span><span class="sxs-lookup"><span data-stu-id="430bb-115">As with other type names, avoid abbreviations.</span></span>  
  
 <span data-ttu-id="430bb-116">**✓ 务必**确保在定义类和接口对时，类名称和接口名称的区别仅在于 "I" 前缀，其中类是接口的标准实现。</span><span class="sxs-lookup"><span data-stu-id="430bb-116">**✓ DO** ensure that the names differ only by the "I" prefix on the interface name when you are defining a class–interface pair where the class is a standard implementation of the interface.</span></span>  
  
## <a name="names-of-generic-type-parameters"></a><span data-ttu-id="430bb-117">泛型类型参数的名称</span><span class="sxs-lookup"><span data-stu-id="430bb-117">Names of Generic Type Parameters</span></span>  
 <span data-ttu-id="430bb-118">.NET Framework 2.0 中增添了泛型。</span><span class="sxs-lookup"><span data-stu-id="430bb-118">Generics were added to .NET Framework 2.0.</span></span> <span data-ttu-id="430bb-119">此功能引入了一种新的标识符称为*类型参数*。</span><span class="sxs-lookup"><span data-stu-id="430bb-119">The feature introduced a new kind of identifier called *type parameter*.</span></span>  
  
 <span data-ttu-id="430bb-120">**✓ 务必**使用描述性名称命名泛型参数，除非单字母名称可完整体现要传达的含义且描述性名称意义不大。</span><span class="sxs-lookup"><span data-stu-id="430bb-120">**✓ DO** name generic type parameters with descriptive names unless a single-letter name is completely self-explanatory and a descriptive name would not add value.</span></span>  
  
 <span data-ttu-id="430bb-121">**✓ 考虑**使用 `T` 作为具有一个单字母类型参数的类型的类型参数名称。</span><span class="sxs-lookup"><span data-stu-id="430bb-121">**✓ CONSIDER** using `T` as the type parameter name for types with one single-letter type parameter.</span></span>  
  
```  
public int IComparer<T> { ... }  
public delegate bool Predicate<T>(T item);  
public struct Nullable<T> where T:struct { ... }  
```  
  
 <span data-ttu-id="430bb-122">**✓ 务必**使用 `T` 作为描述性类型参数名称的前缀。</span><span class="sxs-lookup"><span data-stu-id="430bb-122">**✓ DO** prefix descriptive type parameter names with `T`.</span></span>  
  
```  
public interface ISessionChannel<TSession> where TSession : ISession {  
    TSession Session { get; }  
}  
```  
  
 <span data-ttu-id="430bb-123">**✓ 考虑**在参数名称中体现出对该类型参数设置的约束。</span><span class="sxs-lookup"><span data-stu-id="430bb-123">**✓ CONSIDER** indicating constraints placed on a type parameter in the name of the parameter.</span></span>  
  
 <span data-ttu-id="430bb-124">例如，被限制为 `ISession` 的参数可能名为 `TSession`。</span><span class="sxs-lookup"><span data-stu-id="430bb-124">For example, a parameter constrained to `ISession` might be called `TSession`.</span></span>

## <a name="naming-enumerations"></a><span data-ttu-id="430bb-142">命名枚举</span><span class="sxs-lookup"><span data-stu-id="430bb-142">Naming Enumerations</span></span>  
 <span data-ttu-id="430bb-143">一般情况下，枚举类型（也称为枚举）的名称应遵循标准的类型命名规则（PascalCasing 等）。</span><span class="sxs-lookup"><span data-stu-id="430bb-143">Names of enumeration types (also called enums) in general should follow the standard type-naming rules (PascalCasing, etc.).</span></span> <span data-ttu-id="430bb-144">但还有专用于枚举的其他准则。</span><span class="sxs-lookup"><span data-stu-id="430bb-144">However, there are additional guidelines that apply specifically to enums.</span></span>  
  
 <span data-ttu-id="430bb-145">**✓ 务必**为枚举使用单数形式的类型名称，除非枚举值是位域。</span><span class="sxs-lookup"><span data-stu-id="430bb-145">**✓ DO** use a singular type name for an enumeration unless its values are bit fields.</span></span>  
  
 <span data-ttu-id="430bb-146">**✓ 务必**为值为位域的枚举使用复数形式的类型名称，这类枚举也称为标志枚举。</span><span class="sxs-lookup"><span data-stu-id="430bb-146">**✓ DO** use a plural type name for an enumeration with bit fields as values, also called flags enum.</span></span>  
  
 <span data-ttu-id="430bb-147">**X 不要**在枚举类型名称中使用 "Enum" 作为后缀。</span><span class="sxs-lookup"><span data-stu-id="430bb-147">**X DO NOT** use an "Enum" suffix in enum type names.</span></span>  
  
 <span data-ttu-id="430bb-148">**X 不要**在枚举类型名称中使用 "Flag" 或 "Flags" 作为后缀。</span><span class="sxs-lookup"><span data-stu-id="430bb-148">**X DO NOT** use "Flag" or "Flags" suffixes in enum type names.</span></span>  
  
 <span data-ttu-id="430bb-149">**X 不要**在枚举值名称中使用前缀（例如，对 ADO 枚举使用 "ad”，对富文本枚举使用 "rtf" 等）。</span><span class="sxs-lookup"><span data-stu-id="430bb-149">**X DO NOT** use a prefix on enumeration value names (e.g., "ad" for ADO enums, "rtf" for rich text enums, etc.).</span></span>
