# <a name="conclusion"></a><span data-ttu-id="edd67-104">结束语</span><span class="sxs-lookup"><span data-stu-id="edd67-104">Conclusion</span></span>

<span data-ttu-id="edd67-105">以下关键结论是本指南中最重要的结论。</span><span class="sxs-lookup"><span data-stu-id="edd67-105">The following key takeaways are the most important conclusions from this guide.</span></span>

<span data-ttu-id="edd67-106">**使用无服务器的好处。**</span><span class="sxs-lookup"><span data-stu-id="edd67-106">**Benefits of using serverless.**</span></span> <span data-ttu-id="edd67-107">无服务器解决方案提供节省成本的重要优势，因为无服务器实现在每个用即付模型中。</span><span class="sxs-lookup"><span data-stu-id="edd67-107">Serverless solutions provide the important benefit of cost savings because serverless is implemented in a pay-per-use model.</span></span> <span data-ttu-id="edd67-108">无服务器使可能独立缩放、 测试和部署你的应用程序的各个组件。</span><span class="sxs-lookup"><span data-stu-id="edd67-108">Serverless makes it possible to independently scale, test, and deploy individual components of your application.</span></span> <span data-ttu-id="edd67-109">无服务器是唯一适合实现微服务体系结构和完全集成到 DevOps 管道。</span><span class="sxs-lookup"><span data-stu-id="edd67-109">Serverless is uniquely suited to implement microservices architectures and integrates fully into a DevOps pipeline.</span></span>

<span data-ttu-id="edd67-110">**作为一个单元部署的代码。**</span><span class="sxs-lookup"><span data-stu-id="edd67-110">**Code as a unit of deployment.**</span></span> <span data-ttu-id="edd67-111">无服务器提取硬件、 操作系统和离开应用程序的运行时。</span><span class="sxs-lookup"><span data-stu-id="edd67-111">Serverless abstracts the hardware, OS, and runtime away from the application.</span></span> <span data-ttu-id="edd67-112">无服务器，部署单元的将重点放在代码中的业务逻辑。</span><span class="sxs-lookup"><span data-stu-id="edd67-112">Serverless enables focusing on business logic in code as the unit of deployment.</span></span>

<span data-ttu-id="edd67-113">**触发器和绑定。**</span><span class="sxs-lookup"><span data-stu-id="edd67-113">**Triggers and bindings.**</span></span> <span data-ttu-id="edd67-114">无服务器方便地存储、 Api 和其他云资源与集成。</span><span class="sxs-lookup"><span data-stu-id="edd67-114">Serverless eases integration with storage, APIs, and other cloud resources.</span></span> <span data-ttu-id="edd67-115">Azure Functions 提供触发器，以执行代码和绑定与资源进行交互。</span><span class="sxs-lookup"><span data-stu-id="edd67-115">Azure Functions provides triggers to execute code and bindings to interact with resources.</span></span>

<span data-ttu-id="edd67-116">**微服务。**</span><span class="sxs-lookup"><span data-stu-id="edd67-116">**Microservices.**</span></span> <span data-ttu-id="edd67-117">微服务体系结构正成为基于多个独立子系统形式是自治服务的分布式和大型或复杂任务关键应用的首选的方法。</span><span class="sxs-lookup"><span data-stu-id="edd67-117">The microservices architecture is becoming the preferred approach for distributed and large or complex mission-critical applications that are based on multiple independent subsystems in the form of autonomous services.</span></span> <span data-ttu-id="edd67-118">在基于微服务的体系结构，该应用程序生成为可在开发、 测试、 版本控制、 部署和缩放独立的服务集合。</span><span class="sxs-lookup"><span data-stu-id="edd67-118">In a microservice-based architecture, the application is built as a collection of services that can be developed, tested, versioned, deployed, and scaled independently.</span></span> <span data-ttu-id="edd67-119">无服务器是非常适用于构建这些服务的体系结构。</span><span class="sxs-lookup"><span data-stu-id="edd67-119">Serverless is an architecture well-suited for building these services.</span></span>

<span data-ttu-id="edd67-120">**无服务器平台。**</span><span class="sxs-lookup"><span data-stu-id="edd67-120">**Serverless platforms.**</span></span> <span data-ttu-id="edd67-121">无服务器并不是不仅仅是代码。</span><span class="sxs-lookup"><span data-stu-id="edd67-121">Serverless isn't just about the code.</span></span> <span data-ttu-id="edd67-122">支持无服务器体系结构的平台包括无服务器工作流和业务流程、 无服务器消息事件、 服务和无服务器数据库。</span><span class="sxs-lookup"><span data-stu-id="edd67-122">Platforms that support serverless architectures include serverless workflows and orchestration, serverless messaging and event services, and serverless databases.</span></span>

<span data-ttu-id="edd67-123">**无服务器的挑战。**</span><span class="sxs-lookup"><span data-stu-id="edd67-123">**Serverless challenges.**</span></span> <span data-ttu-id="edd67-124">无服务器引入了挑战与分布式应用程序开发，例如零碎和独立的数据模型、 复原能力、 版本和服务发现。</span><span class="sxs-lookup"><span data-stu-id="edd67-124">Serverless introduces challenges related to distributed application development, such as fragmented and independent data models, resiliency, versioning, and service discovery.</span></span> <span data-ttu-id="edd67-125">无服务器可能不理想情况下适合长时间运行进程或受益于更紧密耦合的组件。</span><span class="sxs-lookup"><span data-stu-id="edd67-125">Serverless may not be ideally suited to long running processes or components that benefit from tighter coupling.</span></span>

<span data-ttu-id="edd67-126">**作为一种工具，不是工具箱中无服务器。**</span><span class="sxs-lookup"><span data-stu-id="edd67-126">**Serverless as a tool, not the toolbox.**</span></span> <span data-ttu-id="edd67-127">无服务器是为应用程序体系结构不独占的解决方案。</span><span class="sxs-lookup"><span data-stu-id="edd67-127">Serverless is not the exclusive solution to application architecture.</span></span> <span data-ttu-id="edd67-128">它是一种工具，可以利用传统的层，整体应用程序后端和容器可能包含混合应用程序的一部分。</span><span class="sxs-lookup"><span data-stu-id="edd67-128">It is a tool that can be leveraged as part of a hybrid application that may contain traditional tiers, monolith back ends, and containers.</span></span> <span data-ttu-id="edd67-129">无服务器可用于增强现有解决方案并不是应用程序开发、 孤注一掷的方法。</span><span class="sxs-lookup"><span data-stu-id="edd67-129">Serverless can be used to enhance existing solutions and is not an all-or-nothing approach to application development.</span></span>
