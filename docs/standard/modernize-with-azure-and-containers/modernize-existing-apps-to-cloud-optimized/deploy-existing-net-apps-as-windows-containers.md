## <a name="windows-container-types"></a><span data-ttu-id="35097-209">Windows 容器类型</span><span class="sxs-lookup"><span data-stu-id="35097-209">Windows container types</span></span>

<span data-ttu-id="35097-210">Linux 容器，如 Windows Server 容器通过 Docker 引擎管理。</span><span class="sxs-lookup"><span data-stu-id="35097-210">Like Linux containers, Windows Server containers are managed by using Docker Engine.</span></span> <span data-ttu-id="35097-211">与 Linux 容器不同 Windows 容器包括两种不同的容器类型，或运行时间 Windows Server 容器和 HYPER-V 隔离。</span><span class="sxs-lookup"><span data-stu-id="35097-211">Unlike Linux containers, Windows containers include two different container types, or run times-Windows Server containers and Hyper-V isolation.</span></span>

<span data-ttu-id="35097-212">**Windows Server 容器**:提供应用程序通过进程和命名空间隔离技术隔离。</span><span class="sxs-lookup"><span data-stu-id="35097-212">**Windows Server containers**: Provides application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="35097-213">Windows Server 容器与容器主机和主机运行的所有容器共享内核。</span><span class="sxs-lookup"><span data-stu-id="35097-213">A Windows Server container shares a kernel with the container host and all containers that are running on the host.</span></span> <span data-ttu-id="35097-214">这些容器不提供恶意安全边界，不应使用隔离不受信任的代码。</span><span class="sxs-lookup"><span data-stu-id="35097-214">These containers do not provide a hostile security boundary and should not be used to isolate untrusted code.</span></span> <span data-ttu-id="35097-215">由于共享的内核空间，这些容器需要相同的内核版本和配置。</span><span class="sxs-lookup"><span data-stu-id="35097-215">Because of the shared kernel space, these containers require the same kernel version and configuration.</span></span>

<span data-ttu-id="35097-216">**HYPER-V 隔离**:提供由 Windows Server 容器通过在高度优化的 VM 上运行每个容器的隔离上扩展。</span><span class="sxs-lookup"><span data-stu-id="35097-216">**Hyper-V isolation**: Expands on the isolation provided by Windows Server Containers by running each container on a highly optimized VM.</span></span> <span data-ttu-id="35097-217">在此配置中，与同一主机上的其他容器不共享容器主机的内核。</span><span class="sxs-lookup"><span data-stu-id="35097-217">In this configuration, the kernel of the container host is not shared with other containers on the same host.</span></span> <span data-ttu-id="35097-218">这些容器旨在用于恶意多租户托管，与 VM 的相同的安全保证。</span><span class="sxs-lookup"><span data-stu-id="35097-218">These containers are designed for hostile multitenant hosting, with the same security assurances of a VM.</span></span> <span data-ttu-id="35097-219">这些容器不与主机或主机上的其他容器共享内核，因为它们可以使用不同版本和配置 （具有支持的版本） 运行的内核。</span><span class="sxs-lookup"><span data-stu-id="35097-219">Because these containers don't share the kernel with the host or other containers on the host, they can run kernels with different versions and configurations (with supported versions).</span></span> <span data-ttu-id="35097-220">例如，Windows 10 上的所有 Windows 容器都使用的 HYPER-V 隔离来利用 Windows Server 内核版本和配置。</span><span class="sxs-lookup"><span data-stu-id="35097-220">For example, all Windows containers on Windows 10 use Hyper-V isolation to utilize the Windows Server kernel version and configuration.</span></span>

<span data-ttu-id="35097-221">在 Windows 上运行容器，带或不带 HYPER-V 隔离是运行时决定。</span><span class="sxs-lookup"><span data-stu-id="35097-221">Running a container on Windows with or without Hyper-V isolation is a run-time decision.</span></span> <span data-ttu-id="35097-222">可以选择最初，创建具有 HYPER-V 隔离的容器，并在运行时，选择为 Windows Server 容器运行它。</span><span class="sxs-lookup"><span data-stu-id="35097-222">You might choose to create the container with Hyper-V isolation initially, and at run time, choose to run it as a Windows Server container instead.</span></span>
