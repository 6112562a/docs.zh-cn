## <a name="monolithic-application-deployed-as-a-container"></a><span data-ttu-id="c0079-137">部署为容器的整体式应用程序</span><span class="sxs-lookup"><span data-stu-id="c0079-137">Monolithic application deployed as a container</span></span>

<span data-ttu-id="c0079-138">没有为使用容器管理整体式部署带来的好处。</span><span class="sxs-lookup"><span data-stu-id="c0079-138">There are benefits to using containers to manage monolithic deployments.</span></span> <span data-ttu-id="c0079-139">缩放容器实例比部署额外的 VM 要快得多，也容易得多。</span><span class="sxs-lookup"><span data-stu-id="c0079-139">Scaling the instances of containers is far faster and easier than deploying additional VMs.</span></span> <span data-ttu-id="c0079-140">尽管 VM 规模集是一个非常不错的功能来缩放 Vm，所需托管 Docker 容器，但它们需要时间来设置。</span><span class="sxs-lookup"><span data-stu-id="c0079-140">Although VM Scale Sets are a great feature to scale VMs, which are required to host your Docker containers, they take time to set up.</span></span> <span data-ttu-id="c0079-141">部署为应用实例时，应用的配置将作为 VM 的一部分进行管理。</span><span class="sxs-lookup"><span data-stu-id="c0079-141">When deployed as app instances, the configuration of the app is managed as part of the VM.</span></span>

<span data-ttu-id="c0079-142">将更新部署为 Docker 映像会快得多，并且网络效率更高。</span><span class="sxs-lookup"><span data-stu-id="c0079-142">Deploying updates as Docker images is far faster and network efficient.</span></span> <span data-ttu-id="c0079-143">可以消除导致的更多 Vm 添加了的成本随着 Vn 1 实例，在同一主机上设置 Vn 实例。</span><span class="sxs-lookup"><span data-stu-id="c0079-143">The Vn instances can be set up on the same hosts as your Vn-1 instances, eliminating added costs resulting from additional VMs.</span></span> <span data-ttu-id="c0079-144">Docker 映像通常会启动以秒为单位，加快了推出速度。</span><span class="sxs-lookup"><span data-stu-id="c0079-144">Docker images typically start in seconds, speeding rollouts.</span></span> <span data-ttu-id="c0079-145">拆除 Docker 实例非常简单，只调用`docker stop`命令，通常少于一秒内完成。</span><span class="sxs-lookup"><span data-stu-id="c0079-145">Tearing down a Docker instance is as easy as invoking the `docker stop` command, typically completing in less than a second.</span></span>

<span data-ttu-id="c0079-146">由于容器本质上是不可变，按照设计，永远不需要担心 Vm 损坏，因为更新脚本忘记考虑某些特定配置或磁盘上剩余的文件。</span><span class="sxs-lookup"><span data-stu-id="c0079-146">Because containers are inherently immutable, by design, you never need to worry about corrupted VMs because an update script forgot to account for some specific configuration or file left on disk.</span></span>

<span data-ttu-id="c0079-147">尽管整体式应用可能得益于 Docker，但我们触摸上仅优势的提示。</span><span class="sxs-lookup"><span data-stu-id="c0079-147">Although monolithic apps can benefit from Docker, we're touching on only the tips of the benefits.</span></span> <span data-ttu-id="c0079-148">使用管理各种实例和生命周期的每个容器实例的容器协调器部署来自管理容器的更大的好处。</span><span class="sxs-lookup"><span data-stu-id="c0079-148">The larger benefits of managing containers comes from deploying with container orchestrators that manage the various instances and life cycle of each container instance.</span></span> <span data-ttu-id="c0079-149">将整体式应用程序分解为可以单独缩放、开发和部署的子系统是进入微服务领域的切入点。</span><span class="sxs-lookup"><span data-stu-id="c0079-149">Breaking up the monolithic application into subsystems that can be scaled, developed, and deployed individually is your entry point into the realm of microservices.</span></span>
