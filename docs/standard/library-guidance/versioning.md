<span data-ttu-id="c9422-120">由于 NuGet 包版本是开发人员最易于可见的版本，因此最好使用[语义化版本控制 (SemVer)](https://semver.org/) 进行更新。</span><span class="sxs-lookup"><span data-stu-id="c9422-120">Because the NuGet package version is the most visible version to developers, it's a good idea to update it using [Semantic Versioning (SemVer)](https://semver.org/).</span></span> <span data-ttu-id="c9422-121">SemVer 指出版本之间的重大更改，并且可以帮助开发人员在选择要使用哪个版本时做出明智的决策。</span><span class="sxs-lookup"><span data-stu-id="c9422-121">SemVer indicates the significance of changes between release and helps developers make an informed decision when choosing what version to use.</span></span> <span data-ttu-id="c9422-122">例如，从 `1.0` 到 `2.0` 指示可能有重大更改。</span><span class="sxs-lookup"><span data-stu-id="c9422-122">For example, going from `1.0` to `2.0` indicates that there are potentially breaking changes.</span></span>

### <a name="assembly-version"></a><span data-ttu-id="c9422-127">程序集版本</span><span class="sxs-lookup"><span data-stu-id="c9422-127">Assembly version</span></span>

<span data-ttu-id="c9422-128">程序集版本是 CLR 在运行时用来选择要加载哪个程序集版本的依据。</span><span class="sxs-lookup"><span data-stu-id="c9422-128">The assembly version is what the CLR uses at runtime to select which version of an assembly to load.</span></span> <span data-ttu-id="c9422-129">使用版本控制选择程序集仅适用于具有强名称的程序集。</span><span class="sxs-lookup"><span data-stu-id="c9422-129">Selecting an assembly using versioning only applies to assemblies with a strong name.</span></span>

```xml
<AssemblyVersion>1.0.0.0</AssemblyVersion>
```

<span data-ttu-id="c9422-130">Windows .NET Framework CLR 要求完全匹配以便加载具有强名称的程序集。</span><span class="sxs-lookup"><span data-stu-id="c9422-130">The Windows .NET Framework CLR demands an exact match to load a strong named assembly.</span></span> <span data-ttu-id="c9422-131">例如，`Libary1, Version=1.0.0.0` 已使用对 `Newtonsoft.Json, Version=11.0.0.0` 的引用进行编译。</span><span class="sxs-lookup"><span data-stu-id="c9422-131">For example, `Libary1, Version=1.0.0.0` was compiled with a reference to `Newtonsoft.Json, Version=11.0.0.0`.</span></span> <span data-ttu-id="c9422-132">.NET Framework 将仅加载该确切版本 `11.0.0.0`。</span><span class="sxs-lookup"><span data-stu-id="c9422-132">The .NET Framework will only load that exact version `11.0.0.0`.</span></span> <span data-ttu-id="c9422-133">若要在运行时加载其他版本，必须向 .NET 应用程序的配置文件添加绑定重定向。</span><span class="sxs-lookup"><span data-stu-id="c9422-133">To load a different version at runtime, a binding redirect must be added to the .NET application's config file.</span></span>

<span data-ttu-id="c9422-134">与程序集版本组合使用的强命名启用[严格的程序集版本加载](../assembly/versioning.md)。</span><span class="sxs-lookup"><span data-stu-id="c9422-134">Strong naming combined with assembly version enables [strict assembly version loading](../assembly/versioning.md).</span></span> <span data-ttu-id="c9422-135">虽然强命名库具有很多好处，但通常会导致出现找不到程序集的运行时异常，并且在要修复的 `app.config`/`web.config` 中[需要绑定重定向](../../framework/configure-apps/redirect-assembly-versions.md)。</span><span class="sxs-lookup"><span data-stu-id="c9422-135">While strong naming a library has a number of benefits, it often results in runtime exceptions that an assembly can't be found and [requires binding redirects](../../framework/configure-apps/redirect-assembly-versions.md) in `app.config`/`web.config` to be fixed.</span></span> <span data-ttu-id="c9422-136">已放宽 .NET Core 程序集加载，并且 .NET Core CLR 将在运行时自动加载更高版本的程序集。</span><span class="sxs-lookup"><span data-stu-id="c9422-136">.NET Core assembly loading has been relaxed, and the .NET Core CLR will automatically load assemblies at runtime with a higher version.</span></span>

<span data-ttu-id="c9422-137"> ✔️请考虑在 AssemblyVersion 中仅包括主版本。</span><span class="sxs-lookup"><span data-stu-id="c9422-137">**✔️ CONSIDER** only including a major version in the AssemblyVersion.</span></span>

> <span data-ttu-id="c9422-138">例如，库 1.0 和库 1.0.1 都具有 AssemblyVersion `1.0.0.0`，而库 2.0 具有 AssemblyVersion `2.0.0.0`。</span><span class="sxs-lookup"><span data-stu-id="c9422-138">e.g. Library 1.0 and Library 1.0.1 both have an AssemblyVersion of `1.0.0.0`, while Library 2.0 has AssemblyVersion of `2.0.0.0`.</span></span> <span data-ttu-id="c9422-139">当程序集版本不常更改时，会减少绑定重定向。</span><span class="sxs-lookup"><span data-stu-id="c9422-139">When the assembly version changes less often, it reduces binding redirects.</span></span>

<span data-ttu-id="c9422-140"> ✔️请考虑使 AssemblyVersion 的主版本号与 NuGet 包版本保持同步。</span><span class="sxs-lookup"><span data-stu-id="c9422-140">**✔️ CONSIDER** keeping the major version number of the AssemblyVersion and the NuGet package version in sync.</span></span>

> <span data-ttu-id="c9422-141">AssemblyVersion 包含在向用户显示的一些信息性消息中，例如异常消息中的程序集名称和程序集限定类型名称。</span><span class="sxs-lookup"><span data-stu-id="c9422-141">The AssemblyVersion is included in some informational messages displayed to the user, e.g. the assembly name and assembly qualified type names in exception messages.</span></span> <span data-ttu-id="c9422-142">维持版本之间的关系可向开发人员提供有关所用版本的详细信息。</span><span class="sxs-lookup"><span data-stu-id="c9422-142">Maintaining a relationship between the versions provides more information to developers about which version they are using.</span></span>

<span data-ttu-id="c9422-143"> ❌不要有固定的 AssemblyVersion。</span><span class="sxs-lookup"><span data-stu-id="c9422-143">**❌ DO NOT** have a fixed AssemblyVersion.</span></span>

> <span data-ttu-id="c9422-144">虽然不变的 AssemblyVersion 不需要绑定重定向，但意味着在全局程序集缓存 (GAC) 中只能安装单个程序集版本。</span><span class="sxs-lookup"><span data-stu-id="c9422-144">While an unchanging AssemblyVersion avoids the need for binding redirects, it means that only a single version of the assembly can be installed in the Global Assembly Cache (GAC).</span></span> <span data-ttu-id="c9422-145">此外，引用 GAC 中的程序集的应用程序将中断，前提是另一个应用程序使用重大更改更新 GAC 程序集。</span><span class="sxs-lookup"><span data-stu-id="c9422-145">Also, the applications that reference the assembly in the GAC will break if another application updates the GAC assembly with breaking changes.</span></span>
