## <a name="remarks"></a><span data-ttu-id="cb19b-111">备注</span><span class="sxs-lookup"><span data-stu-id="cb19b-111">Remarks</span></span>  
 <span data-ttu-id="cb19b-112">`-delaysign`选项没有任何影响，除非用于[-keyfile](../../../visual-basic/reference/command-line-compiler/keyfile.md)或[-keycontainer](../../../visual-basic/reference/command-line-compiler/keycontainer.md)。</span><span class="sxs-lookup"><span data-stu-id="cb19b-112">The `-delaysign` option has no effect unless used with [-keyfile](../../../visual-basic/reference/command-line-compiler/keyfile.md) or [-keycontainer](../../../visual-basic/reference/command-line-compiler/keycontainer.md).</span></span>  
  
 <span data-ttu-id="cb19b-113">在请求完全签名的程序集时，编译器会对包含清单（程序集元数据）的文件进行哈希处理，并使用私钥对哈希进行签名。</span><span class="sxs-lookup"><span data-stu-id="cb19b-113">When you request a fully signed assembly, the compiler hashes the file that contains the manifest (assembly metadata) and signs that hash with the private key.</span></span> <span data-ttu-id="cb19b-114">产生的数字签名存储在包含清单的文件中。</span><span class="sxs-lookup"><span data-stu-id="cb19b-114">The resulting digital signature is stored in the file that contains the manifest.</span></span> <span data-ttu-id="cb19b-115">程序集延迟签名时，编译器不会计算和存储文件中的签名，但预留空间，以便可以稍后添加该签名。</span><span class="sxs-lookup"><span data-stu-id="cb19b-115">When an assembly is delay signed, the compiler does not compute and store the signature but reserves space in the file so the signature can be added later.</span></span>  
  
 <span data-ttu-id="cb19b-116">例如，通过使用`-delaysign+`，在组织中的开发人员可以分发测试人员可注册到全局程序集缓存和使用程序集的无符号的测试版本。</span><span class="sxs-lookup"><span data-stu-id="cb19b-116">For example, by using `-delaysign+`, a developer in an organization can distribute unsigned test versions of an assembly that testers can register with the global assembly cache and use.</span></span> <span data-ttu-id="cb19b-117">在完成工作的程序集时，负责组织的私钥的人员完全可以登录该程序集。</span><span class="sxs-lookup"><span data-stu-id="cb19b-117">When work on the assembly is completed, the person responsible for the organization's private key can fully sign the assembly.</span></span> <span data-ttu-id="cb19b-118">这种划分可防止组织的专用密钥泄露，同时允许所有开发人员若要运行的程序集。</span><span class="sxs-lookup"><span data-stu-id="cb19b-118">This compartmentalization protects the organization's private key from disclosure, while allowing all developers to work on the assemblies.</span></span>  
  
 <span data-ttu-id="cb19b-119">请参阅[创建和使用具有强名称程序集](../../../framework/app-domains/create-and-use-strong-named-assemblies.md)有关为程序集签名的详细信息。</span><span class="sxs-lookup"><span data-stu-id="cb19b-119">See [Creating and Using Strong-Named Assemblies](../../../framework/app-domains/create-and-use-strong-named-assemblies.md) for more information on signing an assembly.</span></span>
