# <a name="type-argument-typeargumentname-is-declared-mustinherit-and-does-not-satisfy-the-new-constraint-for-type-parameter-typeparametername"></a><span data-ttu-id="76106-102">类型实参\<g u > 声明为 MustInherit 并且不满足类型参数的 New 约束\<typeparametername ></span><span class="sxs-lookup"><span data-stu-id="76106-102">Type argument '\<typeargumentname>' is declared 'MustInherit' and does not satisfy the 'New' constraint for type parameter '\<typeparametername>'</span></span>
<span data-ttu-id="76106-103">使用 `MustInherit` 类将泛型类型调用为类型实参，而使用 `New` 约束声明相应的类型形参。</span><span class="sxs-lookup"><span data-stu-id="76106-103">A generic type is invoked with a `MustInherit` class as a type argument, while the corresponding type parameter is declared with the `New` constraint.</span></span>  
  
 <span data-ttu-id="76106-104">`New` 约束要求在相应类型参数中传递的类型必须支持创建对象。</span><span class="sxs-lookup"><span data-stu-id="76106-104">The `New` constraint requires that the type passed in the corresponding type argument must support the creation of objects.</span></span> <span data-ttu-id="76106-105">但是， *“抽象”* 类（即声明为 `MustInherit`的类）不公开任何构造函数，因为你不能从它创建任何对象。</span><span class="sxs-lookup"><span data-stu-id="76106-105">However, an *abstract* class, that is, a class declared as `MustInherit`, does not expose any constructors because you cannot create any objects from it.</span></span>  
  
 <span data-ttu-id="76106-106">**错误 ID:** BC32082</span><span class="sxs-lookup"><span data-stu-id="76106-106">**Error ID:** BC32082</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="76106-107">更正此错误</span><span class="sxs-lookup"><span data-stu-id="76106-107">To correct this error</span></span>

3. <span data-ttu-id="76106-110">如果相应的类型参数不需要从传递到它的类型创建任何对象，则从其声明中删除 `New` 约束。</span><span class="sxs-lookup"><span data-stu-id="76106-110">If the corresponding type parameter does not need to create any objects from the type passed to it, then remove the `New` constraint from its declaration.</span></span>
