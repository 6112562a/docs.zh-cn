 <span data-ttu-id="ff395-104">当变量或表达式的计算结果为 `Object`时，编译器必须执行 *后期绑定*，这将导致在运行时产生额外的操作。</span><span class="sxs-lookup"><span data-stu-id="ff395-104">When a variable or expression evaluates to `Object`, the compiler must perform *late binding*, which causes extra operations at run time.</span></span> <span data-ttu-id="ff395-105">它还使应用程序易于发生潜在的运行时错误。</span><span class="sxs-lookup"><span data-stu-id="ff395-105">It also exposes your application to potential run-time errors.</span></span> <span data-ttu-id="ff395-106">例如，如果将 <xref:System.Windows.Forms.Form> 分配到 `Object` 变量，然后尝试将它与数字进行比较，则运行时会引发 <xref:System.InvalidCastException> ，因为 Visual Basic 不能将 <xref:System.Windows.Forms.Form> 对象转换为数字值。</span><span class="sxs-lookup"><span data-stu-id="ff395-106">For example, if you assign a <xref:System.Windows.Forms.Form> to an `Object` variable and then try to compare it with a number, the runtime throws an <xref:System.InvalidCastException> because Visual Basic cannot convert a <xref:System.Windows.Forms.Form> object to a numeric value.</span></span>  
  
 <span data-ttu-id="ff395-107">`Select`...`Case` 构造中的表达式均应属于相同的数据类型或者属于能够彼此相互转换的密切相关的数据类型。</span><span class="sxs-lookup"><span data-stu-id="ff395-107">The expressions in a `Select`...`Case` construction should all be of the same data type or of closely related data types that can be converted to each other.</span></span> <span data-ttu-id="ff395-108">这是因为每个 `Case` 语句将针对 `Select`...`Case` 构造基于的测试表达式至少比较一个值。</span><span class="sxs-lookup"><span data-stu-id="ff395-108">This is because each `Case` statement compares at least one value against the test expression on which the `Select`...`Case` construction is based.</span></span>  
  
 <span data-ttu-id="ff395-109">默认情况下，此消息是一个警告。</span><span class="sxs-lookup"><span data-stu-id="ff395-109">By default, this message is a warning.</span></span> <span data-ttu-id="ff395-110">有关隐藏警告或将警告视为错误的信息，请参见 [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic)。</span><span class="sxs-lookup"><span data-stu-id="ff395-110">For information on hiding warnings or treating warnings as errors, see [Configuring Warnings in Visual Basic](/visualstudio/ide/configuring-warnings-in-visual-basic).</span></span>
