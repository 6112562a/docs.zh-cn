 <span data-ttu-id="a30de-104">在调用泛型过程时，通常会为过程所定义的每个类型形参提供类型实参。</span><span class="sxs-lookup"><span data-stu-id="a30de-104">When you call a generic procedure, you normally supply a type argument for each type parameter defined by the procedure.</span></span> <span data-ttu-id="a30de-105">但是，还可以选择忽略整个类型实参列表。</span><span class="sxs-lookup"><span data-stu-id="a30de-105">However, you have the alternative of omitting the type argument list entirely.</span></span> <span data-ttu-id="a30de-106">执行此操作时，编译器将尝试推断调用上下文中每个类型实参的类型。</span><span class="sxs-lookup"><span data-stu-id="a30de-106">When you do this, the compiler attempts to infer the type of each type argument from the context of your call.</span></span> <span data-ttu-id="a30de-107">有关详细信息，请参见 [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md)中的“类型推理”。</span><span class="sxs-lookup"><span data-stu-id="a30de-107">For more information, see "Type Inference" in [Generic Procedures in Visual Basic](../../visual-basic/programming-guide/language-features/data-types/generic-procedures.md).</span></span>
