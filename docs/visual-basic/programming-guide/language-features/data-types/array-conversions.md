## <a name="conversion-to-an-object-array"></a><span data-ttu-id="7909b-117">转换为对象数组</span><span class="sxs-lookup"><span data-stu-id="7909b-117">Conversion to an Object Array</span></span>  
 <span data-ttu-id="7909b-118">当你声明`Object`而无需初始化它，其元素类型的数组是`Object`，只要它保持未初始化。</span><span class="sxs-lookup"><span data-stu-id="7909b-118">When you declare an `Object` array without initializing it, its element type is `Object` as long as it remains uninitialized.</span></span> <span data-ttu-id="7909b-119">当将其设置为特定类的数组时，它将采用该类的类型。</span><span class="sxs-lookup"><span data-stu-id="7909b-119">When you set it to an array of a specific class, it takes on the type of that class.</span></span> <span data-ttu-id="7909b-120">但是，其基础类型仍是`Object`，随后可以将其设置为一个不相关的类的另一个数组。</span><span class="sxs-lookup"><span data-stu-id="7909b-120">However, its underlying type is still `Object`, and you can subsequently set it to another array of an unrelated class.</span></span> <span data-ttu-id="7909b-121">由于所有类都派生自`Object`，可以从任何类更改数组的元素类型，与其他任何类。</span><span class="sxs-lookup"><span data-stu-id="7909b-121">Since all classes derive from `Object`, you can change the array's element type from any class to any other class.</span></span>  
  
 <span data-ttu-id="7909b-122">在以下示例中，不存在转换类型之间`student`并`String`，但同时派生`Object`，因此，所有分配都都有效。</span><span class="sxs-lookup"><span data-stu-id="7909b-122">In the following example, no conversion exists between types `student` and `String`, but both derive from `Object`, so all assignments are valid.</span></span>  
  
```  
' Assume student has already been defined as a class.  
Dim testArray() As Object  
' testArray is still an Object array at this point.  
Dim names() As String = New String(3) {"Name0", "Name1", "Name2", "Name3"}  
testArray = New student(3) {}  
' testArray is now of type student().  
testArray = names  
' testArray is now a String array.  
```  
  
### <a name="underlying-type-of-an-array"></a><span data-ttu-id="7909b-123">数组的基础类型。</span><span class="sxs-lookup"><span data-stu-id="7909b-123">Underlying Type of an Array</span></span>  
 <span data-ttu-id="7909b-124">如果您最初声明具有特定类的数组，其基础的元素类型是该类。</span><span class="sxs-lookup"><span data-stu-id="7909b-124">If you originally declare an array with a specific class, its underlying element type is that class.</span></span> <span data-ttu-id="7909b-125">如果您随后将其设置到另一个类的一个数组，必须有两个类之间的转换。</span><span class="sxs-lookup"><span data-stu-id="7909b-125">If you subsequently set it to an array of another class, there must be a conversion between the two classes.</span></span>  
  
 <span data-ttu-id="7909b-126">在以下示例中，`students`是`student`数组。</span><span class="sxs-lookup"><span data-stu-id="7909b-126">In the following example, `students` is a `student` array.</span></span> <span data-ttu-id="7909b-127">由于不存在转换之间`String`和`student`，最后一个语句将失败。</span><span class="sxs-lookup"><span data-stu-id="7909b-127">Since no conversion exists between `String` and `student`, the last statement fails.</span></span>  
  
```  
Dim students() As student  
Dim names() As String = New String(3) {"Name0", "Name1", "Name2", "Name3"}  
students = New Student(3) {}  
' The following statement fails at compile time.

```
