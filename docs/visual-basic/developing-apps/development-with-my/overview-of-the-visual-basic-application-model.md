 <span data-ttu-id="75766-108">典型的应用程序在启动和关闭时需要执行任务。</span><span class="sxs-lookup"><span data-stu-id="75766-108">A typical application needs to perform tasks when it starts up and shuts down.</span></span> <span data-ttu-id="75766-109">例如，当启动时，应用程序可以显示初始屏幕、进行数据库连接、加载已保存状态等。</span><span class="sxs-lookup"><span data-stu-id="75766-109">For example, when it starts up, the application can display a splash screen, make database connections, load a saved state, and so on.</span></span> <span data-ttu-id="75766-110">当应用程序关闭时，它可以关闭数据库连接，保存当前状态，等等。</span><span class="sxs-lookup"><span data-stu-id="75766-110">When the application shuts down, it can close database connections, save the current state, and so on.</span></span> <span data-ttu-id="75766-111">此外，应用程序可以在应用程序意外关闭时（例如在未经处理的异常期间）执行特定代码。</span><span class="sxs-lookup"><span data-stu-id="75766-111">In addition, the application can execute specific code when the application shuts down unexpectedly, such as during an unhandled exception.</span></span>

 <span data-ttu-id="75766-124">通过处理 `StartupNextInstance` 事件，可以控制单实例应用程序的行为方式。</span><span class="sxs-lookup"><span data-stu-id="75766-124">By handling the `StartupNextInstance` event, you can control how your single-instance application behaves.</span></span> <span data-ttu-id="75766-125">例如，Microsoft Outlook 通常作为单实例应用程序运行;当 Outlook 正在运行并且您尝试再次启动 Outlook 时，焦点将移到原始实例上，但其他实例不会打开。</span><span class="sxs-lookup"><span data-stu-id="75766-125">For example, Microsoft Outlook typically runs as a single-instance application; when Outlook is running and you attempt to start Outlook again, focus shifts to the original instance but another instance does not open.</span></span>
